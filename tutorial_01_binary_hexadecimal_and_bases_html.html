Binary, Hexadecimal, and Bases

Learning to read the machine’s handwriting: bits, bytes, hex values, place value, and fixed-width numbers.

Roadmap Practice One-page recap

Why this tutorial exists

Purpose

Cryptography constantly talks in bits, bytes, hexadecimal values, and fixed-width chunks of data. If those representations feel slippery, the rest of cryptography becomes harder than it needs to be. This tutorial builds fluency with the number systems that modern ciphers use every day.

The goal is not to memorize conversion tricks. The goal is to see that binary, decimal, and hexadecimal are different written languages for the same underlying quantities. A value can wear many outfits. The value is the person underneath.

What you should know first

This tutorial assumes only ordinary counting and basic arithmetic. You do not need algebra, programming, or prior cryptography knowledge.

The big idea

Intuition first

A base is a counting rhythm. Base 10 counts with ten symbols before carrying into the next place. Base 2 counts with two symbols. Base 16 counts with sixteen symbols. The rhythm changes, but the idea of place value stays the same.

Place value: the shared skeleton

A positional number system uses positions. Each position has a value based on a power of the base. The rightmost position is position 0. Moving left increases the position number by 1.

Definition: positional notation

In base \( b \), the digit in position \( k \) is multiplied by \( b^k \). The total value is found by adding all of those digit-times-place-value contributions.

\[ \text{value} = d_m b^m + d_{m-1}b^{m-1} + \cdots + d_2b^2 + d_1b^1 + d_0b^0 \]

Here, each \( d_i \) is a digit, and the subscript tells which position that digit occupies.

Decimal example

The decimal number \(345_{10}\) means:

\[ 345_{10} = 3\cdot 10^2 + 4\cdot 10^1 + 5\cdot 10^0 \] \[ 345_{10} = 3\cdot 100 + 4\cdot 10 + 5\cdot 1 \]

Binary and hexadecimal use the same structure. Only the base changes.

Binary: base 2

Binary uses only two digits: 0 and 1. Because there are only two symbols, each position is a power of 2.

Technical definition

Binary is a base-2 positional numeral system. Its allowed digits are \(0\) and \(1\). In position \(k\), a binary digit contributes either \(0\cdot 2^k\) or \(1\cdot 2^k\).

Why hex is so useful

One hex digit exactly matches one nibble, which is 4 bits. Two hex digits exactly match one byte, which is 8 bits. Hex is compact, but it still lines up cleanly with binary. It is a tidy translation, not a mysterious new kind of number.

Nibbles and bytes

A byte has 8 bits. Since a nibble has 4 bits, a byte can be split into two nibbles. Each nibble becomes one hexadecimal digit.

Therefore:

\[ 00101101_2 = \texttt{0x2D} \]

This is why cryptography examples often write byte values in hex. The byte 00101101 is much easier to read as 0x2D.

Worked example 1: binary to decimal

Convert the binary number 101101 into decimal.

Step 1: Write the place values

The number has six bits, so its positions are 5 through 0.

\[ 101101_2 = 1\cdot 2^5 + 0\cdot 2^4 + 1\cdot 2^3 + 1\cdot 2^2 + 0\cdot 2^1 + 1\cdot 2^0 \]

Step 2: Keep only the positions with a 1

\[ 101101_2 = 2^5 + 2^3 + 2^2 + 2^0 \]

Step 3: Substitute the values

\[ 101101_2 = 32 + 8 + 4 + 1 = 45_{10} \]

Answer

The binary number 101101 represents the decimal value \(45\).

Worked example 2: decimal to binary

Convert \(45_{10}\) into binary.

Step 1: Find the largest power of 2 that fits

The largest power of 2 less than or equal to 45 is 32.

\[ 2^5 = 32 \]

Step 2: Subtract powers of 2

\[ 45 - 32 = 13 \] \[ 13 - 8 = 5 \] \[ 5 - 4 = 1 \] \[ 1 - 1 = 0 \]

Step 3: Mark the powers used

We used \(2^5\), \(2^3\), \(2^2\), and \(2^0\). We did not use \(2^4\) or \(2^1\).

Power \(2^5\) \(2^4\) \(2^3\) \(2^2\) \(2^1\) \(2^0\)
Value 32 16 8 4 2 1
Used? 1 0 1 1 0 1

Answer

\[ 45_{10} = 101101_2 \]

Worked example 3: binary to hexadecimal

Convert 01010111 into hexadecimal.

Step 1: Split the byte into two nibbles

\[ 01010111_2 = 0101\;0111_2 \]

Step 2: Convert each nibble

The left nibble 0101 is 5. The right nibble 0111 is 7.

Answer

\[ 01010111_2 = \texttt{0x57} \]

This value, 0x57, appears often in AES finite-field examples. Later, when we say 0x57, you should be able to see the byte hiding inside it: 01010111.

Fixed-width values and overflow

Computers often work with fixed-size containers. A byte has exactly 8 bits. That means a byte can store exactly \(2^8\) different patterns.

\[ 2^8 = 256 \]

Since counting starts at 0, the unsigned values of one byte are:

\[ 0 \text{ through } 255 \]
Definition: overflow

Overflow happens when a calculation produces a value too large for the fixed-width container. In an 8-bit byte, anything beyond 11111111 cannot fit unless the system uses a special rule for wrapping, truncating, or reducing the result.

Mental picture

Think of a byte as an eight-slot display. You can flip the slots on or off, but you cannot grow a ninth slot unless you move to a larger container. Cryptography often cares deeply about what happens when the ninth slot tries to appear.

Common mistakes

Mistake 1: treating hex as a different value

0x2D, 00101101, and \(45_{10}\) are not three different quantities. They are three different representations of the same quantity.

Mistake 2: forgetting that position 0 is worth 1

The rightmost binary position is \(2^0\), and \(2^0 = 1\). This is not a special exception. Any nonzero number to the zero power is 1.

Mistake 3: forgetting leading zeros

The value 101101 and the byte 00101101 have the same numeric value, but the byte form explicitly shows all 8 positions. In cryptography, width matters.

Practice problems

  1. Convert 1101 to decimal.
    Show solution

    1101 has 1 bits in positions 3, 2, and 0.

    \[ 1101_2 = 2^3 + 2^2 + 2^0 = 8 + 4 + 1 = 13_{10} \]
  2. Convert \(27_{10}\) to binary.
    Show solution

    Break 27 into powers of 2:

    \[ 27 = 16 + 8 + 2 + 1 \] \[ 27_{10} = 11011_2 \]
  3. Convert 10101110 to hexadecimal.
    Show solution

    Split the byte into nibbles:

    \[ 10101110_2 = 1010\;1110_2 \]

    1010 is A. 1110 is E.

    \[ 10101110_2 = \texttt{0xAE} \]
  4. Convert 0xC1 to binary.
    Show solution

    C is 1100. 1 is 0001.

    \[ \texttt{0xC1} = 11000001_2 \]
  5. What is the largest unsigned decimal value that fits in one byte?
    Show solution

    A byte has 8 bits, so it has \(2^8 = 256\) possible patterns. Since counting begins at 0, the largest value is:

    \[ 256 - 1 = 255 \]

Why this matters for cryptography

Cryptographic algorithms usually operate on fixed-size pieces of data. AES, for example, works heavily with bytes. Those bytes are often written in hexadecimal because hex is compact and maps cleanly to binary.

  • Binary shows the actual bits that bitwise operations manipulate.
  • Hexadecimal gives a readable shorthand for bytes and larger blocks.
  • Fixed-width thinking prepares you for overflow, modular arithmetic, and finite-field reduction.
  • Byte fluency makes AES examples such as 0x57 · 0x83 much easier to follow later.

One-page recap

What to remember
  • A base is the counting rhythm of a number system.
  • Binary is base 2 and uses only 0 and 1.
  • Decimal is base 10 and uses digits 0 through 9.
  • Hexadecimal is base 16 and uses 0 through 9, then A through F.
  • One nibble is 4 bits.
  • One byte is 8 bits.
  • One hex digit matches one nibble.
  • Two hex digits match one byte.
