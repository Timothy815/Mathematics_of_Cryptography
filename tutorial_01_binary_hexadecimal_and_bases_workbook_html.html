Part A: Warm-up vocabulary

A1. A bit is a single binary digit. Friendly: one tiny on/off value. Technical: one binary digit with value 0 or 1.

A2. A nibble is a group of 4 bits. It represents exactly one hexadecimal digit.

A3. A byte is a group of 8 bits. An unsigned byte can represent values from 0 through 255.

A4. Hexadecimal is useful because one hex digit represents one nibble, and two hex digits represent one byte. This makes bytes easier to read than long strings of bits.

Part C: Decimal to binary

C1. \(13 = 8 + 4 + 1 = 2^3 + 2^2 + 2^0\). Therefore \(13_{10} = 1101_2\).

C2. \(27 = 16 + 8 + 2 + 1 = 2^4 + 2^3 + 2^1 + 2^0\). Therefore \(27_{10} = 11011_2\).

C3. \(45 = 32 + 8 + 4 + 1 = 2^5 + 2^3 + 2^2 + 2^0\). As an 8-bit byte: \(00101101_2\).

C4. \(128 = 2^7\). As an 8-bit byte: \(10000000_2\).

C5. \(255 = 128 + 64 + 32 + 16 + 8 + 4 + 2 + 1\). As an 8-bit byte: \(11111111_2\).

E2 sample answer. Width matters because cryptography often works with fixed-size containers. The value 101101 and 00101101 have the same decimal value, 45, but 00101101 explicitly shows an 8-bit byte. The leading zeros help preserve the expected width.

E3 sample answer. A byte has \(2^8 = 256\) possible patterns. Since unsigned byte values start at 0, the values are 0 through 255. That is 256 total values, but the largest value is 255.

Part F: Challenge problems

F1. 0x57 → 5 is 0101, 7 is 0111, so binary is 01010111. Decimal: \(64 + 16 + 4 + 2 + 1 = 87\).

F2. 0x83 → 8 is 1000, 3 is 0011, so binary is 10000011. Decimal: \(128 + 2 + 1 = 131\).

F3 sample answer. 0x57 and 01010111 represent the same byte because each hex digit maps to one 4-bit nibble. Hex digit 5 maps to 0101, and hex digit 7 maps to 0111. Together they form 01010111.
