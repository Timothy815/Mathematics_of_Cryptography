Bitwise Operations as Algebra

Learning how XOR, AND, OR, NOT, shifts, rotations, and masks transform bits one position at a time.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

Modern cryptography is built from operations that act directly on bits. Before we can understand stream ciphers, block ciphers, substitution-permutation networks, finite fields, linear feedback shift registers, or key schedules, we need to understand what basic bitwise operations actually do.

A bitwise operation is a small rule applied across a row of bits. It is like giving every bit position its own tiny workstation. Each station looks only at the bit or bits in that position, follows the rule, and writes down a result.

What you should know first

You should be comfortable reading 8-bit binary values and hexadecimal byte notation such as 0x57. If that still feels foggy, review Tutorial 1 before continuing.

The big idea

Intuition first

Bitwise operations are not ordinary arithmetic. They do not add a whole byte the way decimal addition does. Instead, they line up two bit strings and apply a small rule to each column. Each column becomes its own little truth test.

XOR: the reversible difference operation

XOR, written \(\oplus\), outputs 1 when two bits are different and 0 when they are the same. For cryptography, XOR is especially important because it is reversible when one input is known.

Technical definition

XOR is addition modulo 2. That means the result is the remainder after ordinary bit addition is divided by 2.

\[ 0 \oplus 0 = 0, \quad 0 \oplus 1 = 1, \quad 1 \oplus 0 = 1, \quad 1 \oplus 1 = 0 \]

Worked example: \(0x57 \oplus 0x83\)

\[ \texttt{0x57} = 01010111_2 \] \[ \texttt{0x83} = 10000011_2 \]

Why XOR reverses itself

XOR is like a toggle. Toggling by the same pattern twice returns you to where you started.

\[ (A \oplus K) \oplus K = A \]

This is why XOR appears in one-time pads, stream ciphers, Feistel networks, and round-key mixing.

AND, OR, and NOT

XOR is the star of many cryptographic operations, but AND, OR, and NOT are also essential. They help select bits, combine conditions, build nonlinear Boolean functions, and create masks.

AND

AND outputs 1 only when both input bits are 1. It is often used for selecting bits.

\[ 1 \land 1 = 1, \quad 1 \land 0 = 0, \quad 0 \land 1 = 0, \quad 0 \land 0 = 0 \]

OR

OR outputs 1 when at least one input bit is 1. It is often used for forcing bits on.

\[ 1 \lor 1 = 1, \quad 1 \lor 0 = 1, \quad 0 \lor 1 = 1, \quad 0 \lor 0 = 0 \]

NOT

NOT flips each bit. In a fixed-width byte, every 0 becomes 1 and every 1 becomes 0.

\[ \lnot 01010111_2 = 10101000_2 \]
Important width warning

NOT depends on width. The complement of 01010111 as an 8-bit byte is 10101000. But if you used a different width, the result would show a different number of flipped positions. In cryptography, always ask: “How many bits wide is this value?”

Bit masks: choosing which bits matter

A mask is a bit pattern used to control another bit pattern. Masks can select bits, clear bits, force bits on, or test whether certain positions are set.

Definition: mask

A mask is a binary word used with a bitwise operation to affect specific bit positions while ignoring or preserving others.

Shifts: moving bits left and right

A shift moves every bit in a direction. In a logical shift, empty positions are filled with 0. Bits that fall off the edge are lost.

Definition: logical shift

A logical shift moves bits left or right by a chosen number of positions. New empty positions are filled with 0. Any bits pushed beyond the fixed width are discarded.

Multiplication and division intuition

For unsigned numbers, shifting left by 1 often behaves like multiplying by 2, and shifting right by 1 often behaves like dividing by 2 and dropping the remainder. But fixed width matters. If a bit falls off the edge, information is lost.

Rotations: moving bits without dropping them

A rotation also moves bits left or right, but instead of losing the bit that falls off the edge, it wraps around to the other side.

Definition: rotation

A rotation is a cyclic shift of a fixed-width bit string. Bits that move past one edge re-enter on the opposite edge.

Which operations are reversible?

Cryptography cares deeply about reversibility. If an encryption step loses information, decryption may become impossible unless the lost information is recoverable from somewhere else.

Operation Usually reversible? Why?
XOR with a known value Yes Applying the same XOR value again restores the original.
NOT Yes Flipping all bits twice restores the original.
Rotation Yes Rotate back in the opposite direction.
Logical shift No, generally Bits can fall off the edge and be lost.
AND mask No, generally Masked-out bits are cleared and cannot be recovered from the result alone.
OR mask No, generally Forced-on bits hide whether the original bits were 0 or 1.

A first cryptographic example: XOR with a key byte

Suppose a plaintext byte \(P\) is XORed with a key byte \(K\) to produce a ciphertext byte \(C\).

\[ C = P \oplus K \]

Let:

\[ P = \texttt{0x57}, \qquad K = \texttt{0x83} \]

Then:

\[ C = \texttt{0x57} \oplus \texttt{0x83} = \texttt{0xD4} \]

To decrypt, XOR the ciphertext with the same key:

\[ P = C \oplus K \] \[ P = \texttt{0xD4} \oplus \texttt{0x83} = \texttt{0x57} \]
The toggle metaphor

The key byte tells which bit positions to toggle. Toggling the same positions a second time undoes the change. This makes XOR one of the cleanest reversible operations in bit-level cryptography.

Common mistakes

Mistake 1: treating XOR like ordinary addition

In XOR, \(1 \oplus 1 = 0\). There is no carry. This is different from ordinary decimal addition, where \(1 + 1 = 2\).

Mistake 2: forgetting width

NOT, shifts, and rotations depend on the number of bits in the value. Always know whether you are working with 4 bits, 8 bits, 16 bits, or another width.

Mistake 3: assuming every bitwise operation is reversible

XOR, NOT, and rotations can be reversible. AND masks, OR masks, and logical shifts often lose information.

Why this matters for cryptography

Bitwise operations are the gears and levers of modern symmetric cryptography. They are small, fast, and mathematically understandable.

  • XOR is used for key mixing, one-time pads, stream ciphers, and Feistel networks.
  • AND, OR, and NOT help build Boolean functions and bit masks.
  • Shifts and rotations move information across positions.
  • Reversibility helps determine whether an operation can safely appear inside an encryption path.
  • Information-losing operations can still be useful in key schedules, hash functions, and round functions, but they must be used carefully.

One-page recap

What to remember
  • Bitwise operations act column by column on aligned bits.
  • XOR outputs 1 when bits are different.
  • AND outputs 1 only when both bits are 1.
  • OR outputs 1 when at least one bit is 1.
  • NOT flips bits, but the result depends on width.
  • Shifts move bits and may lose information.
  • Rotations move bits while wrapping edge bits around.
  • Masks use bit patterns to select, clear, or force positions.
