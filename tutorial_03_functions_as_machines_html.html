Functions as Machines

Understanding functions as input-output rules, then using that idea to explain encryption, decryption, composition, and invertibility.

🏠 Dashboard

Why this tutorial exists

Purpose

Cryptography is full of transformations. A plaintext becomes a ciphertext. A key becomes round keys. A byte becomes another byte through an S-box. A block passes through many rounds. The mathematical language for a predictable transformation is a function.

A function is a machine with a rule. You feed it an input, it follows its rule, and it produces an output. That simple idea becomes powerful because encryption itself can be understood as a carefully designed function.

What you should know first

You should understand that data can be represented as numbers, bits, bytes, and hexadecimal values. You should also be familiar with the idea that operations such as XOR transform bit patterns.

Symbols used in this tutorial

Function notation can look like tiny machinery labels. This table gives each label a friendly meaning and a precise meaning before we use it.

Symbol or notation Friendly meaning Technical meaning
\(f\) The name of a function, like the name on a machine. A rule that assigns each allowed input exactly one output.
\(x\) An input placed into the function. An element of the function’s domain.
\(f(x)\) The output produced when \(x\) goes into the function named \(f\). The value of function \(f\) evaluated at input \(x\).
\(A\) The set of allowed inputs. The domain of the function.
\(B\) The set where outputs are supposed to live. The codomain of the function.
\(f: A \to B\) The function \(f\) takes inputs from \(A\) and sends them into \(B\). A function with domain \(A\) and codomain \(B\).
\(g(f(x))\) First run \(x\) through \(f\), then run the result through \(g\). The composition of \(g\) after \(f\), often written \((g\circ f)(x)\).
\(f^{-1}\) The undoing machine for \(f\), if such a machine exists. The inverse function of \(f\).
\(E_K\) An encryption machine using key \(K\). An encryption function parameterized by key \(K\).
\(D_K\) A decryption machine using key \(K\). A decryption function parameterized by key \(K\), usually the inverse of \(E_K\).
\(P\) Plaintext, the original message or data. An element of the plaintext space.
\(C\) Ciphertext, the transformed encrypted data. An element of the ciphertext space.

The big idea

Intuition first

A function is like a dependable machine. The same input must always produce the same output. If the machine turns 3 into 10 today, it must turn 3 into 10 tomorrow, next Tuesday, and during a thunderstorm.

The key requirement is this: one allowed input cannot produce two different outputs. A function may send two different inputs to the same output, but one input cannot split into multiple answers.

Technical definition

More precise version

A function from a set \(A\) to a set \(B\) is a rule that assigns every element of \(A\) exactly one element of \(B\).

\[ f: A \to B \]

The set \(A\) is called the domain. The set \(B\) is called the codomain. If \(x\) is an input from \(A\), then \(f(x)\) is the output assigned to \(x\).

Domain, codomain, and range

Three terms are easy to blur together, so we will separate them carefully.

Worked example 1: a simple number function

Let the function \(f\) be defined by this rule:

\[ f(x) = 2x + 1 \]

Step 1: Understand the rule

The rule says: take the input, multiply it by 2, then add 1.

Step 2: Evaluate the function

If \(x = 4\), then:

\[ f(4) = 2(4) + 1 \] \[ f(4) = 8 + 1 = 9 \]

Step 3: Interpret the answer

The input 4 produces the output 9. In machine language: 4 goes in, 9 comes out.

Functions can be shown as tables

A function does not always need a formula. It can also be shown as a table, as long as every input has exactly one output.

Input \(x\) 0 1 2 3 4 5
Output \(f(x)\) 3 5 1 4 0 2

This kind of table is very close to how substitution boxes are represented. An S-box is a lookup function: input bits select an output value.

One-to-one functions and why they matter

A function is one-to-one if different inputs always produce different outputs. In plain language, no two inputs crash into the same output.

Definition: one-to-one

A function \(f\) is one-to-one if whenever \(x_1\neq x_2\), we also have \(f(x_1)\neq f(x_2)\).

One-to-one functions are important for encryption because if two different plaintexts produce the same ciphertext, decryption would not know which plaintext to return.

Information collision

If Alice and Bob both check their coats at a coatroom and receive the same ticket number, the ticket system is broken. The same idea applies to reversible encryption. If two inputs share one output, the output no longer contains enough information to distinguish the originals.

Onto functions

A function is onto if every value in the codomain is actually reached by some input.

Definition: onto

A function \(f: A\to B\) is onto if for every element \(y\) in \(B\), there is at least one element \(x\) in \(A\) such that \(f(x)=y\).

In many cryptographic settings, especially when encrypting fixed-size blocks, we want a transformation that rearranges all possible block values. That kind of function is both one-to-one and onto.

Permutations as functions

A permutation is a function that rearranges a finite set without losing or duplicating anything. Every input has one output, every output is used, and no two inputs collide.

Definition: permutation

A permutation of a finite set is a one-to-one and onto function from the set to itself.

Input position 0 1 2 3
Output position 2 0 3 1

This table says: item 0 goes to position 2, item 1 goes to position 0, item 2 goes to position 3, and item 3 goes to position 1. Since every output position appears exactly once, this is reversible.

Composition: feeding one machine into another

Composition means running functions in sequence. The output of one function becomes the input of the next function.

Definition: composition

The composition \(g\circ f\) means “do \(f\) first, then do \(g\).”

\[ (g\circ f)(x) = g(f(x)) \]

Worked example

Let:

\[ f(x)=x+3 \] \[ g(x)=2x \]

Now evaluate \((g\circ f)(5)\). Remember: do \(f\) first.

\[ f(5)=5+3=8 \] \[ g(f(5))=g(8)=2(8)=16 \] \[ (g\circ f)(5)=16 \]

Inverse functions: undoing the machine

An inverse function reverses what the original function did. If \(f\) sends \(x\) to \(y\), then \(f^{-1}\) sends \(y\) back to \(x\).

Definition: inverse function

A function \(f^{-1}\) is the inverse of \(f\) if applying \(f\) and then \(f^{-1}\) returns the original input.

\[ f^{-1}(f(x))=x \]

Worked example

Let:

\[ f(x)=x+3 \]

To undo “add 3,” we subtract 3. So:

\[ f^{-1}(x)=x-3 \]

Check it:

\[ f(10)=13 \] \[ f^{-1}(13)=10 \]

Encryption as a function

In cryptography, encryption can be described as a function that takes plaintext and produces ciphertext. The key controls which encryption function is being used.

\[ C = E_K(P) \]

Read this as: ciphertext \(C\) is produced by applying the encryption function \(E\), using key \(K\), to plaintext \(P\).

Worked example 2: XOR encryption as a function

Define an encryption function that XORs a byte with the key byte 0x83.

\[ E_K(P)=P\oplus K \]

Let:

\[ P=\texttt{0x57}, \qquad K=\texttt{0x83} \]

Then:

\[ E_K(\texttt{0x57})=\texttt{0x57}\oplus\texttt{0x83}=\texttt{0xD4} \]

So:

\[ C=\texttt{0xD4} \]

To decrypt:

\[ D_K(C)=C\oplus K \] \[ D_K(\texttt{0xD4})=\texttt{0xD4}\oplus\texttt{0x83}=\texttt{0x57} \]

Because XOR undoes itself with the same key, the encryption and decryption functions have the same shape in this simple example.

Common mistakes

Mistake 1: thinking \(f(x)\) means multiplication

The notation \(f(x)\) does not mean \(f\times x\). It means “apply function \(f\) to input \(x\).”

Mistake 2: confusing codomain and range

The codomain is where outputs are allowed to live. The range is where outputs actually land.

Mistake 3: assuming every function has an inverse

A function only has an inverse function when outputs preserve enough information to recover the original inputs. If two inputs collide into one output, the function cannot be undone uniquely.

Practice problems

  1. Let \(f(x)=3x+2\). Find \(f(4)\).
    Show solution
    \[ f(4)=3(4)+2=12+2=14 \]
  2. Let \(f(x)=x+5\). What is \(f^{-1}(x)\)?
    Show solution

    To undo adding 5, subtract 5.

    \[ f^{-1}(x)=x-5 \]
  3. Let \(f(x)=x+2\) and \(g(x)=x^2\). Find \((g\circ f)(3)\).
    Show solution

    Do \(f\) first:

    \[ f(3)=3+2=5 \] \[ g(f(3))=g(5)=5^2=25 \]

Why this matters for cryptography

Functions give us the language for describing cryptographic machines precisely.

  • A cipher can be viewed as a keyed function from plaintexts to ciphertexts.
  • Decryption is possible only when encryption can be undone for the intended key.
  • Permutation functions are central to reversible block cipher design.
  • Composition explains rounds: each round is a function, and the full cipher is a chain of functions.
  • S-boxes, permutations, key schedules, and mixing layers can all be studied as functions.

One-page recap

What to remember
  • A function assigns each allowed input exactly one output.
  • \(f(x)\) means “the output of function \(f\) on input \(x\).”
  • The domain is the set of allowed inputs.
  • The codomain is the declared set of possible outputs.
  • The range is the set of outputs actually produced.
  • A one-to-one function has no output collisions.
  • An onto function reaches every value in the codomain.
  • A permutation is a reversible rearrangement of a finite set.
  • Composition means chaining functions together.
  • An inverse function undoes another function.
