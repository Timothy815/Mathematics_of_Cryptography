Quick Reference — Function Concepts & Notation

Vocabulary

TermMeaning
FunctionRule: each input → exactly one output
DomainSet of allowed inputs
CodomainDeclared set of possible outputs
RangeOutputs actually produced
One-to-oneNo two inputs → same output
OntoEvery codomain value is reached
PermutationOne-to-one AND onto (reversible rearrangement)
InverseUndoes the original function

Core Notation

\(f : A \to B\) — function from domain \(A\) to codomain \(B\)
\(f(x)\) — output of \(f\) on input \(x\)
\((g \circ f)(x) = g(f(x))\) — do \(f\) first, then \(g\)
\(f^{-1}(f(x)) = x\) — inverse undoes \(f\)
\(C = E_K(P)\) — encryption keyed by \(K\)
\(P = D_K(C) = E_K^{-1}(C)\) — decryption

Key Properties

  • One input → exactly one output (always)
  • Two inputs → same output: allowed, but breaks invertibility
  • One input → two outputs: never a function
  • Composition order matters: \(g \circ f \neq f \circ g\) in general
  • Inverse exists only when function is one-to-one
  • XOR is self-inverse: \((P \oplus K) \oplus K = P\)

Crypto Connection

  • AES SubBytes = lookup function (S-box)
  • AES ShiftRows = byte permutation function
  • AES round = composition of 4 sub-functions
  • Full AES = composition of 10 round functions
  • Decryption = inverse function composition in reverse order
  • Block cipher key → selects one permutation from a family
