How to use this workbook

Treat each function like a machine: identify the input, apply the rule, and write the output. When checking whether a function is reversible, look for collisions: two different inputs producing the same output.

Part A: Warm-up vocabulary

A1. \(f(x)\) means the output produced when input \(x\) is put into function \(f\). It does not mean multiplication.

A2. The domain is the set of allowed inputs.

A3. The range is the set of outputs the function actually produces.

A4. Output collisions are a problem because if two different plaintexts produce the same ciphertext, decryption cannot know which original plaintext to return.

D5 sample answer. One-to-one is important because reversible encryption needs each ciphertext to point back to exactly one plaintext. If two plaintexts collide into one ciphertext, decryption becomes ambiguous.

Part E: Function composition

E1. \(f(5)=5+3=8\). Then \(g(8)=2(8)=16\). So \((g\circ f)(5)=16\).

E2. \(f(3)=3+2=5\). Then \(g(5)=5^2=25\). So \((g\circ f)(3)=25\).

E3. \(g(3)=3^2=9\). Then \(f(9)=9+2=11\). So \((f\circ g)(3)=11\).

E4 sample answer. Function order matters. Doing \(f\) first and then \(g\) can produce a different result from doing \(g\) first and then \(f\).

F5 sample answer. If two inputs produce the same output, the inverse would not know which input to return. One output would point backward to multiple possible inputs, so the undoing rule would not be a proper function.
