Multiplicative Inverses Modulo \(n\)

Learning when modular multiplication can be undone, why some numbers have inverses, and why this matters for cryptography.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

Modular addition is easy to undo: subtract the same amount. But modular multiplication is more selective. Sometimes multiplication can be undone, and sometimes it cannot. This tutorial explains how to tell the difference.

A multiplicative inverse modulo \(n\) is an undoing number. If multiplying by \(a\) scrambles a value around a modular circle, multiplying by the inverse of \(a\) brings it back. But not every number gets an undoing number. Some numbers leave footprints that collapse together.

What you should know first

You should understand modular arithmetic, especially the meaning of \(a\bmod n\), congruence, and standard residues \(0\) through \(n-1\). You do not need the Euclidean algorithm yet. That comes next.

Symbols used in this tutorial

Inverses use compact notation, so we define every symbol before the machinery starts moving.

\(n\) The wraparound size. A positive integer modulus.

The big idea

Intuition first

In ordinary arithmetic, the inverse of 5 is \(1/5\), because \(5\cdot(1/5)=1\). In modular arithmetic, we stay inside whole-number residues. So instead of asking for a fraction, we ask: is there a whole number \(x\) that makes \(a\cdot x\) land on 1 after wrapping around modulo \(n\)?

The essential test is simple to state:

\[ a\cdot a^{-1}\equiv1\pmod n \]

Technical definition

Definition: multiplicative inverse modulo \(n\)

A number \(x\) is a multiplicative inverse of \(a\) modulo \(n\) if multiplying \(a\) by \(x\) gives a result congruent to 1 modulo \(n\).

\[ ax\equiv1\pmod n \]

When this happens, we may write:

\[ x\equiv a^{-1}\pmod n \]

Since \(5\cdot21=105\), and \(105\bmod26=1\), we have:

\[ 5^{-1}\equiv21\pmod{26} \]
What this means

Multiplying by 5 modulo 26 can be undone by multiplying by 21 modulo 26. The number 21 is the undoing number for 5 in this modular world.

Using the inverse to undo multiplication

Suppose a value \(P\) was multiplied by 5 modulo 26:

\[ C\equiv5P\pmod{26} \]

Let \(P=7\). Then:

\[ C\equiv5\cdot7\pmod{26} \] \[ C\equiv35\pmod{26} \] \[ C\equiv9\pmod{26} \]

To get back \(P\), multiply by the inverse of 5 modulo 26. We just found that inverse is 21.

\[ P\equiv21C\pmod{26} \] \[ P\equiv21\cdot9\pmod{26} \] \[ P\equiv189\pmod{26} \] \[ P\equiv7\pmod{26} \]

We recovered the original value.

When an inverse does not exist

Not every number has an inverse modulo \(n\). Let us try to find the inverse of 2 modulo 6.

\[ 2x\equiv1\pmod6 \]

The result never lands on 1. So 2 has no multiplicative inverse modulo 6.

The hidden problem

Multiplication by 2 modulo 6 only reaches 0, 2, and 4. It never reaches 1, 3, or 5. It collapses the modular world into fewer landing spots. That collapse means information is lost.

The collision view

Multiplication by a number modulo \(n\) is a function from residues to residues. If the function has collisions, it cannot be inverted.

Multiplication by 2 modulo 6

0 → 0
1 → 2
2 → 4
3 → 0
4 → 2
5 → 4

Inputs 0 and 3 both land on 0. Inputs 1 and 4 both land on 2. Inputs 2 and 5 both land on 4. There are collisions everywhere, so the function cannot be undone uniquely.

Multiplication by 5 modulo 26

Multiplication by 5 modulo 26 does not collapse the residues. It rearranges them. That is why an inverse exists.

The greatest common divisor rule

There is a clean test for whether an inverse exists.

The inverse-existence rule

A number \(a\) has a multiplicative inverse modulo \(n\) exactly when \(a\) and \(n\) are coprime.

\[ a^{-1}\pmod n\text{ exists exactly when }\gcd(a,n)=1 \]

Friendly version: \(a\) gets an undoing number modulo \(n\) only when \(a\) and \(n\) do not share any factor bigger than 1.

Finding inverses by search

For small moduli, we can find an inverse by testing residues until the product lands on 1.

Example: find \(3^{-1}\pmod{10}\)

We need:

\[ 3x\equiv1\pmod{10} \]
3·1=3 3·2=6 3·3=9 3·4=12≡2 3·5=15≡5 3·6=18≡8 3·7=21≡1

Therefore:

\[ 3^{-1}\equiv7\pmod{10} \]

Cryptography preview: why the affine cipher cares

The affine cipher has the form:

\[ C\equiv aP+b\pmod{26} \]

Here \(P\) is the plaintext letter number, \(C\) is the ciphertext letter number, \(a\) is the multiplication part of the key, and \(b\) is the shift part of the key.

To decrypt, we must undo the addition by subtracting \(b\), and then undo multiplication by \(a\). That second step only works if \(a\) has an inverse modulo 26.

\[ P\equiv a^{-1}(C-b)\pmod{26} \]
Why this is not optional

If \(a\) has no inverse modulo 26, then different plaintext letters can collapse into the same ciphertext letter. Decryption would become guesswork. A cipher that cannot reliably decrypt is not a cipher. It is a paper shredder wearing a monocle.

Common mistakes

Mistake 1: thinking \(a^{-1}\) always means \(1/a\)

In modular arithmetic, \(a^{-1}\pmod n\) means a whole-number residue that multiplies with \(a\) to give 1 modulo \(n\). It is not usually a fraction.

Mistake 2: assuming every nonzero number has an inverse

In ordinary arithmetic, every nonzero number has a reciprocal. In modular arithmetic, a number has an inverse modulo \(n\) only if it is coprime to \(n\).

Mistake 3: forgetting the modulus

The inverse depends on the modulus. The inverse of 3 modulo 10 is 7, but the inverse of 3 modulo 26 is 9. The modulus changes the world.

Practice problems

  1. Check whether 7 is the inverse of 3 modulo 10.
    Show solution
    \[ 3\cdot7=21 \] \[ 21\bmod10=1 \]

    Yes. \(3^{-1}\equiv7\pmod{10}\).

  2. Find \(7^{-1}\pmod{12}\) by testing residues.
    Show solution

    Test \(7x\pmod{12}\). Since \(7\cdot7=49\), and \(49\bmod12=1\), the inverse is 7.

    \[ 7^{-1}\equiv7\pmod{12} \]
  3. Does 4 have an inverse modulo 10? Explain.
    Show solution

    No. \(\gcd(4,10)=2\), so 4 and 10 are not coprime. Therefore 4 has no inverse modulo 10.

  4. Find \(9^{-1}\pmod{26}\).
    Show solution

    We need \(9x\equiv1\pmod{26}\). Since \(9\cdot3=27\), and \(27\bmod26=1\), the inverse is 3.

    \[ 9^{-1}\equiv3\pmod{26} \]
  5. If \(C\equiv5P\pmod{26}\) and \(C=15\), recover \(P\).
    Show solution

    The inverse of 5 modulo 26 is 21.

    \[ P\equiv21C\pmod{26} \] \[ P\equiv21\cdot15\pmod{26} \] \[ P\equiv315\pmod{26} \] \[ P\equiv3\pmod{26} \]

Why this matters for cryptography

Multiplicative inverses are one of the first places where cryptographic mathematics becomes selective. Some operations are reversible, and some are not.

  • Affine cipher decryption requires a modular inverse.
  • RSA private key generation relies on modular inverses.
  • Diffie-Hellman and elliptic curve cryptography use inverse-like reasoning inside modular structures.
  • AES uses multiplicative inverses in \(\mathrm{GF}(2^8)\) when constructing its S-box.
  • Invertibility is central to deciding whether a transformation can be used in a reversible encryption path.

One-page recap

What to remember
  • A multiplicative inverse modulo \(n\) is an undoing number for multiplication modulo \(n\).
  • \(x\) is the inverse of \(a\) modulo \(n\) if \(ax\equiv1\pmod n\).
  • Not every nonzero number has a modular inverse.
  • An inverse exists exactly when \(\gcd(a,n)=1\).
  • Numbers with inverses modulo \(n\) are called units modulo \(n\).
  • If multiplication causes collisions, it cannot be undone uniquely.
  • The inverse depends on the modulus.
