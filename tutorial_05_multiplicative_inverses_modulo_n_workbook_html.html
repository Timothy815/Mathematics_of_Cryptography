How to use this workbook

A multiplicative inverse is an undoing number. For each problem, ask whether multiplying by a candidate lands on 1 modulo \(n\). When using the greatest common divisor rule, remember: an inverse exists exactly when \(\gcd(a,n)=1\).

Part A: Warm-up vocabulary

A1. A multiplicative inverse modulo \(n\) is an undoing number for multiplication in a wraparound number system.

A2. \(a^{-1}\pmod n\) means a residue \(x\) such that \(ax\equiv1\pmod n\).

A3. Two numbers are coprime if their greatest common divisor is 1.

A4. An inverse is important because decryption often needs to undo multiplication. Without an inverse, the original value may not be recoverable.

Part E: Non-invertible examples

E1. The row is: \(0,2,4,0,2,4\). It never equals 1, so 2 has no inverse modulo 6.

E2 sample answer. Multiplication by 2 modulo 6 causes collisions because different inputs land on the same output. For example, 0 and 3 both land on 0, 1 and 4 both land on 2, and 2 and 5 both land on 4.

E3 sample answer. \(\gcd(13,26)=13\), not 1, so 13 has no inverse modulo 26.

Part G: Affine cipher preview

G1.

\[ P\equiv21(C-8)\equiv21(13-8)\equiv21\cdot5=105\equiv1\pmod{26} \]

G2.

\[ P\equiv3(C-2)\equiv3(20-2)\equiv3\cdot18=54\equiv2\pmod{26} \]

G3 sample answer. It is bad because 13 has no inverse modulo 26. Since \(\gcd(13,26)=13\), multiplication by 13 cannot be undone uniquely.

Part H: Challenge problems

H1 sample answer. The inverse depends on the modulus because the wraparound size changes which products land on 1. \(3\cdot7=21\equiv1\pmod{10}\), but modulo 26, \(3\cdot9=27\equiv1\pmod{26}\).

H2 sample answer. Nonzero is not enough because the number also must be coprime to the modulus. For example, 2 is nonzero modulo 6, but \(\gcd(2,6)=2\), so it has no inverse.
