Bytes as Polynomials

The Alphabet of AES: learning how an 8-bit byte can be read as a polynomial with coefficients 0 and 1.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

We are preparing for finite-field arithmetic, especially the arithmetic used inside AES. Before we can multiply bytes in \(\mathrm{GF}(2^8)\), we need one quiet but powerful idea: a byte can be interpreted as a polynomial.

This tutorial does not teach Galois Field multiplication yet. It only teaches the alphabet. A byte such as 0x57 can be read as eight bits, as a number, or as a polynomial. Same bit pattern, different lens.

What you should know first

You should understand binary place values, hexadecimal bytes, bit positions, and basic polynomial notation such as \(x^2+x+1\). You do not need polynomial long division yet.

Symbols used in this tutorial

The symbol \(x\) in this tutorial is not a secret number to solve for. It is a formal placeholder used to label positions.

Symbol or notation Friendly meaning Technical meaning
byte A group of 8 bits. An 8-bit binary word.
bit A single 0 or 1. A binary digit.
coefficient A number that tells whether a term is included. A multiplier attached to a term in a polynomial.
\(x\) A position marker, not an unknown to solve for here. An indeterminate in a polynomial.
\(x^k\) The term connected to bit position \(k\). The \(k\)-th power of the indeterminate \(x\).
\(a_k\) The bit sitting in position \(k\). The coefficient of \(x^k\), where \(a_k\in\{0,1\}\).
\(a_7x^7+a_6x^6+\cdots+a_0\) The full polynomial form of an 8-bit byte. A polynomial of degree at most 7 with binary coefficients.
\(\mathrm{GF}(2)\) The tiny arithmetic world with only 0 and 1. The finite field with two elements, 0 and 1.
\(\mathrm{GF}(2^8)\) The 256-element byte-sized field we are preparing to understand. A finite field with \(2^8=256\) elements, often represented using byte-polynomials reduced by an irreducible polynomial.

The big idea

Intuition first

A byte is a row of eight switches. In ordinary binary, each switch controls a power of 2. In polynomial form, each switch controls a power of \(x\). A 1 means “include that term.” A 0 means “leave that term out.”

Ordinary binary place values

In ordinary binary, bit positions correspond to powers of 2.

Bit position 76543210
Power of 2 \(2^7\)\(2^6\)\(2^5\)\(2^4\)\(2^3\)\(2^2\)\(2^1\)\(2^0\)
Value 1286432168421

For example:

\[ 01010111_2=64+16+4+2+1=87 \]

That is the ordinary integer interpretation of the byte.

Polynomial place values

In polynomial form, bit positions correspond to powers of \(x\), not powers of 2.

Bit position 76543210
Polynomial term \(x^7\)\(x^6\)\(x^5\)\(x^4\)\(x^3\)\(x^2\)\(x\)\(1\)
Definition: byte-polynomial

A byte-polynomial is a polynomial whose coefficients are the bits of an 8-bit byte. Each bit is either 0 or 1, and bit position \(k\) becomes the coefficient of \(x^k\).

\[ a_7x^7+a_6x^6+a_5x^5+a_4x^4+a_3x^3+a_2x^2+a_1x+a_0 \]

Going the other direction: polynomial to byte

To convert a polynomial back to a byte, mark 1 for every power that appears and 0 for every power that does not appear.

Why the coefficients are only 0 and 1

The byte has only bits, and bits only have two values: 0 or 1. So the polynomial coefficients are also only 0 or 1.

Technical version

We are using polynomials with coefficients in \(\mathrm{GF}(2)\). That means every coefficient is either 0 or 1, and coefficient arithmetic will eventually happen modulo 2.

For now, do not worry about the arithmetic. The important idea is representation: bits become coefficients.

Same byte, three lenses

A byte does not change when we interpret it differently. We are changing the lens, not the underlying pattern.

Lens How 0x57 appears What that means
Hexadecimal 0x57 Compact base-16 notation for one byte.
Binary 01010111 The actual 8-bit pattern.
Integer 87 The ordinary base-2 place-value number.
Polynomial \(x^6+x^4+x^2+x+1\) The same bits read as coefficients of powers of \(x\).

Important: we are not plugging in a value for \(x\)

In algebra class, you may be used to seeing \(x\) as an unknown number. Here, \(x\) is not something we are solving for. It is a formal symbol that organizes positions.

Do not evaluate the polynomial

When we write \(x^6+x^4+x^2+x+1\), we are not asking you to plug in \(x=2\). If you plug in \(x=2\), you get the ordinary integer value 87, but finite-field arithmetic will treat \(x\) as a formal algebraic object, not as the number 2.

This is subtle, but important. The polynomial is a structured representation of the byte, not merely another way to evaluate the decimal number.

Preview: why XOR will become addition

In the next few tutorials, we will eventually add byte-polynomials. Since coefficients are bits, adding coefficients will happen modulo 2.

Coefficient sumModulo 2 resultXOR result
\(0+0\)00
\(0+1\)11
\(1+0\)11
\(1+1\)00

That table is exactly XOR. This is why, in \(\mathrm{GF}(2)\), addition and subtraction look like XOR. We will use that later, but today we are only preparing the map.

Why AES uses this viewpoint

AES works heavily with bytes. But for some of its mathematics, a byte is treated as a polynomial with binary coefficients. That lets AES define a special kind of byte multiplication in \(\mathrm{GF}(2^8)\).

Bridge idea

Integer modular arithmetic wrapped numbers around a clock. Finite-field byte arithmetic will wrap polynomials using a special polynomial rule. Before we learn the wrapping rule, we need to know how bytes become polynomials in the first place.

Common mistakes

Mistake 1: treating bit position 7 as \(2^7\) in polynomial form

In ordinary binary, position 7 means \(2^7\). In polynomial form, position 7 means \(x^7\). Same position, different lens.

Mistake 2: including zero terms

A 0 bit means the term is absent. For 01010111, we do not include \(x^7\), \(x^5\), or \(x^3\).

Mistake 3: thinking \(x\) is a value to solve for

Here, \(x\) is a formal placeholder for polynomial position. We are not solving for \(x\).

Mistake 4: jumping into multiplication too soon

This tutorial is about representation. Multiplication in \(\mathrm{GF}(2^8)\) comes later.

Why this matters for cryptography

Bytes-as-polynomials is the doorway from ordinary bit operations into finite-field arithmetic.

  • AES represents bytes as polynomials with binary coefficients.
  • Polynomial addition over \(\mathrm{GF}(2)\) corresponds to XOR.
  • Polynomial multiplication prepares us for AES byte multiplication.
  • Polynomial reduction will become the finite-field version of modular wraparound.
  • The AES S-box uses multiplicative inverses in \(\mathrm{GF}(2^8)\), so this representation is foundational.

What to remember
  • A byte is an 8-bit pattern.
  • The same byte can be read as hex, binary, an integer, or a polynomial.
  • In polynomial form, bit position \(k\) corresponds to \(x^k\).
  • A 1 bit means include the term.
  • A 0 bit means leave the term out.
  • Bit position 0 corresponds to the constant term \(1\).
  • We are not solving for \(x\). It is a formal placeholder.
  • The coefficients are in \(\{0,1\}\), also called \(\mathrm{GF}(2)\).
  • This representation prepares us for \(\mathrm{GF}(2^8)\) arithmetic.
