Polynomial Addition over \(\mathrm{GF}(2)\)

Why adding byte-polynomials is the same as XORing bytes, and why duplicate terms cancel instead of carrying.

🏠 Dashboard

Why this tutorial exists

Purpose

In Tutorial 8, we learned to read bytes as polynomials. Now we learn the first operation on those polynomials: addition over \(\mathrm{GF}(2)\). This is the point where XOR stops being just a bit trick and becomes polynomial addition in a two-symbol arithmetic world.

The strange-looking rule is this: when the same term appears twice, it cancels. In ordinary algebra, \(x+x=2x\). In \(\mathrm{GF}(2)\), \(x+x=0\). That cancellation is the tiny hinge on which AES-style byte arithmetic begins to turn.

What you should know first

You should know how to convert bytes into polynomial form and back. You should also remember XOR truth tables from bitwise operations. You do not need polynomial multiplication or polynomial reduction yet.

Learning objectives

After completing this module you will be able to
  • State and apply the GF(2) coefficient addition rule: \(1+1=0\), \(1+0=1\), \(0+0=0\).
  • Add two byte-polynomials over \(\mathrm{GF}(2)\) by combining like terms and cancelling duplicates.
  • Explain why byte XOR and polynomial addition over \(\mathrm{GF}(2)\) are the same operation.
  • Identify which terms cancel and which survive when two polynomials are added.
  • Explain why subtraction is identical to addition in \(\mathrm{GF}(2)\), and why \(P + P = 0\) for any polynomial \(P\).

The big idea

Intuition first

Polynomial addition over \(\mathrm{GF}(2)\) is term-by-term cancellation. If a term appears in one polynomial but not the other, it survives. If it appears in both, it cancels out. This is exactly what XOR does to bits.

Why duplicate terms cancel

Consider the term \(x^4\). If both polynomials contain \(x^4\), then its coefficient becomes:

\[ 1+1=0\quad\text{in }\mathrm{GF}(2) \]

So:

\[ x^4+x^4=0 \]
Duplicate terms cancel
\(x^4\)
1 + 1 = 0
\(x^2\)
1 + 0 = 1
\(x\)
0 + 1 = 1

This cancellation rule is not an exception. It is the rule.

Worked example 1: add two polynomials

Add:

\[ (x^6+x^4+x^2+x+1)+(x^7+x+1) \]

First, align like terms:

Power \(x^7\)\(x^6\)\(x^5\)\(x^4\)\(x^3\)\(x^2\)\(x\)\(1\)
First polynomial 01010111
Second polynomial 10000011
Sum mod 2 11010100

The \(x\) terms cancel, and the constant terms cancel. The result is:

\[ x^7+x^6+x^4+x^2 \]

Worked example 2: the same addition as byte XOR

The two polynomials from the previous example came from these bytes:

\[ \texttt{0x57}=01010111_2=x^6+x^4+x^2+x+1 \] \[ \texttt{0x83}=10000011_2=x^7+x+1 \]

XOR the bytes:

First byte01010111
Second byte10000011
XOR result11010100
\[ 01010111_2\oplus10000011_2=11010100_2 \] \[ \texttt{0x57}\oplus\texttt{0x83}=\texttt{0xD4} \]

Convert 0xD4 back to polynomial form:

\[ \texttt{0xD4}=11010100_2=x^7+x^6+x^4+x^2 \]
The bridge

Polynomial addition over \(\mathrm{GF}(2)\) and byte XOR are the same operation viewed through two different lenses.

Subtraction is the same as addition

In ordinary arithmetic, addition and subtraction are different. In \(\mathrm{GF}(2)\), they behave the same because every element is its own additive inverse.

\[ 1+1=0 \]

This means adding 1 undoes adding 1. So subtraction over \(\mathrm{GF}(2)\) uses the same bitwise operation: XOR.

Technical version

In \(\mathrm{GF}(2)\), \(-1=1\), so subtracting a coefficient is the same as adding it. Therefore polynomial subtraction and polynomial addition are identical over \(\mathrm{GF}(2)\).

Worked example 3: adding a polynomial to itself

Add a polynomial to itself:

\[ (x^6+x^4+x^2+x+1)+(x^6+x^4+x^2+x+1) \]

Every term appears twice, so every term cancels.

\[ x^6+x^6=0 \] \[ x^4+x^4=0 \] \[ x^2+x^2=0 \] \[ x+x=0 \] \[ 1+1=0 \]

Therefore:

\[ (x^6+x^4+x^2+x+1)+(x^6+x^4+x^2+x+1)=0 \]

In byte language:

\[ \texttt{0x57}\oplus\texttt{0x57}=\texttt{0x00} \]

Common mistakes

Mistake 1: using ordinary algebra and writing \(x+x=2x\)

In ordinary algebra, yes. But over \(\mathrm{GF}(2)\), coefficients are reduced modulo 2, so \(x+x=0\).

Mistake 2: carrying like ordinary binary addition

XOR has no carrying. Polynomial addition over \(\mathrm{GF}(2)\) has no carrying either. Each coefficient position is handled independently.

Mistake 3: mixing integer value with polynomial representation

The byte 0x57 has integer value 87, but polynomial addition does not mean adding 87 as an ordinary integer. It means XORing coefficients.

Mistake 4: jumping to multiplication rules

Addition over \(\mathrm{GF}(2)\) is straightforward XOR. Multiplication needs a separate lesson because it creates higher powers of \(x\).

Practice problems

  1. Add \((x^4+x+1)+(x^3+x+1)\) over \(\mathrm{GF}(2)\).
    Show solution

    The \(x\) terms cancel and the constant terms cancel.

    \[ (x^4+x+1)+(x^3+x+1)=x^4+x^3 \]
  2. Add \((x^7+x^2+1)+(x^7+x^5+x^2)\) over \(\mathrm{GF}(2)\).
    Show solution

    The \(x^7\) terms cancel, and the \(x^2\) terms cancel.

    \[ (x^7+x^2+1)+(x^7+x^5+x^2)=x^5+1 \]
  3. Compute 0x57 XOR 0x83 and give the polynomial result.
    Show solution
    \[ \texttt{0x57}\oplus\texttt{0x83}=\texttt{0xD4} \] \[ \texttt{0xD4}=11010100_2=x^7+x^6+x^4+x^2 \]
  4. What is \((x^6+x^4+x^2+x+1)+(x^6+x^4+x^2+x+1)\)?
    Show solution

    Every term cancels with itself, so the result is 0.

  5. Compute 0x1B XOR 0x03 and convert the result to polynomial form.
    Show solution
    \[ \texttt{0x1B}=00011011_2 \] \[ \texttt{0x03}=00000011_2 \] \[ 00011011_2\oplus00000011_2=00011000_2=\texttt{0x18} \] \[ \texttt{0x18}=x^4+x^3 \]

Why this matters for cryptography

Polynomial addition over \(\mathrm{GF}(2)\) explains why XOR appears so naturally in modern cryptography.

  • XOR is addition for bits.
  • Byte XOR is polynomial addition when bytes are read as polynomials over \(\mathrm{GF}(2)\).
  • No carrying means each bit position can be handled independently.
  • A value XORed with itself becomes zero, matching the cancellation rule.
  • This prepares us for polynomial multiplication, where terms combine and powers of \(x\) grow.

One-page recap

What to remember
  • Polynomial addition over \(\mathrm{GF}(2)\) means add coefficients modulo 2.
  • Coefficient values are only 0 and 1.
  • \(0+0=0\), \(0+1=1\), \(1+0=1\), and \(1+1=0\).
  • This is exactly the XOR truth table.
  • Like terms with matching powers of \(x\) cancel if they appear twice.
  • There is no carrying.
  • Subtraction is the same as addition over \(\mathrm{GF}(2)\).
  • A byte XORed with itself is zero.
