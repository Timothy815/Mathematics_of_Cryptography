How to use this workbook

Add coefficients one position at a time using modulo 2. Matching 1s cancel. Different bits produce 1. There is no carrying. When working with bytes, this is the same as XOR.

Note for self-study

When adding over \(\mathrm{GF}(2)\), look for matching terms. If a term appears twice, it disappears. If it appears once, it stays.

Part A: Warm-up vocabulary

A1. It means add the coefficients and reduce the result to either 0 or 1. In practice, \(1+1\) becomes 0.

A2. Because the coefficients add as \(1+1=2\), and \(2\equiv0\pmod2\). So the two matching \(x\) terms cancel.

A3. XOR.

A4. Each coefficient position is handled independently modulo 2. There is no place-value carrying into the next term.

Part G: Addition and subtraction

G1 sample answer. In \(\mathrm{GF}(2)\), every element is its own additive inverse. Since \(1+1=0\), adding 1 undoes adding 1. So subtraction and addition behave the same.

G2.

\[ (x^5+x^3+1)-(x^3+1)=(x^5+x^3+1)+(x^3+1)=x^5 \]

Part H: Concept checks

H1 sample answer. There is no carrying because coefficients are not place-value digits in ordinary binary addition. Each coefficient is added separately modulo 2.

H2 sample answer. The polynomial for 0x57 is \(x^6+x^4+x^2+x+1\). Adding it to itself makes every term appear twice, and every duplicate term cancels, leaving 0.

H3 sample answer. AES uses byte-polynomials over \(\mathrm{GF}(2)\). Understanding that polynomial addition is XOR prepares us for polynomial multiplication and reduction, which are needed for \(\mathrm{GF}(2^8)\) arithmetic.
