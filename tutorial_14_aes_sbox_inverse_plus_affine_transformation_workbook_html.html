How to use this workbook

Keep the two stages separate. First identify the finite-field inverse. Then apply or reason about the affine transformation. Watch bit order carefully: the affine formula uses bit 0 as the least significant bit.

Quick reference

AES S-box construction
  1. Take the multiplicative inverse in \(\mathrm{GF}(2^8)\).
  2. For 0x00, use 0x00 as the inverse-step value.
  3. Apply the affine transformation.
Affine formula
\[ s_i=b_i\oplus b_{(i+4)\bmod8}\oplus b_{(i+5)\bmod8}\oplus b_{(i+6)\bmod8}\oplus b_{(i+7)\bmod8}\oplus c_i \]

The constant byte is 0x63. Addition is XOR.

Note for self-study

The most important mental separation is this: the inverse step produces an intermediate byte, and the affine step produces the final S-box output.

Part A: Warm-up vocabulary

A1. An S-box is a substitution box: a function or table that maps each input value to a replacement output value.

A2. First, take the multiplicative inverse in \(\mathrm{GF}(2^8)\). Second, apply the affine transformation.

A3. An affine transformation is a structured bit-mixing step followed by XOR with a fixed constant.

A4. \(\oplus\) means XOR, or addition modulo 2.

Part C: Special zero case

C1 sample answer. 0x00 has no true multiplicative inverse because zero times any byte is still zero.

C2 sample answer. AES uses 0x00 as the inverse-step value for input 0x00. Since all inverse bits are 0, the affine formula outputs only the constant bits, giving 0x63.

Part G: Matrix-view concept

G1 sample answer. The inverse byte is treated as a bit vector. AES multiplies it by a fixed binary matrix, then XORs a fixed constant vector into the result.

G2 sample answer. It is not random because the same fixed formula is used every time. It is a structured linear transformation plus a constant over \(\mathrm{GF}(2)\).

Part H: Concept checks

H1 sample answer. The S-box can be stored as a lookup table, but its values come from a defined mathematical construction: finite-field inversion followed by an affine transformation.

H2 sample answer. The inverse operation creates a complex, nonlinear relationship between input bits and output bits. It is not a simple XOR, shift, or linear mixing operation.

H3 sample answer. The affine transformation reshapes the inverse result to avoid undesirable patterns and improve the cryptographic behavior of the S-box.
