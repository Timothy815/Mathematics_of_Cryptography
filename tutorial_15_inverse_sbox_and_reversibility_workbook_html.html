How to use this workbook

Focus on reversal. If the forward S-box says S[x] = y, then the inverse S-box must say InvS[y] = x. When reasoning about the internal procedure, remember that inverse operations happen in the opposite order.

Quick reference

Forward and reverse rule
\[ S(x)=y\Longleftrightarrow S^{-1}(y)=x \]

The inverse S-box is the forward S-box mapping read backward.

Internal reversal
\[ S(a)=A(a^{-1}) \] \[ S^{-1}(y)=\left(A^{-1}(y)\right)^{-1} \]

Undo affine first, then undo the field inverse step.

Note for self-study

Reversibility is not the same as weakness. A block cipher must be reversible for someone with the key, but its transformations should still hide patterns from someone without the key.

Part A: Warm-up vocabulary

A1. A function is reversible if you can recover the original input from the output.

A2. Bijective means every input has exactly one output and every output comes from exactly one input.

A3. A permutation of byte values is a rearrangement of all 256 byte values with no duplicates and no missing values.

A4. AES must be reversible because decryption needs to recover the original plaintext from ciphertext when the correct key is used.

Part H: Concept checks

H1 sample answer. The inverse S-box is determined by the forward S-box. Every forward mapping must be swapped to create the inverse mapping.

H2 sample answer. If two inputs share one output, decryption would see that output and could not uniquely determine which input created it.

H3 sample answer. The full S-box includes the affine transformation after the inverse step. Even if the field inverse step undoes itself, the complete composition is not the same as its inverse.

H4 sample answer. During AES decryption, the inverse S-box reverses the substitution layer so the cipher can reconstruct the previous round state.
