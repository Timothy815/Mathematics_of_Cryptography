Diffusion, Branch Number, and Avalanche

How ciphers spread small input changes into many output changes, and how designers measure the strength of that spreading.

🏠 Dashboard

Why this tutorial exists

Purpose

We have studied S-boxes, matrices, and linear transformations. Now we ask a design question: what should those pieces accomplish? One major answer is diffusion. A small change in the input should spread widely through the cipher state so patterns become hard to track.

Diffusion is the art of making a tiny crack in the ice spread across the pond. In cryptography, the crack is a changed bit. A good cipher makes that single change ripple outward until many bits are affected in ways that are difficult to predict without the key.

What you should know first

You should know XOR, bit vectors, matrix-based linear transformations, and the basic idea that block ciphers are built from repeated rounds. You do not need advanced probability yet. This tutorial uses small examples before naming the formal ideas.

Symbols used in this tutorial

This tutorial introduces measurement language. Each symbol below names something we can count or compare.

Symbol or notation Friendly meaning Technical meaning
diffusion Spreading one small change across many positions. A cipher property where plaintext or state statistics are spread across ciphertext/state bits.
avalanche A one-bit change causing many output bits to change. A sensitivity property where small input changes cause large output changes.
Hamming weight The number of 1s in a bit string. For a vector \(x\), often written \(wt(x)\).
Hamming distance The number of positions where two bit strings differ. For strings \(a\) and \(b\), the Hamming weight of \(a\oplus b\).

\(L(x)\) The output of linear layer \(L\) on input difference \(x\). A linear transformation applied to a nonzero input vector.

active component A chunk affected by a difference. A nonzero unit, often a bit, nibble, byte, or S-box input/output in a trail.

Intuition first

Confusion makes the relationship between key, plaintext, and ciphertext complicated. Diffusion spreads local changes across the state. Strong ciphers need both: nonlinear confusion to bend patterns, and diffusion to carry those bent patterns everywhere.

Definition: Hamming weight

The Hamming weight of a bit string is the number of 1 bits it contains. It is often written as \(wt(x)\).

Hamming distance: counting differences

Definition: Hamming distance

The Hamming distance between two bit strings is the number of positions where they differ. To compute it, XOR the two strings and count the 1s.

Example:

\[ a=10110010 \] \[ b=10110111 \] \[ a\oplus b=00000101 \]

The XOR result has two 1s, so the Hamming distance is 2.

Avalanche effect

Definition: avalanche effect

The avalanche effect describes the idea that a small change in the input, often just one bit, should cause many output bits to change.

In an ideal-looking block cipher output, flipping one input bit should eventually change about half of the output bits on average. Not exactly half every time, but roughly half across many trials.

Important distinction

Avalanche is usually observed across the whole cipher or across several rounds. A single small component may not avalanche by itself. Instead, the round structure gradually spreads and amplifies differences.

A small diffusion layer example

Consider this 4-bit linear layer:

\[ L\begin{bmatrix}v_0\\v_1\\v_2\\v_3\end{bmatrix} = \begin{bmatrix} v_0\oplus v_1\\ v_1\oplus v_2\\ v_2\oplus v_3\\ v_0\oplus v_3 \end{bmatrix} \]

If the input difference is only in \(v_0\):

\[ x=\begin{bmatrix}1\\0\\0\\0\end{bmatrix} \]

Then:

\[ L(x)=\begin{bmatrix}1\\0\\0\\1\end{bmatrix} \]

One active input bit became two active output bits. This is diffusion beginning to work.

Branch number

Definition: branch number

The branch number of a linear layer \(L\) is the smallest possible value of \(wt(x)+wt(L(x))\) over all nonzero input differences \(x\):

\[ B=\min_{x\ne0}\bigl(wt(x)+wt(L(x))\bigr) \]

Friendly meaning: branch number measures the worst-case spreading. It asks, “What is the least amount of activity this layer can produce if the input difference is not zero?”

Why worst-case matters

Attackers look for weak trails, not average trails. A strong diffusion layer should avoid letting a small difference remain small too easily.

Worked example: branch number for a tiny layer

Use the same layer:

\[ L(v_0,v_1,v_2,v_3)=(v_0\oplus v_1,\ v_1\oplus v_2,\ v_2\oplus v_3,\ v_0\oplus v_3) \]

Test several nonzero differences:

Active components

In cipher analysis, we often track whether a component is active rather than tracking every bit in detail. A component is active if it has a nonzero difference.

ComponentDifferenceActive?
Nibble 00000No

Active-component thinking is especially important when studying S-box layers. A difference entering an S-box makes that S-box active.

Wide-trail intuition

AES was designed so that differences do not stay trapped in a small part of the state. S-boxes provide nonlinear confusion, and linear layers spread activity across more S-boxes in later rounds.

Big-picture design idea

Good diffusion does not merely change many bits for decoration. It forces future nonlinear components to become active, making simple attack trails harder to maintain.

AES MixColumns preview

AES MixColumns is a linear diffusion layer. It operates on columns of four bytes. Its entries are not just 0 and 1; they are values in \(\mathrm{GF}(2^8)\), such as 0x02 and 0x03. But the design goal is similar: spread input byte differences across output bytes.

\[ \begin{bmatrix} 02&03&01&01\\ 01&02&03&01\\ 01&01&02&03\\ 03&01&01&02 \end{bmatrix} \]

We will study MixColumns more carefully later. For now, see it as a byte-level matrix designed for diffusion.

Diffusion versus confusion

IdeaPlain-language meaningTypical mechanism
ConfusionMake relationships complicated and nonlinear.S-boxes, inverses, nonlinear operations.
DiffusionSpread local changes across many positions.Permutations, matrices, MixColumns, linear layers.

A cipher with only diffusion may be too algebraically predictable. A cipher with only confusion may leave changes too localized. The craft is in combining them across rounds.

Common mistakes

Mistake 1: thinking one changed bit must instantly change half the state

Avalanche is usually a multi-round behavior. Individual components may only begin the spreading process.

Mistake 2: confusing Hamming weight and Hamming distance

Hamming weight counts 1s in one string. Hamming distance counts differences between two strings, often by XORing them first.

Mistake 3: judging diffusion only by average behavior

Worst-case behavior matters. Branch number focuses on the smallest possible amount of activity created by a nonzero input difference.

Mistake 4: treating linear layers as useless because they are linear

Linear layers are essential for diffusion. They need nonlinear layers beside them, not because they are useless, but because they perform a different job.

Practice problems

  1. Find the Hamming weight of 10110100.
    Show solution

    The string has four 1s, so its Hamming weight is 4.

  2. Find the Hamming distance between 10110010 and 00110111.
    Show solution
    \[ 10110010\oplus00110111=10000101 \]

    The XOR result has three 1s, so the Hamming distance is 3.

  3. What does avalanche mean in cipher design?
    Show solution

    A small input change, such as flipping one bit, should eventually cause many output bits to change.

  4. In branch number, why do we use \(x\ne0\)?
    Show solution

    If \(x=0\), then there is no input difference. The branch number measures spreading of nonzero differences.

  5. Why do ciphers combine S-boxes with diffusion layers?
    Show solution

    S-boxes provide nonlinear confusion. Diffusion layers spread changes so more S-boxes become active in later rounds.

Why this matters for cryptography

Diffusion is one of the central design goals of modern block ciphers. It prevents small changes from staying local and helps ensure that patterns cannot be followed easily through the cipher.

  • Hamming weight helps count active or changed bits.
  • Hamming distance helps compare two states.
  • Avalanche describes broad sensitivity to small input changes.
  • Branch number measures worst-case spreading in a linear layer.
  • AES uses S-boxes for confusion and linear layers like ShiftRows and MixColumns for diffusion.
