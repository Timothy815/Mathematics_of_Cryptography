How to use this workbook

Count carefully. Hamming weight counts 1s in one string. Hamming distance compares two strings by XORing them and counting the 1s. Branch number asks for the smallest total activity across all nonzero input differences.

Quick reference

Counting formulas
\[ wt(x)=\text{number of 1 bits in }x \] \[ d(a,b)=wt(a\oplus b) \] \[ B=\min_{x\ne0}\bigl(wt(x)+wt(L(x))\bigr) \]
Design ideas
  • Diffusion: spread local changes.
  • Avalanche: one small change causes many output changes.
  • Active component: a nonzero difference enters or leaves a component.
  • Branch number: worst-case spreading score.

Note for self-study

The key distinction is this: weight counts activity in one object, distance compares two objects, and branch number measures the worst-case spread through a layer.

Part A: Warm-up vocabulary

A1. Diffusion means spreading a small local change across many positions.

A2. Avalanche means a small input change, often one bit, eventually causes many output bits to change.

A3. Hamming weight is the number of 1s in a bit string.

A4. Hamming distance is the number of positions where two strings differ. It can be found by XORing the strings and counting the 1s.

Part I: AES connection

I1 sample answer. S-boxes provide nonlinear confusion. Linear diffusion layers spread those nonlinear effects across the state so more components are affected in later rounds.

I2 sample answer. MixColumns is a diffusion layer because it mixes bytes within a column so a change in one byte can affect multiple output bytes.

I3 sample answer. If a difference spreads to more byte positions, then in the next round more S-boxes receive nonzero differences. This makes attack trails harder to keep simple.
