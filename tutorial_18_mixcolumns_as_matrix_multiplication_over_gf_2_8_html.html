MixColumns as Matrix Multiplication over \(\mathrm{GF}(2^8)\)

How AES mixes each column of four bytes using finite-field multiplication, XOR, and a fixed matrix.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

We have learned finite-field multiplication, matrices, diffusion, and avalanche. MixColumns is where those ideas meet inside AES. It treats each column of the AES state as a four-byte vector and multiplies it by a fixed matrix over \(\mathrm{GF}(2^8)\).

MixColumns is a byte-level mixing machine. Four bytes enter. Four new bytes leave. Each output byte is built by multiplying the input bytes by small constants, then XORing the results together.

What you should know first

You should understand multiplication in \(\mathrm{GF}(2^8)\), especially multiplication by 0x02 and 0x03, matrix-vector multiplication, XOR, and the idea that diffusion spreads changes across a state.

The big picture

Intuition first

Tutorial 16 used matrices made of bits. MixColumns uses a matrix made of bytes. The idea is the same: each row is a recipe for one output. The difference is that the row entries are finite-field multipliers like 0x02 and 0x03, not just include-or-ignore bits.

The AES state as a 4 by 4 byte grid

AES works on 16 bytes at a time. These bytes are arranged as a 4 by 4 state. MixColumns acts on each vertical column separately.

For one column, we can write the four bytes as:

\[ \begin{bmatrix}s_0\\s_1\\s_2\\s_3\end{bmatrix} \]

The MixColumns matrix

AES MixColumns multiplies each column by this fixed matrix:

\[ \begin{bmatrix} \texttt{02}&\texttt{03}&\texttt{01}&\texttt{01}\\ \texttt{01}&\texttt{02}&\texttt{03}&\texttt{01}\\ \texttt{01}&\texttt{01}&\texttt{02}&\texttt{03}\\ \texttt{03}&\texttt{01}&\texttt{01}&\texttt{02} \end{bmatrix} \]

The output column is:

\[ \begin{bmatrix}s'_0\\s'_1\\s'_2\\s'_3\end{bmatrix} = \begin{bmatrix} \texttt{02}&\texttt{03}&\texttt{01}&\texttt{01}\\ \texttt{01}&\texttt{02}&\texttt{03}&\texttt{01}\\ \texttt{01}&\texttt{01}&\texttt{02}&\texttt{03}\\ \texttt{03}&\texttt{01}&\texttt{01}&\texttt{02} \end{bmatrix} \begin{bmatrix}s_0\\s_1\\s_2\\s_3\end{bmatrix} \]

Each row is an output-byte recipe

Reading the matrix row by row gives four formulas:

\[ s'_0=(\texttt{02}\cdot s_0)\oplus(\texttt{03}\cdot s_1)\oplus s_2\oplus s_3 \] \[ s'_1=s_0\oplus(\texttt{02}\cdot s_1)\oplus(\texttt{03}\cdot s_2)\oplus s_3 \] \[ s'_2=s_0\oplus s_1\oplus(\texttt{02}\cdot s_2)\oplus(\texttt{03}\cdot s_3) \] \[ s'_3=(\texttt{03}\cdot s_0)\oplus s_1\oplus s_2\oplus(\texttt{02}\cdot s_3) \]
How this differs from bit matrices

In a bit matrix, 1 means “include this bit.” In MixColumns, 0x01 means “include this byte unchanged,” 0x02 means “multiply this byte by 2 in \(\mathrm{GF}(2^8)\),” and 0x03 means “multiply this byte by 3 in \(\mathrm{GF}(2^8)\).”

Why MixColumns creates diffusion

Each output byte depends on all four input bytes in the column. This means a change in one input byte can affect every output byte of that column.

Diffusion role

MixColumns spreads byte-level differences within each column. When combined with ShiftRows, those changes are moved across columns in later rounds, causing activity to spread through the AES state.

Branch number connection

AES MixColumns has a strong byte-level diffusion property: for nonzero input differences, the number of active input bytes plus active output bytes is at least 5. In branch-number language, MixColumns has branch number 5 at the byte level.

\[ B=5 \]

Friendly meaning: if a column enters MixColumns with a nonzero difference, the activity cannot stay too tiny across input and output combined.

Why this matters

Branch number helps explain why AES forces many S-boxes to become active across rounds. More active S-boxes make simple differential and linear trails harder to exploit.

Inverse MixColumns preview

MixColumns must be reversible for AES decryption. AES uses a different matrix to undo MixColumns:

\[ \begin{bmatrix} \texttt{0E}&\texttt{0B}&\texttt{0D}&\texttt{09}\\ \texttt{09}&\texttt{0E}&\texttt{0B}&\texttt{0D}\\ \texttt{0D}&\texttt{09}&\texttt{0E}&\texttt{0B}\\ \texttt{0B}&\texttt{0D}&\texttt{09}&\texttt{0E} \end{bmatrix} \]

We will study inverse MixColumns later. For now, the key idea is that MixColumns is linear and invertible.

Common mistakes

Mistake 1: using ordinary integer multiplication

02 · DB in MixColumns is multiplication in \(\mathrm{GF}(2^8)\), not ordinary decimal or hexadecimal integer multiplication.

Mistake 2: using addition instead of XOR

The row products are combined with XOR, not ordinary addition with carries.

Mistake 3: mixing rows and columns

MixColumns acts on vertical columns of the AES state. Each row of the MixColumns matrix computes one output byte for that column.

Mistake 4: thinking MixColumns is nonlinear

MixColumns is linear over \(\mathrm{GF}(2^8)\). AES gets nonlinearity from the S-box, not from MixColumns.

Practice problems

  1. What does 0x01 do as a multiplier in MixColumns?
    Show solution

    It leaves the byte unchanged.

  2. How do you compute 0x03 · a using xtime?
    Show solution

    0x03 · a = xtime(a) XOR a.

  3. In the first MixColumns row, which input bytes affect \(s'_0\)?
    Show solution

    All four input bytes affect \(s'_0\): \(s_0\), \(s_1\), \(s_2\), and \(s_3\).

  4. Compute 02 · 13 in \(\mathrm{GF}(2^8)\).
    Show solution

    0x13 has top bit 0, so left shift gives 0x26. Thus 02 · 13 = 26.

  5. Why does MixColumns help diffusion?
    Show solution

    Each output byte depends on all four input bytes in a column, so a change in one byte can spread across the whole column.

Why this matters for cryptography

MixColumns is one of AES's main diffusion layers. It does not provide nonlinearity, but it spreads the nonlinear effects of the S-box across the state over repeated rounds.

  • It treats columns as four-byte vectors.
  • It uses matrix multiplication over \(\mathrm{GF}(2^8)\).
  • It combines finite-field multiplication with XOR.
  • It has strong byte-level diffusion with branch number 5.
  • It is invertible, so AES decryption can undo it.

One-page recap

What to remember
  • MixColumns acts on one vertical AES state column at a time.
  • Each column has four bytes.
  • MixColumns is matrix multiplication over \(\mathrm{GF}(2^8)\).
  • The matrix uses constants 01, 02, and 03.
  • 01 · a = a.
  • 02 · a = xtime(a).
  • 03 · a = xtime(a) XOR a.
  • Each output byte is made by XORing four finite-field products.
  • MixColumns is linear but strongly diffusive.
  • MixColumns is invertible and has byte-level branch number 5.
