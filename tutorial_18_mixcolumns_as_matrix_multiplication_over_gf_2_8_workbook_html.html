How to use this workbook

Work row by row. Each MixColumns output byte is built from four finite-field products XORed together. Use 01 · a = a, 02 · a = xtime(a), and 03 · a = xtime(a) XOR a.

Quick reference

MixColumns matrix
\[ \begin{bmatrix} \texttt{02}&\texttt{03}&\texttt{01}&\texttt{01}\\ \texttt{01}&\texttt{02}&\texttt{03}&\texttt{01}\\ \texttt{01}&\texttt{01}&\texttt{02}&\texttt{03}\\ \texttt{03}&\texttt{01}&\texttt{01}&\texttt{02} \end{bmatrix} \]
Multiplier rules
\[ \texttt{01}\cdot a=a \] \[ \texttt{02}\cdot a=\operatorname{xtime}(a) \] \[ \texttt{03}\cdot a=\operatorname{xtime}(a)\oplus a \]

Note for self-study

The whole operation is matrix multiplication, but the arithmetic inside the matrix is AES byte-field arithmetic. That is the gearshift: byte products first, XOR second.

Part A: Warm-up vocabulary

A1. MixColumns acts on one vertical column of four AES state bytes at a time.

A2. It uses arithmetic over \(\mathrm{GF}(2^8)\), the AES byte field.

A3. Each output byte depends on all four input bytes, so changes can spread through the column.

A4. MixColumns is linear over \(\mathrm{GF}(2^8)\). AES gets nonlinearity from the S-box.

Part F: Concept checks

F1 sample answer. Each row of the MixColumns matrix has four nonzero constants, so every output byte includes a product from all four input bytes.

F2 sample answer. MixColumns uses finite-field multiplication in \(\mathrm{GF}(2^8)\), not ordinary integer multiplication.

F3 sample answer. XOR is addition in the AES byte field, so row products are combined with XOR.

F4 sample answer. Byte-level branch number 5 means any nonzero input difference has input activity plus output activity of at least 5 across the column transformation.

Part G: Inverse MixColumns preview

G1 sample answer. AES must decrypt, so MixColumns must be undoable during decryption.

G2. No. The inverse MixColumns constants are 0E, 0B, 0D, and 09, not 01, 02, and 03.
