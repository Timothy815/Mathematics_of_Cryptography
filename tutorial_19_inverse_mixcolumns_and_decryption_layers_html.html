Inverse MixColumns and Decryption Layers

How AES reverses MixColumns using a different matrix over \(\mathrm{GF}(2^8)\), and why decryption walks round layers backward.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

MixColumns spreads information across each AES column. But AES must also decrypt, so that spreading must be reversible. This tutorial explains the inverse MixColumns matrix and how it restores a mixed column back to its original form.

Forward MixColumns is the blender. Inverse MixColumns is not “unblending” by magic. It is another carefully chosen matrix that reverses the first matrix inside the same finite-field arithmetic world.

What you should know first

You should understand MixColumns as matrix multiplication over \(\mathrm{GF}(2^8)\), finite-field multiplication, XOR as field addition, and the idea that reversible cipher layers must have inverse operations.

Symbols used in this tutorial

Forward MixColumns matrix review

The forward MixColumns matrix is:

\[ M= \begin{bmatrix} \texttt{02}&\texttt{03}&\texttt{01}&\texttt{01}\\ \texttt{01}&\texttt{02}&\texttt{03}&\texttt{01}\\ \texttt{01}&\texttt{01}&\texttt{02}&\texttt{03}\\ \texttt{03}&\texttt{01}&\texttt{01}&\texttt{02} \end{bmatrix} \]

It computes:

\[ c'=Mc \]

where \(c\) is the original column and \(c'\) is the mixed column.

The inverse MixColumns matrix

The inverse MixColumns matrix is:

\[ M^{-1}= \begin{bmatrix} \texttt{0E}&\texttt{0B}&\texttt{0D}&\texttt{09}\\ \texttt{09}&\texttt{0E}&\texttt{0B}&\texttt{0D}\\ \texttt{0D}&\texttt{09}&\texttt{0E}&\texttt{0B}\\ \texttt{0B}&\texttt{0D}&\texttt{09}&\texttt{0E} \end{bmatrix} \]

It computes:

\[ c=M^{-1}c' \]
Why the constants are different

The inverse matrix is not found by merely reversing the order of 01, 02, and 03. It is the true matrix inverse over \(\mathrm{GF}(2^8)\), so it uses constants 09, 0B, 0D, and 0E.

The identity check

The reason \(M^{-1}\) works is that multiplying the inverse matrix by the forward matrix gives the identity matrix:

\[ M^{-1}M=I \]

Friendly meaning: doing MixColumns and then inverse MixColumns returns the original column.

\[ M^{-1}(Mc)=c \]
Important order note

For this AES matrix, \(MM^{-1}=I\) also holds. Both directions work because these matrices are true inverses.

Inverse MixColumns row recipes

Let the mixed column be:

\[ \begin{bmatrix}s'_0\\s'_1\\s'_2\\s'_3\end{bmatrix} \]

The recovered original column is:

\[ s_0=(\texttt{0E}\cdot s'_0)\oplus(\texttt{0B}\cdot s'_1)\oplus(\texttt{0D}\cdot s'_2)\oplus(\texttt{09}\cdot s'_3) \] \[ s_1=(\texttt{09}\cdot s'_0)\oplus(\texttt{0E}\cdot s'_1)\oplus(\texttt{0B}\cdot s'_2)\oplus(\texttt{0D}\cdot s'_3) \] \[ s_2=(\texttt{0D}\cdot s'_0)\oplus(\texttt{09}\cdot s'_1)\oplus(\texttt{0E}\cdot s'_2)\oplus(\texttt{0B}\cdot s'_3) \] \[ s_3=(\texttt{0B}\cdot s'_0)\oplus(\texttt{0D}\cdot s'_1)\oplus(\texttt{09}\cdot s'_2)\oplus(\texttt{0E}\cdot s'_3) \]

How the inverse constants are computed

The inverse constants can be built from repeated xtime values. For any byte \(a\):

\[ \texttt{09}\cdot a=(\texttt{08}\cdot a)\oplus a \] \[ \texttt{0B}\cdot a=(\texttt{08}\cdot a)\oplus(\texttt{02}\cdot a)\oplus a \] \[ \texttt{0D}\cdot a=(\texttt{08}\cdot a)\oplus(\texttt{04}\cdot a)\oplus a \] \[ \texttt{0E}\cdot a=(\texttt{08}\cdot a)\oplus(\texttt{04}\cdot a)\oplus(\texttt{02}\cdot a) \]

Here 04 · a means apply xtime twice, and 08 · a means apply xtime three times.

Decryption layers walk backward

AES encryption rounds apply several operations in a particular order. Decryption must undo those operations in reverse order.

Encryption ideaDecryption undo
SubBytesInvSubBytes
ShiftRowsInvShiftRows
MixColumnsInvMixColumns
AddRoundKeyAddRoundKey again, because XOR undoes itself
Reverse-order rule

If encryption does A, then B, then C, decryption must undo C first, then B, then A. Cipher layers are not undone in the same order they were applied.

Why use an invertible linear layer?

MixColumns is linear, but it is carefully chosen. It spreads byte differences strongly while still being reversible. That balance is important: AES needs diffusion for security and invertibility for decryption.

Design balance

A block cipher should scramble information without destroying it. MixColumns spreads information across a column, but inverse MixColumns can recover the original column when the decryption process is followed correctly.

Common mistakes

Mistake 1: trying to undo MixColumns with the forward matrix

Decryption uses the inverse matrix, not the original MixColumns matrix again.

Mistake 2: thinking inverse constants are decimal reciprocals

The constants 09, 0B, 0D, and 0E are finite-field matrix inverse constants, not ordinary fractions.

Mistake 3: using ordinary multiplication

Inverse MixColumns still uses multiplication in \(\mathrm{GF}(2^8)\), not ordinary integer multiplication.

Mistake 4: undoing decryption layers in the wrong order

Decryption reverses the order of encryption operations. The last forward operation must be the first one undone.

Practice problems

  1. What matrix does AES use to undo MixColumns?
    Show solution

    AES uses the inverse MixColumns matrix with constants 0E, 0B, 0D, and 09.

  2. If forward MixColumns maps DB 13 53 45 to 8E 4D A1 BC, what should inverse MixColumns do to 8E 4D A1 BC?
    Show solution

    It should recover DB 13 53 45.

  3. Why are the inverse MixColumns constants different from the forward constants?
    Show solution

    They come from the true matrix inverse over \(\mathrm{GF}(2^8)\), not from merely reversing the forward constants.

  4. What does \(M^{-1}M=I\) mean in friendly language?
    Show solution

    Doing MixColumns and then inverse MixColumns returns the column to its original value.

  5. Why does AES need inverse layers during decryption?
    Show solution

    Decryption must reverse the transformations applied during encryption to recover the original plaintext.

Why this matters for cryptography

Inverse MixColumns shows a recurring block-cipher design principle: a layer can be strongly diffusive and still reversible. That matters because encryption should hide structure, but decryption must recover the original data with the correct key.

  • MixColumns spreads information through a column.
  • Inverse MixColumns restores the original column.
  • Both use matrix multiplication over \(\mathrm{GF}(2^8)\).
  • The inverse matrix uses different constants from the forward matrix.
  • Decryption applies inverse layers in reverse order.
