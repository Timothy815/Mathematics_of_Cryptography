Directions

How to use this workbook

Keep the direction clear. Forward MixColumns sends an original column to a mixed column. Inverse MixColumns sends the mixed column back to the original. All multiplication still happens in \(\mathrm{GF}(2^8)\), and row products are still combined with XOR.

Quick reference

Forward and inverse relationship
\[ c'=Mc \] \[ c=M^{-1}c' \] \[ M^{-1}M=I \]
Inverse constants

The inverse MixColumns matrix uses:

\[ \texttt{09},\ \texttt{0B},\ \texttt{0D},\ \texttt{0E} \]

These are finite-field matrix inverse constants, not ordinary decimal reciprocals.

Note for self-study

The essential idea is not that inverse MixColumns is mysterious. It is just matrix inversion, but the matrix arithmetic lives inside AES's byte field.

Part A: Warm-up vocabulary

A1. Inverse MixColumns undoes the MixColumns transformation and restores the mixed column to its previous column value.

A2. AES must decrypt, so every encryption layer that changes the state must be undoable.

A3. Doing MixColumns and then inverse MixColumns gives the do-nothing effect. The original column is recovered.

A4. No. The constants are finite-field matrix inverse constants, not ordinary reciprocals or fractions.

Part H: Concept checks

H1 sample answer. Forward MixColumns and inverse MixColumns are different matrices. The inverse matrix is specifically chosen so that applying it after the forward matrix returns the identity effect.

H2 sample answer. A block cipher needs to spread information for security, but it must not destroy information because decryption must recover the plaintext.

H3 sample answer. The inverse constants are the result of matrix inversion over \(\mathrm{GF}(2^8)\). They are not chosen for smallness or visual simplicity.
