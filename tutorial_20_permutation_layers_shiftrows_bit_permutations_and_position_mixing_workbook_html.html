How to use this workbook

Track positions carefully. A permutation moves values without changing them. Nothing should disappear, and nothing should appear twice. For ShiftRows, move each row left by its row number, then use InvShiftRows to move it back.

Quick reference

Permutation rules
  • A permutation rearranges positions.
  • No position is lost.
  • No position is duplicated.
  • Every true permutation has an inverse.
AES ShiftRows
  • Row 0 shifts left by 0.
  • Row 1 shifts left by 1.
  • Row 2 shifts left by 2.
  • Row 3 shifts left by 3.
  • InvShiftRows shifts right by the same amounts.

Note for self-study

A permutation is not powerful because it changes data. It is powerful because it changes where data goes next.

Part G: Permutation matrices

G1. Each row has exactly one 1.

G2. Each column has exactly one 1.

G3. \([a,b,c,d]^T\) maps to \([c,a,d,b]^T\).

G4 sample answer. One 1 per row means every output receives one input. One 1 per column means every input is used once. This prevents loss and duplication.

Part I: Diffusion connection

I1 sample answer. ShiftRows moves bytes into different columns, so the next MixColumns step combines bytes that came from different previous columns.

I2 sample answer. No. ShiftRows moves active bytes but does not increase their number by itself. The next mixing layer can increase activity.

I3 sample answer. Substitution adds nonlinearity, while permutation or mixing layers spread those effects to new positions so later substitutions affect more of the state.
