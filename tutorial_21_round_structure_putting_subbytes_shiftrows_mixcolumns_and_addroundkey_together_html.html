Round Structure: Putting SubBytes, ShiftRows, MixColumns, and AddRoundKey Together

How the four major AES round layers work together as one repeated, reversible encryption machine.

🏠 Dashboard

Why this tutorial exists

Purpose

We have studied the major AES round pieces separately: S-box substitution, ShiftRows, MixColumns, inverse layers, and finite-field arithmetic. This tutorial assembles those pieces into the AES round structure so the cipher begins to look like a complete machine instead of a tray of parts.

An AES round is a carefully ordered sequence of transformations. Each layer has a distinct job: substitute values, move positions, mix columns, and combine with a round key. Repeated together, these layers create confusion and diffusion across the whole state.

What you should know first

You should understand bytes, XOR, the AES S-box, ShiftRows, MixColumns, inverse MixColumns, and the idea of diffusion. You do not need to know the full key schedule yet. For now, treat each round key as a 16-byte value supplied to the round.

Symbols used in this tutorial

A round structure has several moving parts. The table below defines the terms before we use them.

Symbol or notation Friendly meaning Technical meaning

The AES state

AES transforms a 16-byte block. It arranges those bytes into a 4 by 4 state. Each round changes that state.

Every round layer acts on this state in a different way.

The four layer jobs

SubBytes

Replace each byte through the AES S-box. This provides nonlinearity.

ShiftRows

Move bytes sideways. This changes which bytes share columns.

MixColumns

Mix each column using finite-field matrix multiplication. This spreads influence.

XOR the state with a round key. This injects secret key material.

Layer 1: SubBytes

SubBytes applies the AES S-box to each byte independently. If a byte is \(b\), the output is \(S(b)\).

\[ b\longmapsto S(b) \]

This layer is nonlinear. That matters because linear operations alone are too algebraically predictable. SubBytes bends the relationship between input and output.

Purpose of SubBytes

SubBytes supplies the round's main source of confusion. It makes byte changes pass through a nonlinear substitution before later layers move and mix those effects.

Layer 2: ShiftRows

ShiftRows moves bytes sideways across the state rows. It does not change byte values. It changes their positions.

  • Row 0 shifts left by 0.
  • Row 1 shifts left by 1.
  • Row 2 shifts left by 2.
  • Row 3 shifts left by 3.
Purpose of ShiftRows

ShiftRows moves bytes into different columns so the next MixColumns step combines bytes that came from different previous columns.

Layer 3: MixColumns

MixColumns treats each column as a four-byte vector and multiplies it by the AES MixColumns matrix over \(\mathrm{GF}(2^8)\).

\[ \begin{bmatrix}s'_0\\s'_1\\s'_2\\s'_3\end{bmatrix} = \begin{bmatrix} \texttt{02}&\texttt{03}&\texttt{01}&\texttt{01}\\ \texttt{01}&\texttt{02}&\texttt{03}&\texttt{01}\\ \texttt{01}&\texttt{01}&\texttt{02}&\texttt{03}\\ \texttt{03}&\texttt{01}&\texttt{01}&\texttt{02} \end{bmatrix} \begin{bmatrix}s_0\\s_1\\s_2\\s_3\end{bmatrix} \]
Purpose of MixColumns

MixColumns spreads changes within each column. Combined with ShiftRows, it helps changes travel through the whole state across rounds.

Layer 4: AddRoundKey

AddRoundKey XORs the state with the round key. If \(S\) is the state and \(K_r\) is the round key for round \(r\), then:

\[ S\longmapsto S\oplus K_r \]

This is the layer that directly combines the changing state with key material.

Purpose of AddRoundKey

AddRoundKey makes the transformation depend on the secret key. Without key mixing, the round structure would be public and identical for everyone.

Why XOR is convenient here

XOR undoes itself: \(x\oplus k\oplus k=x\). That means AddRoundKey is easy to reverse when the same round key is known.

Normal round versus final round

A normal AES round includes all four layers:

\[ \text{SubBytes}\rightarrow\text{ShiftRows}\rightarrow\text{MixColumns}\rightarrow\text{AddRoundKey} \]

The final AES round omits MixColumns:

\[ \text{SubBytes}\rightarrow\text{ShiftRows}\rightarrow\text{AddRoundKey} \]
For now, remember the structure

The final round is slightly different: no MixColumns. We will study the full AES encryption flow in the next tutorial, including the initial AddRoundKey and the number of rounds.

Round typeLayers includedMixColumns?
Normal roundSubBytes, ShiftRows, MixColumns, AddRoundKeyYes
Final roundSubBytes, ShiftRows, AddRoundKeyNo

Why the order matters

The order is not decorative. Each layer prepares the state for the next layer.

StepWhat happensWhy it prepares the next step
1. SubBytesEach byte is replaced nonlinearly.The state now contains nonlinear substitutions.
2. ShiftRowsBytes move sideways.Bytes are placed into new columns.
3. MixColumnsEach column is mixed.Moved bytes are now combined with new neighbors.
4. AddRoundKeyRound key is XORed in.The state becomes key-dependent for the next round.

If you changed the order, you would get a different cipher structure. It might still be a cipher-like construction, but it would not be AES.

Rounds build cumulative effect

One round does not carry the whole burden. AES repeats round transformations. Each pass adds more substitution, movement, mixing, and key interaction.

The repeating pattern

SubBytes bends. ShiftRows relocates. MixColumns spreads. AddRoundKey binds the result to the key. Repeating that pattern turns local structure into wide, key-dependent structure.

Decryption preview: reverse the layers

Because each layer is reversible, AES decryption can walk backward. But it must undo layers in reverse order.

Forward layerInverse layerReason it can be reversed
AddRoundKeyAddRoundKeyXOR undoes itself.
MixColumnsInvMixColumnsMixColumns matrix is invertible.
ShiftRowsInvShiftRowsIt is a permutation.
SubBytesInvSubBytesThe S-box is bijective.
Reverse-order rule

A normal encryption round applies SubBytes, ShiftRows, MixColumns, then AddRoundKey. To undo that round, decryption must undo AddRoundKey first, then MixColumns, then ShiftRows, then SubBytes.

Toy round analogy

Imagine a tiny toy cipher round on four symbols:

  1. Substitute each symbol using a small S-box.
  2. Permute their positions.
  3. Mix neighboring symbols.
  4. XOR with a small round key.

The AES round is the grown-up version: byte substitutions, row shifts, column mixing over \(\mathrm{GF}(2^8)\), and round-key XOR on a 16-byte state.

Common mistakes

Mistake 1: treating the layers as interchangeable

The layer order matters. SubBytes, ShiftRows, MixColumns, and AddRoundKey each prepare the state differently.

Mistake 2: forgetting the final round omits MixColumns

Normal AES rounds include MixColumns. The final round does not.

Mistake 3: thinking MixColumns provides nonlinearity

MixColumns is linear. SubBytes provides the main nonlinearity.

Mistake 4: thinking AddRoundKey is optional decoration

AddRoundKey is what ties the public round operations to secret key material.

Why this matters for cryptography

The AES round is a compact example of modern block-cipher design. It combines nonlinear substitution, position movement, linear diffusion, and key mixing in a repeated structure.

  • SubBytes provides nonlinear confusion.
  • ShiftRows moves bytes into different columns.
  • MixColumns spreads influence through each column.
  • AddRoundKey makes each round depend on key material.
  • Repeating rounds builds strong cumulative confusion and diffusion.

One-page recap

What to remember
  • A normal AES round has four layers.
  • The order is SubBytes, ShiftRows, MixColumns, AddRoundKey.
  • SubBytes replaces each byte using the AES S-box.
  • ShiftRows moves bytes sideways across rows.
  • MixColumns mixes bytes within each column.
  • AddRoundKey XORs the state with round-key material.
  • The final AES round omits MixColumns.
  • Each layer is reversible.
  • Decryption uses inverse layers in reverse order.
  • The full cipher effect comes from repeating the round structure.
