How to use this workbook

Focus on the jobs and the order of the AES round layers. The round is not just a list of parts. Each layer prepares the state for the next layer, and decryption must undo those layers in reverse order.

Quick reference

Normal AES round
\[ \text{SubBytes}\rightarrow\text{ShiftRows}\rightarrow\text{MixColumns}\rightarrow\text{AddRoundKey} \]
Final AES round
\[ \text{SubBytes}\rightarrow\text{ShiftRows}\rightarrow\text{AddRoundKey} \]

The final round omits MixColumns.

Note for self-study

The round is a choreography: nonlinear substitution, position movement, column diffusion, and key mixing. The order is part of the design.

Part A: Warm-up vocabulary

A1. The AES state is the 16-byte working grid that AES transforms during encryption and decryption.

A2. An AES round is one cycle of transformations applied to the state.

A3. A round key is a 16-byte subkey derived from the original cipher key and used in a specific round.

A4 sample answer. Repeating rounds builds cumulative confusion and diffusion. One round starts the scrambling, but repeated rounds spread and complicate patterns more thoroughly.

Part C: Match each layer to its job

LayerMain jobChanges values?Main cryptographic idea
SubBytesReplace each byte using the S-box.YesConfusion / nonlinearity
ShiftRowsMove bytes sideways in rows.NoPosition movement
MixColumnsMix bytes within each column.YesDiffusion
AddRoundKeyXOR state with round key.YesKey mixing

ShiftRowsInvShiftRowsIt is a permutation.

Part F: Why order matters

StepWhat happens?Why it prepares the next step?
SubBytesBytes are replaced nonlinearly.The state now contains nonlinear substitutions.
ShiftRowsBytes move sideways.Bytes enter different columns.
MixColumnsColumns are mixed.New neighbors are combined.
AddRoundKeyRound key is XORed in.The state becomes key-dependent.

F1 sample answer. Each layer changes the state in a way that affects the input to the next layer. Changing the order changes which values are substituted, moved, mixed, and keyed.

Part G: Decryption reverse-order preview

G1. AddRoundKey must be undone first because it was applied last.

G2. 1. AddRoundKey, 2. InvMixColumns, 3. InvShiftRows, 4. InvSubBytes.

G3 sample answer. XOR undoes itself: \(x\oplus k\oplus k=x\). Applying the same round key again removes it.

Part H: Concept checks

H1 sample answer. SubBytes provides the main nonlinearity. This matters because purely linear operations are too algebraically predictable.

H2 sample answer. ShiftRows moves bytes into different columns, and MixColumns then mixes the bytes in those new columns. Together they spread influence across the state.

H3 sample answer. AddRoundKey is what makes the public round transformations depend on secret key material.

H4 sample answer. An AES round substitutes bytes, moves them, mixes columns, and XORs in round-key material to build confusion and diffusion.
