AES Encryption Flow: Initial Round, Main Rounds, and Final Round

How AES-128 turns one plaintext block into ciphertext using an initial key addition, nine full main rounds, and one final round without MixColumns.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

Tutorial 21 explained the structure of one AES round. This tutorial zooms out and shows the full AES-128 encryption flow: how the plaintext state enters, how round keys are used, how the nine main rounds repeat, and how the final round differs.

AES-128 is not one giant mysterious spell. It is a staged machine: key-whiten the input, run nine full rounds, then run a final round that leaves out MixColumns. The rhythm matters.

What you should know first

You should know the four AES round layers: SubBytes, ShiftRows, MixColumns, and AddRoundKey. You should also know that the final round omits MixColumns. You do not need to know how the round keys are generated yet. That comes in the key expansion tutorials.

Scope note

We are focusing on AES-128 because it is the cleanest starting point: 128-bit block, 128-bit key, 10 rounds, and 11 round keys.

Step 1: Initial AddRoundKey

AES begins by XORing the plaintext state with the first round key, \(K_0\). This happens before any SubBytes, ShiftRows, or MixColumns step.

\[ \text{State}_0=\text{Plaintext}\oplus K_0 \]
Why start with AddRoundKey?

The initial AddRoundKey makes the state key-dependent before the repeated round transformations begin. This first key mixing is sometimes called key whitening.

Step 2: Nine main rounds

AES-128 then performs nine full main rounds. Each main round uses all four layers:

\[ \text{SubBytes}\rightarrow\text{ShiftRows}\rightarrow\text{MixColumns}\rightarrow\text{AddRoundKey} \]

Main round \(r\) uses round key \(K_r\), where \(r\) runs from 1 through 9.

RoundLayersRound key
1SubBytes, ShiftRows, MixColumns, AddRoundKey\(K_1\)
2SubBytes, ShiftRows, MixColumns, AddRoundKey\(K_2\)
⋮Same full-round structure⋮
9SubBytes, ShiftRows, MixColumns, AddRoundKey\(K_9\)

Step 3: Final round

Round 10 is the final round. It uses SubBytes, ShiftRows, and AddRoundKey, but it omits MixColumns.

\[ \text{SubBytes}\rightarrow\text{ShiftRows}\rightarrow\text{AddRoundKey} \]

The final round uses \(K_{10}\). Its output is the ciphertext block.

\[ \text{Ciphertext}=\text{FinalRound}(\text{State},K_{10}) \]
The final-round difference

The easiest beginner rule is: main rounds have MixColumns; the final round does not.

AES-128 timeline

Here is the whole AES-128 encryption flow as a timeline.

Input
Load the 16-byte plaintext block into the AES state.
Initial
Apply AddRoundKey using \(K_0\).
Rounds 1-9
Repeat SubBytes, ShiftRows, MixColumns, AddRoundKey using \(K_1\) through \(K_9\).
Round 10
Apply SubBytes, ShiftRows, AddRoundKey using \(K_{10}\). No MixColumns.
Output
Read the resulting state as the ciphertext block.

AES-128 encryption pseudocode

In plain pseudocode, AES-128 encryption looks like this:

state = plaintext
state = AddRoundKey(state, K0)

for round = 1 to 9:
    state = SubBytes(state)
    state = ShiftRows(state)
    state = MixColumns(state)
    state = AddRoundKey(state, K_round)

state = SubBytes(state)
state = ShiftRows(state)
state = AddRoundKey(state, K10)

ciphertext = state

This is not implementation-ready code. It is a readable map of the flow.

Why no MixColumns in the final round?

For a beginner, the safest first understanding is structural: AES is defined so the final round omits MixColumns. This keeps encryption and decryption layer structure aligned in a useful way and avoids ending the cipher with an extra linear mixing layer after the last key addition.

Practical memory hook

AES starts with key mixing, spends nine rounds substituting, shifting, mixing, and keying, then ends with substitution, shifting, and keying. The final key addition is the last operation.

Round keys preview

AES-128 begins with one 128-bit key, but the encryption flow needs 11 round keys. The key schedule expands the original key into \(K_0\) through \(K_{10}\).

Round keyUsed where?Purpose
\(K_0\)Initial AddRoundKeyKey the state before main rounds.
\(K_1\) through \(K_9\)Main rounds 1 through 9Inject key material each full round.

Confusion and diffusion across the full flow

The full AES flow repeatedly combines the same design roles:

LayerRoleHow repetition helps
SubBytesNonlinear confusionRepeated nonlinear substitution makes relationships harder to model simply.
ShiftRowsPosition movementRepeated movement changes which bytes interact.
MixColumnsLinear diffusionRepeated mixing spreads local changes through more of the state.
AddRoundKeyKey dependenceRepeated key injection binds the state to key material throughout encryption.

Common mistakes

Mistake 1: calling the initial AddRoundKey “round 1”

AES starts with an initial AddRoundKey before the numbered main rounds. Round 1 comes after that.

Mistake 2: putting MixColumns in the final round

The final AES round omits MixColumns. This is one of the most important flow details.

Mistake 3: using the same round key every time

AES uses different round keys generated by the key schedule: \(K_0\) through \(K_{10}\) for AES-128.

Mistake 4: forgetting the output is the final state

After the final AddRoundKey, the state is read as the ciphertext block.

Why this matters for cryptography

AES-128 encryption is a disciplined repeated structure. The initial key addition, nine full rounds, and final reduced round work together to build a reversible but highly scrambled transformation from plaintext to ciphertext.

  • The plaintext is first made key-dependent with \(K_0\).
  • Nine main rounds perform substitution, movement, mixing, and key addition.
  • The final round omits MixColumns.
  • Each round key is different.
  • The ciphertext is the state after the final AddRoundKey.
