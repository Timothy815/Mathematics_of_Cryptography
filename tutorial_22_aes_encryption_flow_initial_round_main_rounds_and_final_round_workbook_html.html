How to use this workbook

Think of AES-128 as a flow map. Start with plaintext, apply the initial AddRoundKey using \(K_0\), run nine full main rounds using \(K_1\) through \(K_9\), then run the final round using \(K_{10}\). The final round does not include MixColumns.

Quick reference

AES-128 flow
  • 128-bit block
  • 128-bit key
  • 10 rounds
  • 11 round keys: \(K_0\) through \(K_{10}\)
Round structures
\[ \text{Main: SB}\rightarrow\text{SR}\rightarrow\text{MC}\rightarrow\text{ARK} \] \[ \text{Final: SB}\rightarrow\text{SR}\rightarrow\text{ARK} \]

SB = SubBytes, SR = ShiftRows, MC = MixColumns, ARK = AddRoundKey.

Part G: Concept checks

G1 sample answer. AES begins with AddRoundKey so the state becomes key-dependent before the main round transformations begin.

G2 sample answer. Different round keys prevent every round from using the exact same key material and make each round's key interaction distinct.

G3 sample answer. AES-128 loads the plaintext into the state, XORs it with \(K_0\), performs nine full rounds using SubBytes, ShiftRows, MixColumns, and AddRoundKey with \(K_1\) through \(K_9\), then performs the final round with SubBytes, ShiftRows, and AddRoundKey using \(K_{10}\). The final state is the ciphertext.

G4. The ciphertext is the final state after the last AddRoundKey.
