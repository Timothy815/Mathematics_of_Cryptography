How to use this workbook

Work backward. Decryption starts from ciphertext, uses \(K_{10}\) first, walks through inverse main rounds using \(K_9\) down to \(K_1\), and finishes by applying AddRoundKey with \(K_0\). The last encryption step is the first decryption step.

Quick reference

Key order

Encryption:

\[ K_0,K_1,K_2,\ldots,K_9,K_{10} \]

Decryption:

\[ K_{10},K_9,K_8,\ldots,K_1,K_0 \]
Inverse layer pairs
  • SubBytes → InvSubBytes
  • ShiftRows → InvShiftRows
  • MixColumns → InvMixColumns
  • AddRoundKey → AddRoundKey again

Part H: Concept checks

H1 sample answer. Decryption uses round keys backward because it must undo encryption in reverse order. The last key applied during encryption, \(K_{10}\), is the first key removed during decryption.

H2 sample answer. Decryption is not simply encryption again because it uses inverse layers and reverse key order. Each encryption transformation must be undone, not repeated forward.

H3 sample answer. AES-128 decryption loads the ciphertext into the state, applies AddRoundKey with \(K_{10}\), then InvShiftRows and InvSubBytes to undo the final encryption round. It then undoes rounds 9 through 1 using AddRoundKey, InvMixColumns, InvShiftRows, and InvSubBytes with \(K_9\) down to \(K_1\). Finally, it applies AddRoundKey with \(K_0\), and the final state is the plaintext.

H4. The plaintext is the final state after the last AddRoundKey with \(K_0\).
