Key Expansion I: Round Keys, Words, RotWord, and SubWord

The first look inside the AES key schedule: how a 128-bit key is split into words and transformed into material for many round keys.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

AES encryption and decryption use many round keys, not just the original cipher key repeated over and over. This tutorial begins the key schedule by explaining words, round keys, RotWord, and SubWord. The next tutorial will add Rcon and walk through full AES-128 key expansion.

The AES key schedule is a key-making workshop. It starts with one 128-bit key and forges a chain of related round keys. Each round key is not random, but it is produced by a structured process that rotates, substitutes, XORs, and injects round constants.

What you should know first

You should know bytes, hexadecimal notation, XOR, AES round keys, the AES S-box, and the encryption flow from Tutorials 21 through 23. You do not need to compute a full key schedule yet. This lesson introduces the parts.

The big picture

Intuition first

AES-128 starts with 16 key bytes. The key schedule splits them into four 4-byte words. Then it keeps generating new words from old words. Every four words form one round key.

AES-128 key expansion at a glance
Original key
16 bytes
→
Split into words
\(w_0,w_1,w_2,w_3\)
→
Generate more words
\(w_4,w_5,\ldots,w_{43}\)
→
11 round keys
\(K_0\) to \(K_{10}\)

Why expand the key?

AES-128 needs 11 round keys: \(K_0\) through \(K_{10}\). Each round key is 16 bytes. That means AES-128 needs 176 bytes of round-key material:

\[ 11\times16=176\text{ bytes} \]

But the original AES-128 key is only 16 bytes. The key schedule expands those 16 bytes into 176 bytes.

Important idea

The key schedule does not create new independent secrets from nowhere. It deterministically derives round-key material from the original secret key.

Words: the 4-byte building blocks

Definition: word

In AES key expansion, a word is a group of four bytes. Since each byte is 8 bits, one word is 32 bits.

Since one round key is 16 bytes, one round key is four words:

\[ 4\text{ words}\times4\text{ bytes per word}=16\text{ bytes} \]

Splitting the original AES-128 key into words

Suppose the original AES-128 key is:

Split it into four words:

These four words form \(K_0\), the initial round key.

The word-generation idea

After the first four words are copied from the original key, AES generates each later word using earlier words. The simplest recurring idea is:

\[ w_i=w_{i-4}\oplus \text{something based on }w_{i-1} \]

Most of the time, the “something” is just \(w_{i-1}\). But at the start of each new round key, AES transforms \(w_{i-1}\) using RotWord, SubWord, and Rcon before XORing.

The two cases

If \(i\) is not a multiple of 4, AES-128 uses \(w_i=w_{i-4}\oplus w_{i-1}\). If \(i\) is a multiple of 4, AES first transforms \(w_{i-1}\) using the key-schedule core.

RotWord

Definition: RotWord

RotWord rotates a four-byte word left by one byte. The first byte moves to the end.

SubWord

Definition: SubWord

SubWord applies the AES S-box to each byte of a word. It substitutes each byte independently.

If the input word is:

CF
4F
3C
09

Then SubWord gives:

8A
84
EB
01
\[ \operatorname{SubWord}([CF,4F,3C,09])=[8A,84,EB,01] \]
Why use the S-box here?

The key schedule uses SubWord to introduce nonlinearity into the round-key generation process. Without nonlinear steps, the expanded key would have simpler algebraic relationships.

The key-schedule core

At the start of each new AES-128 round key, the previous word goes through a special transformation:

Key-schedule core
Previous word
\(w_{i-1}\)
→
RotWord
→
SubWord
→
XOR with Rcon
→
Transformed word

In formula form, when \(i\) is a multiple of 4:

\[ w_i=w_{i-4}\oplus\operatorname{SubWord}(\operatorname{RotWord}(w_{i-1}))\oplus Rcon \]

We will compute this fully in the next tutorial. For now, notice the structure: rotate, substitute, add round constant, XOR with a word from four positions earlier.

Simple word generation by XOR

Once the first word of a new round key is generated, the next three words are made by simple XOR chaining:

\[ w_5=w_1\oplus w_4 \] \[ w_6=w_2\oplus w_5 \] \[ w_7=w_3\oplus w_6 \]
Pattern to remember

The first word of each new round key gets special treatment. The next three words are chained forward by XOR.

Common mistakes

Mistake 1: thinking a word is one byte

In the AES key schedule, a word is four bytes, or 32 bits.

Mistake 2: confusing RotWord with bit rotation

RotWord rotates by bytes, not by individual bits. The first byte moves to the end.

Mistake 3: applying SubWord to only one byte

SubWord applies the AES S-box to all four bytes of the word.

Mistake 4: thinking all words are generated the same way

In AES-128, words whose index is a multiple of 4 use the special key-schedule core. Other words use simpler XOR chaining.

Why this matters for cryptography

The key schedule is not the same thing as encryption rounds, but it feeds them. AES needs fresh round-key material at each stage. RotWord, SubWord, XOR chaining, and Rcon help turn one key into a structured series of round keys.

  • AES-128 begins with a 16-byte key.
  • The key is split into four 4-byte words.
  • Every four words form one round key.
  • AES-128 needs 44 words total.
  • RotWord moves bytes within a word.
  • SubWord applies the AES S-box to each byte.
