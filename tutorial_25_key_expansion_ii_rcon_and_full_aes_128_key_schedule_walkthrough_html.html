Key Expansion II: Rcon and Full AES-128 Key Schedule Walkthrough

Completing the first AES-128 expanded round key by adding Rcon, computing \(w_4\) through \(w_7\), and assembling \(K_1\).

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

Tutorial 24 introduced the AES key-schedule parts: words, round keys, RotWord, and SubWord. This tutorial adds Rcon and performs a complete step-by-step computation of the first expanded round key, \(K_1\), from a standard AES-128 example key.

We are now turning the key schedule crank. We will start with \(w_0\) through \(w_3\), transform \(w_3\), inject Rcon, compute \(w_4\), then use XOR chaining to compute \(w_5\), \(w_6\), and \(w_7\). Those four new words form \(K_1\).

What you should know first

You should know that an AES key-schedule word is four bytes, that AES-128 round keys are four words, and that RotWord rotates a word left by one byte while SubWord applies the AES S-box to all four bytes. You should also know XOR notation \(\oplus\).

The big picture

Intuition first

The first word of a new round key receives special treatment. It is not merely copied or XORed. AES takes the previous word, rotates it, substitutes its bytes, XORs a round constant into it, and only then combines it with the word four places earlier.

What is Rcon?

Definition: Rcon

Rcon is a round constant used in AES key expansion. It is XORed into the transformed word when AES begins generating a new round key.

In AES-128 key expansion, each Rcon value is a four-byte word. Only the first byte is nonzero:

\[ Rcon_j=[RC_j,00,00,00] \]
Round constant4-byte word
\(Rcon_1\)01 00 00 00
\(Rcon_2\)02 00 00 00
\(Rcon_3\)04 00 00 00
\(Rcon_4\)08 00 00 00
\(Rcon_5\)10 00 00 00
Where Rcon comes from

The nonzero byte of Rcon follows powers of \(x\) in \(\mathrm{GF}(2^8)\). For this tutorial, you only need to use the listed constants. The important operational fact is that Rcon is XORed into the first byte of the transformed word.

Starting key and initial words

We will use the same standard AES-128 example key:

2B 7E 15 16  28 AE D2 A6  AB F7 15 88  09 CF 4F 3C
WordBytes
\(w_0\)2B 7E 15 16
\(w_1\)28 AE D2 A6
\(w_2\)AB F7 15 88
\(w_3\)09 CF 4F 3C

These four words form \(K_0\). Our goal is to compute \(w_4,w_5,w_6,w_7\), which form \(K_1\).

Step 1: Compute \(g(w_3)\)

The key-schedule core \(g(w)\) means:

\[ g(w)=\operatorname{SubWord}(\operatorname{RotWord}(w))\oplus Rcon \]

Start with \(w_3\):

StepResult
Start with \(w_3\)09 CF 4F 3C
RotWordCF 4F 3C 09
SubWord8A 84 EB 01
XOR \(Rcon_1\)8B 84 EB 01

The Rcon step is:

\[ [8A,84,EB,01]\oplus[01,00,00,00]=[8B,84,EB,01] \]

So:

\[ g(w_3)=\texttt{8B 84 EB 01} \]

Step 2: Compute \(w_4\)

The first word of \(K_1\) is:

\[ w_4=w_0\oplus g(w_3) \]
ValueBytes
\(w_0\)2B 7E 15 16
\(g(w_3)\)8B 84 EB 01
\(w_4\)A0 FA FE 17

Byte by byte:

\[ \texttt{2B}\oplus\texttt{8B}=\texttt{A0},\quad \texttt{7E}\oplus\texttt{84}=\texttt{FA},\quad \texttt{15}\oplus\texttt{EB}=\texttt{FE},\quad \texttt{16}\oplus\texttt{01}=\texttt{17} \]

Step 3: Compute \(w_5\), \(w_6\), and \(w_7\)

After \(w_4\), the next three words are generated by XOR chaining:

\[ w_5=w_1\oplus w_4 \] \[ w_6=w_2\oplus w_5 \] \[ w_7=w_3\oplus w_6 \]

The pattern repeats

To compute \(K_2\), AES starts with \(w_7\), applies the key-schedule core using \(Rcon_2\), computes \(w_8\), and then chains forward to \(w_9,w_{10},w_{11}\).

AES-128 key expansion pseudocode

In readable pseudocode, the AES-128 key expansion pattern is:

copy original key into w0, w1, w2, w3

for i = 4 to 43:
    temp = w[i - 1]

    if i is a multiple of 4:
        temp = SubWord(RotWord(temp)) XOR Rcon

    w[i] = w[i - 4] XOR temp

This produces 44 words total, which are grouped into 11 round keys.

Common mistakes

Mistake 1: applying Rcon to all four bytes

Rcon is a four-byte word, but only its first byte is nonzero. For \(Rcon_1\), the value is 01 00 00 00.

Mistake 2: forgetting the RotWord step

The key-schedule core starts by rotating the previous word left by one byte before applying SubWord.

Mistake 3: applying the key-schedule core to every word

In AES-128, the key-schedule core is applied only when the new word index is a multiple of 4.

Mistake 4: grouping words incorrectly into round keys

Every AES-128 round key is four consecutive words: \(K_1=w_4,w_5,w_6,w_7\), \(K_2=w_8,w_9,w_{10},w_{11}\), and so on.

Practice problems

  2. If SubWord gives 8A 84 EB 01, what happens after XOR with \(Rcon_1\)?
    Show solution

  5. When does AES-128 apply the key-schedule core?
    Show solution

    When the new word index is a multiple of 4.

Why this matters for cryptography

The AES key schedule controls how key material enters each round. It is not simply repeating the original key. It expands the key using rotation, nonlinear substitution, round constants, and XOR chaining so each round receives its own derived round key.

  • Rcon breaks simple symmetry between rounds.
  • SubWord brings S-box nonlinearity into key expansion.
  • RotWord moves byte positions inside the word.
  • XOR chaining connects new words to previous key material.
  • Every four expanded words become a round key.

What to remember
  • AES-128 key expansion starts with \(w_0,w_1,w_2,w_3\).
  • Those first four words form \(K_0\).
  • To compute \(w_4\), transform \(w_3\) using RotWord, SubWord, and Rcon.
  • For the example key, \(g(w_3)=\texttt{8B 84 EB 01}\).
  • \(w_4=w_0\oplus g(w_3)=\texttt{A0 FA FE 17}\).
  • \(w_5=w_1\oplus w_4=\texttt{88 54 2C B1}\).
  • \(w_6=w_2\oplus w_5=\texttt{23 A3 39 39}\).
  • \(w_7=w_3\oplus w_6=\texttt{2A 6C 76 05}\).
  • \(K_1=w_4,w_5,w_6,w_7\).
  • The pattern repeats until \(w_{43}\).
