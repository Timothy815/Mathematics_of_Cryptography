How to use this workbook

Work slowly and byte by byte. This workbook completes the first AES-128 expanded round key. You will transform \(w_3\), apply \(Rcon_1\), compute \(w_4\), chain forward to \(w_7\), and assemble \(K_1\).

Quick reference

Key-schedule core
\[ g(w)=\operatorname{SubWord}(\operatorname{RotWord}(w))\oplus Rcon \] \[ w_i=w_{i-4}\oplus g(w_{i-1}) \]

Used in AES-128 when \(i\) is a multiple of 4.

Example starting words
  • \(w_0=\)2B 7E 15 16
  • \(w_1=\)28 AE D2 A6
  • \(w_2=\)AB F7 15 88
  • \(w_3=\)09 CF 4F 3C

Part I: Concept checks

I1 sample answer. Rcon changes from round to round and is injected into the transformed word, helping prevent repeated structural patterns from staying too symmetric across round-key generation.

I2 sample answer. To compute \(w_4\), take \(w_3\), rotate it left by one byte, substitute all four bytes with the AES S-box, XOR in \(Rcon_1\), then XOR the result with \(w_0\).

I3 sample answer. \(K_1\) is produced through transformation and XOR chaining. It depends on the original key, but it is not simply the same 16 bytes copied again.
