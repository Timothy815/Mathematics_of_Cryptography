How to use this workbook

ECB is conceptually simple, but the security lesson is important. Track which plaintext blocks are equal. Under ECB with the same key, equal plaintext blocks produce equal ciphertext blocks. The danger is not usually direct plaintext recovery. The danger is pattern leakage.

Quick reference

ECB formula
\[ C_i=E_K(P_i) \]

Each plaintext block is encrypted independently with the same key.

Core ECB leak
\[ P_i=P_j\Longrightarrow C_i=C_j \]

Repeated plaintext blocks create repeated ciphertext blocks.

Note for self-study

ECB is the lesson where “secure part” and “secure system” separate. AES may be strong, but ECB can still leak structure because it treats blocks independently.

Part E: Pattern-leak reasoning

E1 sample answer. No. ECB does not necessarily reveal the plaintext bytes directly. Its main weakness is that it reveals equality patterns between blocks.

E2 sample answer. ECB reveals that two or more plaintext blocks were identical because their ciphertext blocks are identical.

E3 sample answer. Structured data often contains repeated regions, repeated headers, blank space, or regular layouts. ECB can preserve those repetitions as repeated ciphertext blocks, leaving a visible or analyzable pattern.

Part H: Concept checks

H1 sample answer. AES is a strong block cipher for encrypting one block, but ECB is an unsafe way to apply it to many blocks because it reveals repeated-block patterns.

H2 sample answer. “Codebook” suggests that the same plaintext block maps to the same ciphertext block under the same key, like looking up the same entry in a codebook.

H3 sample answer. ECB encrypts each block independently with the same key. This means identical plaintext blocks become identical ciphertext blocks. Even if the attacker cannot read the plaintext directly, they can see repetition and structure, which may reveal useful information about the message.
