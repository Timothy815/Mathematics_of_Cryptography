How to use this workbook

CBC is about dependency. For the first block, the plaintext is XORed with the IV. For every later block, the plaintext is XORed with the previous ciphertext block. Track the chaining value carefully.

Quick reference

CBC encryption
\[ C_1=E_K(P_1\oplus IV) \] \[ C_i=E_K(P_i\oplus C_{i-1})\quad\text{for }i>1 \]
CBC decryption
\[ P_1=D_K(C_1)\oplus IV \] \[ P_i=D_K(C_i)\oplus C_{i-1}\quad\text{for }i>1 \]

Part H: Concept checks

H1 sample answer. CBC reduces ECB-style leakage because the block cipher input is not just the plaintext block. It is the plaintext block XORed with a chaining value, so repeated plaintext blocks do not automatically create repeated ciphertext blocks.

H2 sample answer. CBC needs the IV during decryption because the first plaintext block is recovered by decrypting \(C_1\) and XORing the result with the IV.

H3 sample answer. No. CBC encryption hides content patterns better than ECB, but it does not prove the ciphertext has not been modified. Authentication requires another secure mechanism or an authenticated encryption mode.

H4 sample answer. CBC encrypts each block by first XORing it with a chaining value. The first block uses an IV, and later blocks use the previous ciphertext block. This makes encryption depend on earlier output and reduces ECB-style repeated-block leakage, but CBC still needs proper IV handling and authentication.
