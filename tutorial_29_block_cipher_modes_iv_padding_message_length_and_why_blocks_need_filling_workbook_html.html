How to use this workbook

Padding is mostly careful counting. For AES, the block size is 16 bytes. Find the message length remainder, determine how many bytes are missing, and fill the final block with padding bytes that identify the padding length.

Quick reference

AES block size
  • AES block size: 128 bits
  • AES block size: 16 bytes
  • PKCS#7 padding adds \(n\) bytes
  • Each padding byte has value \(n\)
Padding length formula
\[ r=L\bmod B \] \[ n=\begin{cases} B-r, & \text{if }r\ne0\\ B, & \text{if }r=0 \end{cases} \]

Note for self-study

Padding is counting plus unambiguous removal. The padding must tell the receiver exactly how much filler to remove after decryption.

Part D: Full-block padding

D1. 16 padding bytes are added.

D2. Each padding byte is 10, because 16 decimal is 10 hexadecimal.

D3 sample answer. A full padding block is added so the receiver can always tell that padding is present and know exactly how many bytes to remove. Without it, the final data byte might be confused with a padding marker.

Part G: Padding oracle reasoning

G1 sample answer. A padding oracle reveals whether decrypted padding was valid or invalid.

G2 sample answer. Even a yes/no signal can be dangerous because attackers can submit modified ciphertexts and use the system's responses to learn information about hidden plaintext.

G3 sample answer. Padding checks are easy to get subtly wrong through timing differences, error messages, or different behavior. Vetted libraries are designed to avoid known implementation mistakes.

Part H: Concept checks

H1 sample answer. Zero padding can be ambiguous because the original message might naturally end in zero bytes. The receiver may not know which zero bytes are data and which are padding.

H2 sample answer. No. Padding only makes a message fit the block size. It does not hide the message by itself and does not prove that the ciphertext was not modified.

H3 sample answer. PKCS#7-style padding adds enough bytes to make the message length a multiple of the block size. If \(n\) bytes are needed, it adds \(n\) copies of the byte value \(n\). If a message already fits exactly, it adds a full block of padding so removal is unambiguous.
