Directions

How to use this workbook

Keep AES-GCM separated into two cooperating jobs. CTR-style encryption provides confidentiality. GHASH helps authenticate the ciphertext and associated data. The tag is the final check that decides whether the message should be accepted.

Quick reference

Hash subkey
\[ H=E_K(0^{128}) \]

\(H\) is the key-derived ingredient used by GHASH.

GCM safety rule

Never reuse a nonce with the same key in AES-GCM. Nonce reuse can break the mode's security promises.

Note for self-study

Part I: Concept checks

I1 sample answer. AES-GCM has a CTR-style encryption system that hides the plaintext and a GHASH authentication system that helps produce a tag over the ciphertext and AAD.

I2 sample answer. GHASH uses arithmetic in the finite field \(GF(2^{128})\), which is why the word Galois appears in Galois/Counter Mode.

I3 sample answer. AAD is data that remains visible but is still authenticated. It is useful for headers or metadata that must be readable but should not be modifiable without detection.

I4 sample answer. AES-GCM is an authenticated encryption mode. It uses AES in a counter-like way to encrypt plaintext, and it uses GHASH over AAD and ciphertext to help produce an authentication tag. The receiver checks the tag before accepting the message. Nonce uniqueness with the same key is essential.
