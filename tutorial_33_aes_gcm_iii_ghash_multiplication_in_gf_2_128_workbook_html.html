How to use this workbook

GHASH multiplication is easier if you keep three ideas separate: represent bitstrings as polynomials, add matching terms with XOR, and reduce high-degree products back into the 128-bit field using the GHASH field polynomial.

Quick reference

GHASH field polynomial
\[ p(x)=x^{128}+x^7+x^2+x+1 \] \[ x^{128}\equiv x^7+x^2+x+1 \]
GHASH accumulator
\[ Y_0=0 \] \[ Y_i=(Y_{i-1}\oplus X_i)\cdot H \]

Note for self-study

The workhorse idea is cancellation. Bit-polynomial arithmetic uses XOR as addition, so duplicate terms vanish. Reduction then folds the large product back into the field.

Part I: Concept checks

I1 sample answer. A bitstring can be treated as a polynomial by letting each bit decide whether a power of \(x\) is included. A 1 includes the term, and a 0 leaves it out.

I2 sample answer. In \(GF(2)\), coefficients are added with XOR. Since \(1\oplus1=0\), \(x^2+x^2=0\).

I3 sample answer. GHASH needs an irreducible polynomial to define the finite field and provide a reduction rule that folds high-degree products back into valid 128-bit field elements.

I4 sample answer. GHASH multiplication treats 128-bit blocks as polynomials with bit coefficients. These polynomials are multiplied using XOR-style addition, so repeated terms cancel. The raw product may be larger than 128 bits, so it is reduced modulo \(x^{128}+x^7+x^2+x+1\). The result is another 128-bit field element used in the GHASH accumulator.
