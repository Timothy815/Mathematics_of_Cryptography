AES-GCM IV: Toy GHASH in a Small Field Before Returning to 128 Bits

A hand-worked miniature GHASH model using \(GF(2^4)\), so the accumulator cycle becomes visible before returning to full AES-GCM.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

Tutorial 33 explained GHASH multiplication in \(GF(2^{128})\), but 128-bit arithmetic is too large to comfortably work by hand. This tutorial builds a toy GHASH in \(GF(2^4)\), using 4-bit blocks, so you can see every XOR, multiplication, and reduction step.

This toy GHASH is a tabletop model. It is not secure and is not the real AES-GCM field, but it uses the same kind of accumulator rhythm: mix in a block, multiply by a hash subkey, reduce, carry forward.

What you should know first

You should know that a bitstring can be read as a polynomial, that addition in \(GF(2)\) is XOR, and that GHASH uses an accumulator update of the form \(Y_i=(Y_{i-1}\oplus X_i)\cdot H\). You should also remember that real GHASH uses \(GF(2^{128})\), while this tutorial uses \(GF(2^4)\) only for learning.

The toy field: \(GF(2^4)\)

Definition: toy field

In this tutorial, \(GF(2^4)\) means a finite field whose values are represented by 4-bit blocks. A 4-bit block corresponds to a polynomial of degree at most 3.

We will use this toy irreducible polynomial:

\[ m(x)=x^4+x+1 \]

Since we reduce modulo \(m(x)\), the folding rule is:

\[ x^4\equiv x+1 \]
Why this field helps

Real GHASH uses 128-bit blocks and the polynomial \(x^{128}+x^7+x^2+x+1\). This toy version uses 4-bit blocks and \(x^4+x+1\). Same style of machine, smaller gears.

The toy GHASH update rule

We will use the same accumulator shape as GHASH:

\[ Y_0=0000 \] \[ Y_i=(Y_{i-1}\oplus X_i)\cdot H \]

In this toy lesson:

  • Each \(Y_i\), \(X_i\), and \(H\) is 4 bits.
  • \(\oplus\) means XOR the 4-bit blocks.
  • Multiplication happens as polynomial multiplication over bits.
  • Reduction uses \(m(x)=x^4+x+1\).

Worked example setup

We will process two toy input blocks using a fixed toy hash subkey:

ValueBitsPolynomial
\(Y_0\)0000\(0\)
\(H\)0011\(x+1\)
\(X_1\)1011\(x^3+x+1\)
\(X_2\)0110\(x^2+x\)

Step 1: Process \(X_1\)

Start with:

\[ Y_1=(Y_0\oplus X_1)\cdot H \]

1A. XOR the accumulator with the input block

\[ Y_0\oplus X_1=0000\oplus1011=1011 \]

As a polynomial:

\[ 1011=x^3+x+1 \]

1B. Multiply by \(H\)

\[ (x^3+x+1)(x+1) \] \[ =x^4+x^3+x^2+x+x+1 \] \[ =x^4+x^3+x^2+1 \]

The two \(x\) terms cancel because \(x+x=0\) in \(GF(2)\).

1C. Reduce using \(x^4\equiv x+1\)

\[ x^4+x^3+x^2+1\equiv (x+1)+x^3+x^2+1 \] \[ =x^3+x^2+x+(1+1) \] \[ =x^3+x^2+x \]

So:

\[ Y_1=x^3+x^2+x=1110 \]

Step 2: Process \(X_2\)

Now use \(Y_1=1110\) and \(X_2=0110\):

\[ Y_2=(Y_1\oplus X_2)\cdot H \]

2A. XOR the accumulator with the input block

\[ Y_1\oplus X_2=1110\oplus0110=1000 \]

As a polynomial:

\[ 1000=x^3 \]

2B. Multiply by \(H\)

\[ x^3(x+1)=x^4+x^3 \]

2C. Reduce using \(x^4\equiv x+1\)

\[ x^4+x^3\equiv (x+1)+x^3 \] \[ =x^3+x+1 \]

So the final toy GHASH value after two blocks is:

\[ Y_2=x^3+x+1=1011 \]

Common mistakes

Mistake 1: thinking the toy field is real GCM

The toy field is only a learning model. Real AES-GCM uses \(GF(2^{128})\), not \(GF(2^4)\).

Mistake 2: forgetting to reduce after multiplication

If multiplication creates \(x^4\) or higher in the toy field, use \(x^4\equiv x+1\) to fold it back.

Mistake 3: using ordinary addition instead of XOR cancellation

In this arithmetic, \(x+x=0\), not \(2x\).

Mistake 4: losing the accumulator

Each new block depends on the previous accumulator value. The accumulator is the thread tying the blocks together.

Why this matters for cryptography

Toy models are not shortcuts around rigor. They are microscopes. By shrinking GHASH to 4-bit blocks, we can see the moving parts: the accumulator carries history, XOR mixes in the next block, multiplication by \(H\) transforms the state, and reduction keeps the result inside the field.

  • Toy GHASH uses the same accumulator shape as real GHASH.
  • Every block changes the running value.
  • Multiplication by \(H\) is field multiplication.
  • Reduction keeps the result inside the field.
  • The real version scales this idea to 128-bit field elements.

What to remember
  • This tutorial uses toy \(GF(2^4)\), not real \(GF(2^{128})\).
  • The toy block size is 4 bits.
  • The toy reduction polynomial is \(m(x)=x^4+x+1\).
  • The toy folding rule is \(x^4\equiv x+1\).
  • The accumulator starts at \(Y_0=0000\).
  • The update rule is \(Y_i=(Y_{i-1}\oplus X_i)\cdot H\).
  • In the worked example, \(H=0011=x+1\).
  • After \(X_1=1011\), the accumulator becomes \(Y_1=1110\).
  • After \(X_2=0110\), the accumulator becomes \(Y_2=1011\).
  • Real GHASH uses the same structure with 128-bit blocks.
