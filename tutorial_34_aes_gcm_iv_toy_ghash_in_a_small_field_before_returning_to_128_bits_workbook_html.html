How to use this workbook

Work slowly through each toy GHASH step. The goal is not speed. The goal is to see the rhythm: convert bits to polynomials, XOR the accumulator with the next block, multiply by \(H\), reduce back to 4 bits, then carry the result forward.

Toy GHASH update

Part E: Multiply by \(H=0011=x+1\)

E1. \((x^3+x+1)(x+1)=x^4+x^3+x^2+x+x+1=x^4+x^3+x^2+1\). Reduce: \((x+1)+x^3+x^2+1=x^3+x^2+x\), so result is 1110.

E2. \(x^3(x+1)=x^4+x^3\). Reduce: \((x+1)+x^3=x^3+x+1\), so result is 1011.

E3. \((x^2+x)(x+1)=x^3+x^2+x^2+x=x^3+x\), so result is 1010.

Part G: New toy GHASH example

G1. \(Y_0\oplus X_1=0000\oplus0101=0101=x^2+1\).

Multiply by \(H=x+1\): \((x^2+1)(x+1)=x^3+x^2+x+1\). This already has degree 3, so no reduction is needed.

Therefore \(Y_1=x^3+x^2+x+1=1111\).

G2. \(Y_1\oplus X_2=1111\oplus0010=1101=x^3+x^2+1\).

Multiply by \(H=x+1\): \((x^3+x^2+1)(x+1)=x^4+x^3+x^3+x^2+x+1=x^4+x^2+x+1\).

Reduce: \((x+1)+x^2+x+1=x^2\). Therefore \(Y_2=0100\).

Part I: Concept checks

I1 sample answer. The toy field is useful because the blocks are small enough to work by hand. It lets us see the GHASH rhythm without being buried under 128-bit arithmetic.

I2 sample answer. The accumulator carries the result from one block to the next. Each new block is mixed with the previous accumulator, so the final value depends on the sequence of blocks.

I3 sample answer. Multiplication can create terms too large for the field. Reduction folds those terms back into valid field elements using the reduction polynomial.

I4 sample answer. Toy GHASH starts with \(Y_0=0000\). For each 4-bit input block, it XORs the block with the previous accumulator, multiplies the result by the toy hash subkey \(H\), and reduces the product modulo \(x^4+x+1\). The same accumulator structure is used by real GHASH, but real GHASH uses 128-bit blocks.
