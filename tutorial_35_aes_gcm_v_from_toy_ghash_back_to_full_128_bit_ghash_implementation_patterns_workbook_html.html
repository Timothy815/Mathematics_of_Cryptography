How to use this workbook

This workbook is about connecting math to implementation patterns. You are not writing production cryptography here. You are learning how the same GHASH rhythm appears in real code: XOR, carry-less multiply, reduce, and keep timing behavior disciplined.

Quick reference

Core GHASH formulas
\[ Y_i=(Y_{i-1}\oplus X_i)\cdot H \] \[ p(x)=x^{128}+x^7+x^2+x+1 \] \[ x^{128}\equiv x^7+x^2+x+1 \]
Implementation caution

Equivalent GHASH implementations can look different because of bit ordering, shift direction, reduction constants, and hardware instructions. Use vetted libraries for real systems.

QuestionAnswer
A1. What is carry-less multiplication?Polynomial multiplication over bits, where matching terms combine by XOR and no carries occur.
A2. What is a shift in this implementation context?A movement of bits that corresponds to moving polynomial powers, depending on bit-order convention.
A3. What is a conditional XOR?An XOR performed only when a particular bit condition is true, often implemented with masks for constant-time behavior.
A4. What is a reduction constant?A compact implementation encoding of the field polynomial's folding rule.
A5. What does constant-time mean?Avoiding timing behavior, branches, or memory access patterns that depend on secret data.
A6. What does CLMUL or PMULL help with?Hardware-assisted carry-less multiplication.

Part D: Carry-less multiplication reasoning

D1 sample answer. Ordinary multiplication includes carries between bit positions. Carry-less multiplication treats values as polynomials over bits, so terms combine by XOR and no carries occur.

D2 sample answer. A 1-bit represents an included power of \(x\). Multiplying by that power shifts the other polynomial, so each 1-bit selects a shifted copy.

D3 sample answer. If a bit is 1, that power of \(x\) contributes to the product, so the implementation XORs in the corresponding shifted copy of \(B\). If the bit is 0, that shifted copy contributes nothing.

E1 sample answer. Reduction can be interleaved with shifting because field reduction preserves equivalence modulo the field polynomial. Folding overflow terms as they appear keeps intermediate values manageable.

Part F: Reduction constants and bit ordering

F1 sample answer. A reduction constant encodes the field polynomial's folding rule in a form useful for XORing during implementation.

F2 sample answer. Different implementations may use different bit-order conventions, byte ordering, or shift directions. The same mathematical rule can therefore appear as different constants.

F3 sample answer. Line-by-line comparison can be misleading because equivalent GHASH implementations may represent polynomial coefficients differently while computing the same field operation.

Part J: Concept checks

J1 sample answer. The toy model and real GHASH use the same accumulator rule: XOR in a block, multiply by \(H\), reduce, and carry the result forward. Real GHASH scales the block size from 4 bits to 128 bits and uses the GCM field polynomial.

J2 sample answer. XOR performs addition in \(GF(2)\), shifts represent movement of polynomial powers, and reduction folds overflow terms back into the field. Together, these describe the core finite-field multiplication pattern.

J3 sample answer. Understanding patterns helps you read and reason about cryptographic algorithms. Writing production cryptography requires resistance to side channels, exact standards compliance, extensive testing, and expert review.

J4 sample answer. Real GHASH uses the same XOR, multiply, reduce rhythm as the toy model, but over \(GF(2^{128})\). Implementations use carry-less multiplication, shifts, reduction constants, and sometimes hardware instructions to compute the field multiplication efficiently. Bit ordering can make code look different, and constant-time discipline matters for safety. Real systems should rely on vetted libraries.
