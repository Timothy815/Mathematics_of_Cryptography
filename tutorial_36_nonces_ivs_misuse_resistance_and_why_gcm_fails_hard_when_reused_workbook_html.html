How to use this workbook

Focus on the security requirement: in AES-GCM, a nonce does not need to be secret, but it must be unique under the same key. Many of the problems below ask you to distinguish secrecy, randomness, uniqueness, and misuse resistance.

Quick reference

Core GCM nonce rule

For AES-GCM, the nonce/IV does not need to be secret, but it must not repeat with the same key.

Core reuse equation
\[ C_1=P_1\oplus S \] \[ C_2=P_2\oplus S \] \[ C_1\oplus C_2=P_1\oplus P_2 \]

Part D: Reuse equation practice

StepExpressionExplanation
1\(C_1=P_1\oplus S\)First plaintext is encrypted with keystream \(S\).
2\(C_2=P_2\oplus S\)Second plaintext is encrypted with the same reused keystream \(S\).
3\(C_1\oplus C_2=(P_1\oplus S)\oplus(P_2\oplus S)\)Substitute both ciphertext definitions.
4\(S\oplus S=0\)A value XORed with itself cancels.
5\(C_1\oplus C_2=P_1\oplus P_2\)The keystream cancels, leaving a relationship between plaintexts.

D1 sample answer. \(P_1\oplus P_2\) reveals a relationship between the two plaintexts. If one message is known, guessed, or structured, information about the other may be recovered.

Part E: Authentication risk

E1 sample answer. GCM includes CTR-like encryption and GHASH authentication. Nonce reuse can repeat the keystream and can also expose algebraic relationships involving tags and GHASH.

E2 sample answer. GHASH is algebraic because it uses finite-field arithmetic. Relationships between inputs and outputs can be manipulated mathematically.

Part K: Concept checks

K1 sample answer. AES-GCM's security assumes a key/nonce pair is not reused. The nonce controls the CTR keystream and affects authentication behavior, so uniqueness is part of the security design.

K2 sample answer. If two messages use the same keystream \(S\), then \(C_1=P_1\oplus S\) and \(C_2=P_2\oplus S\). XORing the ciphertexts makes \(S\oplus S=0\), leaving \(P_1\oplus P_2\).

K3 sample answer. Distributed systems are tricky because multiple machines may encrypt independently. Without coordination, they can choose overlapping counters or repeated nonce ranges under the same key.

K4 sample answer. AES-GCM nonces do not need to be secret, but they must be unique under the same key. Reusing a nonce can repeat CTR keystream, causing ciphertexts to reveal relationships between plaintexts. It can also damage authentication because GHASH is algebraic. Safe systems need careful nonce management, and misuse-resistant modes such as AES-GCM-SIV are designed to reduce damage from certain nonce mistakes.
