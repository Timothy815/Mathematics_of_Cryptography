AES-GCM-SIV and Synthetic IV Modes at a High Level

A beginner-friendly bridge from AES-GCM nonce fragility to synthetic IV authenticated encryption and nonce-misuse resistance.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

Tutorial 36 explained that AES-GCM can fail badly if the same nonce is reused with the same key. This tutorial introduces a design response: AES-GCM-SIV and the broader idea of synthetic IV modes. The goal is to understand the design philosophy before studying POLYVAL or the detailed construction.

AES-GCM says, “Give me a unique nonce, and I will be fast.” AES-GCM-SIV says, “I still want good nonce behavior, but I will build more guardrails into the machine.”

What you should know first

You should know that AES-GCM is an AEAD mode, that it uses CTR-style encryption and GHASH authentication, and that nonce reuse can damage confidentiality and authentication. You should also know that misuse resistance means a design fails less catastrophically under certain common mistakes, not that mistakes are harmless.

The problem AES-GCM-SIV tries to reduce

AES-GCM is strong when used correctly, but it depends on strict nonce uniqueness. If the same key and nonce are reused, the CTR-style keystream can repeat, and GHASH relationships can weaken authentication.

\[ C_1=P_1\oplus S \] \[ C_2=P_2\oplus S \] \[ C_1\oplus C_2=P_1\oplus P_2 \]
The AES-GCM danger

Reusing a nonce with AES-GCM can expose relationships between plaintexts and can also harm authentication. This is why GCM is often described as fast but fragile when nonce management fails.

AES-GCM-SIV was designed to reduce this kind of catastrophic failure. It is still best to use unique nonces, but the construction is more forgiving if the nonce system stumbles.

What “synthetic IV” means

Definition: synthetic IV

A synthetic IV is an initialization value computed from cryptographic inputs rather than simply supplied from outside. In AES-GCM-SIV, the construction authenticates/hashes the nonce, plaintext, and AAD to produce a synthetic IV-like value before encryption.

Normal IV idea

The system provides a nonce or IV, and the encryption mode uses it as part of the starting point.

external nonce → encryption starts

Synthetic IV idea

The mode computes a starting value from the data being protected and then uses that value for encryption.

key + nonce + AAD + plaintext
        ↓
 synthetic IV / tag
        ↓
 encryption starts
Friendly analogy

A normal IV is like being handed a starting address. A synthetic IV is like inspecting the package and paperwork first, then generating the starting address from that inspection.

At a high level, AES-GCM-SIV changes the order of operations. It computes a synthetic IV/authentication value first, then uses that value for counter-style encryption.

AES-GCM-SIV is related to AES-GCM, but it is not just AES-GCM with a different nonce size. The construction changes the logic of how the starting value is created.

POLYVAL: GHASH’s cousin

AES-GCM-SIV uses a finite-field authenticator called POLYVAL. For now, do not worry about the detailed field arithmetic. The key idea is that POLYVAL plays a GHASH-like role in AES-GCM-SIV.

Save the details for later

POLYVAL is closely related to GHASH but uses a different convention that is friendlier for certain little-endian implementations. The exact math deserves its own tutorial.

What happens under nonce reuse?

AES-GCM-SIV is designed so that nonce reuse does not immediately repeat the keystream in the same dangerous way as AES-GCM. Because the synthetic IV depends on the message and AAD, different plaintexts under the same nonce can still lead to different encryption starting values.

Careful wording

Nonce reuse is still not recommended. AES-GCM-SIV is more forgiving, not invincible. Misuse resistance means damage reduction, not permission to be careless.

There is also a tradeoff: if the same key, nonce, plaintext, and AAD are used again, the same output may be produced. That can reveal that the exact same protected input was repeated.

Why not always use SIV modes?

Misuse resistance is valuable, but every design has tradeoffs. AES-GCM is very fast, widely supported, and common in protocols and libraries. AES-GCM-SIV adds guardrails, but may have different performance, implementation, availability, and interoperability considerations.

Design goalAES-GCMAES-GCM-SIV

Design philosophy: fast trust versus defensive suspicion

AES-GCM and AES-GCM-SIV represent two useful design instincts.

AES-GCM instinct

“If the system gives me unique nonces, I can be extremely fast and effective.”

AES-GCM-SIV instinct

“I will build more of the starting value from the protected data, so nonce mistakes are less devastating.”

This is a recurring theme in cryptography: the fastest design often assumes disciplined use, while a more defensive design may spend extra work to reduce the blast radius of predictable mistakes.

Common mistakes

Mistake 1: thinking AES-GCM-SIV means nonces do not matter

Nonces still matter. AES-GCM-SIV is more forgiving, not magical.

Mistake 2: thinking misuse resistant means misuse proof

Misuse resistance limits damage from some mistakes. It does not make all incorrect use safe.

Mistake 3: thinking synthetic IV means random IV

A synthetic IV is computed from cryptographic inputs. It is not merely a random number.

Mistake 4: thinking AES-GCM-SIV is just AES-GCM with a different nonce size

AES-GCM-SIV changes the construction by computing a synthetic IV/authentication value before encryption.

Practice problems

  1. What problem is AES-GCM-SIV trying to reduce?
    Show solution

    It tries to reduce the catastrophic failure risk caused by nonce reuse in AES-GCM.

  2. Does nonce-misuse resistance mean nonce reuse is recommended?
    Show solution

    No. It means the design is more forgiving if nonce reuse happens, not that reuse is good practice.

  3. What does “synthetic IV” mean in plain English?
    Show solution

    It means the starting value is computed from cryptographic inputs rather than simply supplied from outside.

  4. What is the high-level order difference between AES-GCM and AES-GCM-SIV?
    Show solution

    AES-GCM encrypts with nonce/counter and authenticates. AES-GCM-SIV first computes a synthetic IV/authentication value, then encrypts using it.

  5. What is POLYVAL's beginner-level role?
    Show solution

    POLYVAL is a GHASH-like authentication/hash function used to help compute the synthetic IV in AES-GCM-SIV.

    Show solution

    If the same key, nonce, plaintext, and AAD are reused, deterministic output can reveal that the exact same protected input was repeated.

Why this matters for cryptography

AES-GCM-SIV shows how cryptographic design evolves around real engineering failure modes. AES-GCM is powerful but trusts nonce discipline. AES-GCM-SIV adds a defensive layer by deriving the starting value from the data being protected. This is not just a new mode name; it is a design philosophy about reducing the blast radius of predictable mistakes.

  • AES-GCM-SIV is an AEAD mode.
  • It is designed to be nonce-misuse resistant.
  • A synthetic IV is computed from cryptographic inputs.
  • POLYVAL is GHASH-like and helps compute the synthetic IV.
  • Nonce reuse is still not recommended.
  • Misuse resistant does not mean mistake proof.
