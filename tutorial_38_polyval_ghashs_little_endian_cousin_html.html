POLYVAL: GHASH’s Little-Endian Cousin

How POLYVAL relates to GHASH, why AES-GCM-SIV uses it, and what “little-endian cousin” means without getting lost in implementation fog.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

Tutorial 37 introduced AES-GCM-SIV and said that POLYVAL plays a GHASH-like role inside it. This tutorial explains what POLYVAL is at a high level, why it exists, how it resembles GHASH, and how bit-order conventions make it feel like GHASH’s little-endian cousin.

POLYVAL is not an entirely alien machine. It lives in the same finite-field neighborhood as GHASH, but it arranges the furniture differently so certain implementations can move through the room more naturally.

What you should know first

You should know that GHASH is the polynomial authentication function used in AES-GCM, that AES-GCM-SIV computes a synthetic IV before encryption, and that finite-field authenticators process blocks using XOR and multiplication in a binary field. You do not need to be fluent in the full POLYVAL reduction math yet.

The big picture

Intuition first

GHASH and POLYVAL are both polynomial authenticators. They take blocks of data, mix them with a hash key using finite-field arithmetic, and produce an authentication value. The main beginner difference is not their job. It is the convention they use for reading and multiplying the 128-bit field elements.

Key idea

In AES-GCM-SIV, POLYVAL helps bind the protected data into the synthetic IV. That synthetic IV then influences the encryption step.

What “endian” means

Endianness is about how a sequence is interpreted. When we look at a group of bytes or bits, do we treat the first part as the highest-value part or the lowest-value part?

ConventionPlain meaningCommon mental image
Big-endianMost significant part first.Read the largest place value first.
Little-endianLeast significant part first.Read the smallest place value first.

For a tiny 8-bit demonstration, imagine the bit pattern 10000001. It contains a 1 at each end. If we disagree about which end represents the highest power of \(x\), the polynomial interpretation changes.

Why this matters

GHASH and POLYVAL are closely related, but they use different conventions for how bits map into polynomial coefficients. That difference makes equivalent-looking ideas appear reversed or rearranged in code and specifications.

GHASH and POLYVAL field polynomials

GHASH and POLYVAL both work with 128-bit field elements, but their polynomial definitions are written differently because of their conventions.

FunctionField polynomial commonly associated with itWhat to remember now
GHASH\(x^{128}+x^7+x^2+x+1\)Used in AES-GCM.
POLYVAL\(x^{128}+x^{127}+x^{126}+x^{121}+1\)Used in AES-GCM-SIV.
Do not panic about the different polynomial

At this stage, treat the two polynomials as different written forms for related finite-field machinery. The important lesson is that POLYVAL is engineered around a different bit-order convention that fits AES-GCM-SIV’s implementation goals.

POLYVAL as an accumulator

Just like GHASH, POLYVAL processes formatted 128-bit blocks using an accumulator. The broad pattern is familiar:

\[ S_0=0 \] \[ S_i=(S_{i-1}\oplus X_i)\cdot H \]

Here:

SymbolRole
\(S_i\)The running POLYVAL accumulator after block \(i\).
\(X_i\)The current formatted input block.
\(H\)The POLYVAL hash key.
\(\oplus\)XOR, used as addition in the binary field.
Memory hook

If GHASH was “XOR a block, multiply by \(H\), repeat,” then POLYVAL is the same song played with a different instrument tuning.

Why AES-GCM-SIV uses POLYVAL

POLYVAL was chosen for AES-GCM-SIV because its convention is convenient for efficient implementations on common little-endian processors. It provides the polynomial-authentication machinery needed to compute the synthetic IV while fitting the implementation style of the mode.

ReasonMeaning
Related to GHASHThe conceptual learning transfers from GHASH to POLYVAL.
Little-endian-friendlyIts conventions better match many common processor layouts.
Supports SIV constructionIt helps compute the synthetic IV/authentication value.
Efficient finite-field operationIt can benefit from carry-less multiplication patterns.

A tiny toy comparison

Suppose we have a 4-bit pattern:

1010

If the leftmost bit is treated as the high power, the polynomial is:

\[ 1010 \rightarrow x^3+x \]

If the rightmost bit is treated as the high power, the same visual pattern could be interpreted differently:

\[ 1010 \rightarrow x^2+1 \]
What this toy example teaches

The bits did not change. The reading convention changed. This is why GHASH and POLYVAL can feel confusing at first: the arithmetic is related, but the map from bits to polynomials is different.

What not to overlearn yet

At this stage, do not try to memorize all bit-reversal formulas or implementation constants. The beginner goal is simpler:

  • POLYVAL is used in AES-GCM-SIV.
  • POLYVAL is GHASH-like.
  • POLYVAL processes 128-bit blocks using XOR and finite-field multiplication.
  • POLYVAL uses different endian/bit-order conventions from GHASH.
  • Those conventions make it convenient for AES-GCM-SIV implementations.

Common mistakes

Mistake 1: thinking POLYVAL is completely unrelated to GHASH

POLYVAL and GHASH are closely related polynomial authenticators. Their conventions differ, but their broad job is similar.

Mistake 2: thinking little-endian means weaker or less secure

Endianness is an interpretation convention, not a security ranking.

Mistake 3: comparing implementation constants without checking bit order

The same mathematical idea can appear different when bit order or shift direction changes.

Mistake 4: trying to master POLYVAL before understanding its role

First understand where POLYVAL sits in AES-GCM-SIV. Then study its exact arithmetic.

Why this matters for cryptography

POLYVAL shows that cryptographic mathematics is not only about abstract correctness. It is also about representation. The same broad finite-field idea can be arranged to fit a processor, a protocol, or a mode’s construction. This is where algebra meets the workbench: the field is the same kind of creature, but the harness is built for a different road.

  • POLYVAL is used in AES-GCM-SIV.
  • POLYVAL is related to GHASH.
  • Both use finite-field polynomial authentication ideas.
  • Endian and bit-order conventions affect how the math is represented.
  • POLYVAL helps compute the synthetic IV/authentication value.
  • Understanding POLYVAL prepares us for the detailed AES-GCM-SIV construction.
