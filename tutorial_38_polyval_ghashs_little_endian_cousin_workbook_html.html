How to use this workbook

This workbook focuses on POLYVAL’s role and interpretation. You are not expected to master every implementation constant yet. The goal is to understand that POLYVAL is GHASH-like, used in AES-GCM-SIV, and shaped by different endian/bit-order conventions.

Part G: POLYVAL accumulator reasoning

SymbolRole
\(S_i\)The running POLYVAL accumulator after block \(i\).
\(X_i\)The current formatted input block.
\(H\)The POLYVAL hash key.
\(\oplus\)XOR, used as addition in the binary field.

G1 sample answer. POLYVAL starts with a zero accumulator. Each formatted input block is XORed into the accumulator, then the result is multiplied by the hash key \(H\) in the finite field.

Part J: Concept checks

J1 sample answer. GHASH and POLYVAL are both polynomial authentication/hash functions that process 128-bit blocks using finite-field arithmetic. POLYVAL is used in AES-GCM-SIV and is closely related to GHASH.

J2 sample answer. POLYVAL is described as GHASH’s little-endian cousin because it performs a similar finite-field authentication job but uses different endian and bit-order conventions that fit AES-GCM-SIV implementations.

J3 sample answer. POLYVAL is important because it helps compute the synthetic IV/authentication material that AES-GCM-SIV uses before counter-style encryption.

J4 sample answer. POLYVAL is a GHASH-like polynomial authenticator used in AES-GCM-SIV. It processes 128-bit blocks with XOR and finite-field multiplication, but it uses different bit-order conventions from GHASH. Those conventions make POLYVAL more natural for AES-GCM-SIV’s implementation goals. Understanding POLYVAL’s role prepares us to study synthetic IV construction in detail.
