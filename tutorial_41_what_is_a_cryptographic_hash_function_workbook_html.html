How to use this workbook

This workbook opens Volume 2. Focus on the central contrast: encryption is reversible with a key, while hashing produces a fixed-length digest that is designed to be one-way. Work carefully through the resistance properties because they will become the vocabulary for the next several tutorials.

Quick reference

Core notation
\[ h=H(M) \]

Read as: digest \(h\) is produced by applying hash function \(H\) to message \(M\).

Hashing is not encryption

Encryption is reversible with the right key. Hashing is designed to be one-way and fixed-length.

Note for self-study

The big idea is that cryptographic hashes are fixed-length, deterministic, one-way fingerprints designed to resist deliberate attack.

Part E: Three core security properties

PropertyAttacker challengeCompact math form
Preimage resistanceGiven \(h\), find \(M\) such that \(H(M)=h\).Find \(M\) from \(h\).
Second preimage resistanceGiven \(M_1\), find \(M_2\ne M_1\) with the same digest.\(H(M_1)=H(M_2)\)
Collision resistanceFind any two different messages with the same digest.\(M_1\ne M_2\), \(H(M_1)=H(M_2)\)

Part F: Avalanche effect

F1 sample answer. The avalanche effect means a tiny change in the input should cause many unpredictable changes in the digest.

F2 sample answer. Predictable digest changes could help attackers manipulate inputs, infer structure, or search for collisions more efficiently.

Part L: Concept checks

L1 sample answer. Hashes are useful because even though collisions exist mathematically, secure hash functions make finding useful collisions infeasible. This allows digests to serve as practical fingerprints.

L2 sample answer. Preimage resistance starts with a digest and asks the attacker to find an input. Collision resistance asks the attacker to find any two different inputs with the same digest.

L3 sample answer. Anyone can compute a plain hash. Authenticity requires something only the legitimate party can produce or protect, such as a MAC or digital signature.

L4 sample answer. A cryptographic hash function maps input data to a fixed-length digest. It is deterministic, one-way by design, and should resist preimage, second preimage, and collision attacks. Hashing is not encryption because it is not meant to be reversed. Hashes are used for integrity checks, MACs, signatures, password hashing, Merkle trees, blockchains, and key derivation, but plain hashes do not prove authenticity by themselves.
