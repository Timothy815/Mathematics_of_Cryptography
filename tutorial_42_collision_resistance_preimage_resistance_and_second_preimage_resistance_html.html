Collision Resistance, Preimage Resistance, and Second Preimage Resistance

A careful separation of the three major hash-security games: finding an input from a digest, finding a matching input for a given message, and finding any two messages that collide.

Roadmap Previous Tutorial Practice One-page recap

Why this tutorial exists

Purpose

Tutorial 41 introduced cryptographic hash functions as fixed-length, one-way fingerprints. This tutorial slows down and separates the three major security properties that often get blurred together: preimage resistance, second preimage resistance, and collision resistance.

These three ideas are easy to confuse because they all involve hashes being “hard to break.” But each one gives the attacker a different starting point and a different goal. Think of them as three different games played on the same board.

What you should know first

You should know that a hash function maps a message \(M\) to a digest \(h\), written \(h=H(M)\). You should also know that hash outputs are fixed length, that collisions must exist in principle, and that security means collisions and reversals should be computationally infeasible to find.

The three attacker games

The easiest way to separate the properties is to ask: what is the attacker given, and what must the attacker find?

Preimage resistance

Given a digest, find any message that produces it.

Second preimage resistance

Given one message, find a different message with the same digest.

Collision resistance

Find any two different messages with the same digest.

PropertyAttacker is givenAttacker must findMath form
Preimage resistanceA digest \(h\)A message \(M\) such that \(H(M)=h\)Find \(M\) from \(h\)
Second preimage resistanceA specific message \(M_1\)A different message \(M_2\ne M_1\) with the same digest\(H(M_1)=H(M_2)\)
Collision resistanceNothing specific, just the hash functionAny two different messages with the same digest\(M_1\ne M_2\), \(H(M_1)=H(M_2)\)

Preimage resistance

Definition: preimage resistance

A hash function is preimage resistant if, given a digest \(h\), it is computationally infeasible to find any message \(M\) such that \(H(M)=h\).

\[ \text{Given } h, \text{ find } M \text{ such that } H(M)=h. \]
Preimage game
Given:
digest \(h\)
→
Attacker searches
→
Goal:
some \(M\) with \(H(M)=h\)

Preimage resistance is the “do not reverse the fingerprint” property. If someone sees only the digest, they should not be able to work backward to a message that produces it.

Memory hook

Preimage resistance asks: “Can you find any original-looking document that fits this fingerprint?”

Second preimage resistance

Definition: second preimage resistance

A hash function is second preimage resistant if, given a specific message \(M_1\), it is computationally infeasible to find a different message \(M_2\ne M_1\) such that \(H(M_1)=H(M_2)\).

\[ \text{Given } M_1, \text{ find } M_2\ne M_1 \text{ such that } H(M_1)=H(M_2). \]
Second preimage game
Given:

Second preimage resistance is important when a specific document already exists. For example, if a contract has a digest, an attacker should not be able to create a different contract with the same digest.

Memory hook

Second preimage resistance asks: “Given this exact document, can you forge another document with the same fingerprint?”

Collision resistance

Definition: collision resistance

A hash function is collision resistant if it is computationally infeasible to find any two different messages \(M_1\ne M_2\) such that \(H(M_1)=H(M_2)\).

\[ \text{Find } M_1\ne M_2 \text{ such that } H(M_1)=H(M_2). \]
Collision game
Given:
hash function \(H\)
→
Attacker searches freely
→
Goal:
any \(M_1\ne M_2\)
same digest

Collision resistance is a more flexible attacker game than second preimage resistance because the attacker does not have to match a specific existing message. They can choose both messages.

Memory hook

Collision resistance asks: “Can you create any two different documents that share a fingerprint?”

Similar, but not the same

Second preimage resistance and collision resistance both involve two messages with the same digest, but the attacker’s freedom is different.

Why collisions must exist

Fixed-length hash functions have a finite output space. But the possible input space is much larger. By the pigeonhole principle, at least two different inputs must share some digest.

Pigeonhole principle

If more pigeons than holes are placed into holes, at least one hole must contain more than one pigeon. For hashes, messages are pigeons and digests are holes.

The security claim is not “collisions do not exist.” The security claim is “collisions should be infeasible to find.”

Output sizeNumber of possible digestsCollision reality
8 bits\(2^8=256\)Collisions are easy to find.
128 bits\(2^{128}\)Huge space, but birthday attacks matter.
256 bits\(2^{256}\)Enormous space; secure hashes aim to make attacks infeasible.

A preview of security levels

For an ideal \(n\)-bit hash function, brute-force preimage search is roughly on the order of \(2^n\) work. Collision search is different because of the birthday phenomenon, which we will study next.

Attack typeIdeal rough work for \(n\)-bit hashWhy
Preimage attackAbout \(2^n\)Need to hit a specific digest.
Second preimage attackAbout \(2^n\)Need to match a specific message’s digest.
Collision attackAbout \(2^{n/2}\)Birthday phenomenon makes any matching pair easier to find.
Do not memorize blindly yet

The \(2^{n/2}\) collision estimate deserves its own tutorial. For now, remember the shape: finding any pair is easier than hitting a specific target.

Practical examples

SituationMost relevant propertyWhy
Someone sees a digest and tries to find any matching message.Preimage resistance.The attacker starts from only the digest.
Someone has a signed contract and tries to create a different contract with the same digest.Second preimage resistance.The attacker must match a specific existing message.
Someone creates two different files with the same digest before choosing which one to use.Collision resistance.The attacker chooses both messages.
Someone wants to replace a software download while keeping the same trusted digest.Second preimage resistance.The attacker wants a malicious file matching a known digest.

Common mistakes

Mistake 1: treating all hash attacks as “finding a collision”

Not every attack is a collision attack. Sometimes the attacker must match a specific digest or a specific message.

Mistake 2: confusing second preimage resistance with collision resistance

In a second preimage attack, the first message is fixed. In a collision attack, the attacker can choose both messages.

Mistake 3: thinking collisions are impossible

Collisions exist for fixed-length hashes. The goal is making them infeasible to find.

Mistake 4: assuming avalanche equals security

Avalanche behavior is important, but it does not by itself prove preimage, second preimage, or collision resistance.

Practice problems

  1. In a preimage attack, what is the attacker given?
    Show solution

    The attacker is given a digest \(h\).

  2. In a second preimage attack, what is fixed in advance?
    Show solution

    The first message \(M_1\) is fixed in advance.

  3. In a collision attack, can the attacker choose both messages?
    Show solution

    Yes. The attacker can search for any two different messages with the same digest.

  4. Why must collisions exist for fixed-length hash functions?
    Show solution

    Because the input space is larger than the finite output space, so multiple inputs must map to at least some outputs.

  5. Which is usually easier for an ideal \(n\)-bit hash: finding any collision or finding a preimage for a specific digest?
    Show solution

    Finding any collision is usually easier because of the birthday phenomenon, roughly \(2^{n/2}\) work instead of \(2^n\).

  6. Give a one-sentence difference between second preimage resistance and collision resistance.
    Show solution

    Second preimage resistance requires matching a specific given message, while collision resistance allows the attacker to choose both messages freely.

Why this matters for cryptography

These three properties are the grammar of hash-function security. Digital signatures, file integrity checks, certificates, Merkle trees, blockchains, commitments, and password-related systems all rely on different forms of “hard to match this digest” or “hard to make two things share a digest.” Without this vocabulary, later hash constructions become a fog bank with equations floating in it.

  • Preimage resistance protects against reversing a digest.
  • Second preimage resistance protects a specific existing message from substitution.
  • Collision resistance protects against freely generated matching pairs.
  • Collisions must exist, but secure hashes make them infeasible to find.
  • Collision attacks are affected by the birthday phenomenon.
  • Hash length matters because it determines the size of the search space.
