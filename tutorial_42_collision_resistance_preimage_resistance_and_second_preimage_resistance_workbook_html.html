How to use this workbook

The goal is to separate three similar-looking ideas. For every problem, ask: What is the attacker given? What must the attacker find? Is the attacker matching a specific target, a specific message, or any pair?

Quick reference

Memory phrase

Second preimage:

Collision:
Find any matching pair.

Part B: The three attacker games

PropertyAttacker is givenAttacker must findMath form
Preimage resistanceA digest \(h\)A message \(M\) such that \(H(M)=h\)Find \(M\) from \(h\)
Second preimage resistanceA specific message \(M_1\)A different message \(M_2\ne M_1\) with the same digest\(H(M_1)=H(M_2)\)
Collision resistanceNothing specific, just the hash functionAny two different messages with the same digest\(M_1\ne M_2\), \(H(M_1)=H(M_2)\)

Part C: Definition practice

C1 sample answer. Preimage resistance means that given only a digest, it should be infeasible to find any input that hashes to that digest.

C2 sample answer. Second preimage resistance means that given a specific message, it should be infeasible to find a different message with the same digest.

C3 sample answer. Collision resistance means it should be infeasible to find any two different messages that share the same digest.

Part E: Second preimage versus collision resistance

QuestionSecond preimage resistanceCollision resistance
Is the first message fixed in advance?Yes.No.
Can attacker choose both messages?No.Yes.
GoalMatch a specific message’s digest.Find any pair that shares a digest.

E1 sample answer. In a second preimage attack, the attacker must match a message that already exists. In a collision attack, the attacker can search freely for any two different inputs that collide.

Part F: Why collisions must exist

F1 sample answer. Messages are like pigeons and digests are like holes. Since there are far more possible messages than fixed-length digests, at least some digests must correspond to more than one message.

Output sizeNumber of possible digestsCollision reality
8 bits\(2^8=256\)Collisions are easy to find.
128 bits\(2^{128}\)Huge space, but birthday attacks matter.
256 bits\(2^{256}\)Enormous space; secure hashes aim to make attacks infeasible.

F2 sample answer. Collisions must exist for fixed-length hashes. The right security claim is that useful collisions should be infeasible to find.

Part K: Concept checks

K1 sample answer. Preimage resistance means it is hard to find an input for a given digest. Second preimage resistance means it is hard to find a different input with the same digest as a specific given message. Collision resistance means it is hard to find any two different messages with the same digest.

K2 sample answer. Collision resistance is affected by the birthday phenomenon because the attacker does not need to hit a specific digest. They only need any pair of matching digests, which gives many more chances for success.

K3 sample answer. Hash length matters because it determines the size of the digest space. Larger output sizes make brute-force preimage and collision searches harder.

K4 sample answer. This tutorial separated the three main hash resistance properties. Preimage resistance starts with a digest and asks the attacker to find an input. Second preimage resistance starts with a specific message and asks the attacker to find a different message with the same digest. Collision resistance lets the attacker choose both messages and asks for any matching pair. Collisions must exist, but secure hashes make useful collisions infeasible to find.
