How to use this workbook

Track the machinery carefully. Padding determines the block sequence. Length encoding records the original message size. A length extension attack does not reverse the hash; it continues the hash from a digest that behaves like a final chaining value.

Quick reference

Padding shape
\[ pad(M)=M\,\|\,1\,\|\,0\ldots0\,\|\,\text{len}(M) \]
Length extension shape
\[ \text{Given } H(M), \text{ compute } H(M\,\|\,pad(M)\,\|\,X) \]

Note for self-study

The hinge idea is this: a length extension attack does not recover hidden data. It treats the known digest as a state from which hashing can continue.

Part F: Define length extension attack

F1 sample answer. A length extension attack lets an attacker compute the hash of an unknown original message plus its valid padding plus attacker-chosen extra data, starting from the known digest and an estimated original length.

Part H: Naive secret-prefix MAC danger

QuestionAnswer
What is the intended purpose?To authenticate the message using a secret.
Why might someone think it is safe?They assume the secret prefix makes the digest unforgeable.
Why can it be dangerous with Merkle-Damgård-style hashes?The digest may allow continuation of the hash state, enabling length extension.
What should be used instead?HMAC or another standard MAC construction.

H1 sample answer. The attacker can use the visible digest as the final state after the secret and message were processed. If they guess the secret-prefix length, they can append glue padding and extra data, then continue hashing from that state.

L1 sample answer. HMAC does not simply place the key before the message. It uses inner and outer keyed hash layers with derived key pads, which prevents the naive length-extension problem.

Part O: Concept checks

O1 sample answer. Padding adds a marker, enough zeros, and often an encoded original length so that the message fits the block structure unambiguously. Length encoding helps bind the padded form to the original message length.

O2 sample answer. Length extension works by treating a known digest as the final chaining value of a Merkle-Damgård-style hash. If the attacker can guess the original length, they can reconstruct the needed padding and continue hashing attacker-chosen extension blocks.

O3 sample answer. Hash(secret || message) can be dangerous because the digest may allow the attacker to continue hashing from the state after the secret and message, producing a valid tag for a longer message without knowing the secret.

O4 sample answer. This tutorial explained how padding and length encoding define the block sequence for Merkle-Damgård-style hashes. It then showed that the final digest can act like a visible chaining state, enabling length extension attacks in naive constructions such as Hash(secret || message). The attack does not reveal the secret or reverse the hash; it continues hashing additional data. The correct keyed-hash construction is HMAC.
