How to use this workbook

Focus on the difference between a plain hash, a naive secret-prefix hash, and HMAC. HMAC is not just “hash the key and message.” Its inner-and-outer structure is the important part.

Quick reference

HMAC formula
\[ HMAC(K,M)=H((K'\oplus opad)\,\|\,H((K'\oplus ipad)\,\|\,M)) \]
Memory phrase
Inner hash:
key-with-inner-pad + message

Outer hash:
key-with-outer-pad + inner digest

Note for self-study

The central distinction is this: HMAC is not a raw secret-prefix hash. It is a two-layer keyed construction built to authenticate messages safely using a hash function.

Part D: Unpacking the HMAC formula

PieceMeaning
\(K'\oplus ipad\)The block-sized key mixed with the inner pad.
\(H((K'\oplus ipad)\,\|\,M)\)The inner hash over the inner keyed prefix and message.
\(K'\oplus opad\)The block-sized key mixed with the outer pad.
\(H((K'\oplus opad)\,\|\,\text{inner digest})\)The outer hash that produces the final HMAC tag.

D1 sample answer. HMAC first mixes the key with an inner pad and hashes that with the message. Then it mixes the key with an outer pad and hashes that with the inner digest to produce the final tag.

Part E: Inner and outer layers

LayerInputOutput
Inner hash\((K'\oplus ipad)\,\|\,M\)Inner digest
Outer hash\((K'\oplus opad)\,\|\,\text{inner digest}\)Final HMAC tag

E1 sample answer. The two-layer structure prevents the visible tag from simply being the final state of a secret-prefix hash over the original message. The protected message is inside the inner hash, and the final tag comes from an outer keyed hash.

Part F: Key processing

Key caseWhat HMAC does
Key is longer than block sizeHash the key first, then pad as needed.
Key is shorter than block sizePad it with zeros to the block size.
Key is exactly block sizeUse it as the block-sized key.

F1 sample answer. HMAC XORs the key with block-sized pad patterns, so the key must first be represented at the hash function’s block size.

Part G: ipad and opad

PadUsed where?Purpose
ipadInner hashCreates the inner keyed prefix.
opadOuter hashCreates the outer keyed prefix.

G1 sample answer. The two different key-derived values separate the inner and outer hash roles, preventing HMAC from collapsing into a simple secret-prefix hash.

Part H: Why HMAC avoids the naive length-extension problem

ConstructionLength-extension riskReason
\(H(K\,\|\,M)\)DangerousVisible tag may let attacker continue from secret-prefix state.
HMACDesigned to avoid this problemUses separate inner and outer keyed hashes.

H1 sample answer. The visible HMAC tag is produced by the outer hash over an inner digest, not directly by hashing the original message with a secret prefix. An attacker cannot compute a valid new inner digest for an extended message without the key.

Part O: Concept checks

O1 sample answer. HMAC is a standard Message Authentication Code built from a hash function and a secret key. It computes an inner hash using the key mixed with an inner pad and the message, then computes an outer hash using the key mixed with an outer pad and the inner digest.

O2 sample answer. \(H(K\,\|\,M)\) is a simple secret-prefix hash and can be vulnerable to length extension with Merkle-Damgård-style hashes. HMAC uses two keyed layers with separate pads, so it is a different and safer construction.

O3 sample answer. HMAC uses a shared secret key, so only parties with that key can verify the tag. A digital signature uses a private/public key pair, allowing anyone with the public key to verify the signature.

O4 sample answer. This tutorial introduced HMAC as a safe keyed-hash construction. It reviewed why \(H(K\,\|\,M)\) is unsafe with Merkle-Damgård-style hashes, then showed that HMAC uses an inner hash and an outer hash with key-derived pads. HMAC provides integrity and authenticity in shared-key settings, but it does not encrypt and does not provide public verification like a digital signature.
