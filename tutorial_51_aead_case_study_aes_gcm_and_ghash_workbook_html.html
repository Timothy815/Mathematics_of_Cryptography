How to use this workbook

Keep AES-GCM separated into its two cooperating paths: AES-CTR hides the plaintext, and GHASH authenticates the associated data and ciphertext using finite-field arithmetic.

Quick reference

AES-CTR confidentiality
\[ S_i=AES_K(ctr_i) \] \[ C_i=P_i\oplus S_i \] \[ P_i=C_i\oplus S_i \]
GHASH authentication
\[ H=AES_K(0^{128}) \] \[ Y_0=0 \] \[ Y_i=(Y_{i-1}\oplus X_i)\cdot H \] \[ T=AES_K(J_0)\oplus GHASH_H(A,C) \]

B1 sample answer. AES-GCM has one path that encrypts plaintext using AES-CTR and another path that authenticates associated data and ciphertext using GHASH. The two paths come together in the final tag.

Part C: Counter mode practice

FormulaMeaning
\(S_i=AES_K(ctr_i)\)AES encrypts counter block \(ctr_i\) to make keystream block \(S_i\).
\(C_i=P_i\oplus S_i\)Plaintext block is XORed with keystream to produce ciphertext.
\(P_i=C_i\oplus S_i\)Ciphertext is XORed with same keystream to recover plaintext.

C1 sample answer. AES-CTR generates keystream blocks from encrypted counters, then XORs those blocks with plaintext. That resembles a stream cipher because encryption is plaintext XOR keystream.

Part D: GHASH purpose

GHASH factMeaning
GHASH authenticatesIt helps bind associated data and ciphertext into the tag.
GHASH does not encryptIt does not hide plaintext or ciphertext contents.
GHASH uses \(H=AES_K(0^{128})\)The hash subkey is derived by encrypting the all-zero block with AES.
GHASH uses \(GF(2^{128})\)It treats 128-bit blocks as finite-field elements.

D1 sample answer. GHASH is a keyed authentication component inside AES-GCM. It is not designed as a standalone public hash function like SHA-256 or as a general MAC like HMAC outside its specified mode.

Part E: Finite-field connection

E1 sample answer. GHASH uses the same broad idea as binary finite-field arithmetic studied earlier: bit strings represent polynomials, addition is XOR, multiplication is carryless polynomial multiplication, and reduction keeps the result inside the field.

Part J: Nonce reuse catastrophe

FailureWhat can go wrong?
CTR keystream reuseTwo ciphertexts may reveal relationships between two plaintexts.
GHASH structure exposureRepeated nonces can help attackers derive information about authentication equations.
Forgery riskAuthentication security may collapse under serious nonce misuse.
Protocol failureThe mode may no longer provide its promised AEAD security.

J1 sample answer. Nonce reuse can reuse CTR keystream and weaken the authentication equations in GHASH. It can break both confidentiality and authenticity, so it is a serious cryptographic failure.

Part O: Concept checks

O1 sample answer. AES-GCM is an AEAD mode that combines AES in counter mode for encryption with GHASH for authentication. AES-CTR hides the plaintext, while GHASH authenticates associated data and ciphertext.

O2 sample answer. GHASH is the authentication hash used inside GCM. It treats 128-bit blocks as elements of \(GF(2^{128})\), absorbs each block with XOR, and mixes the state using finite-field multiplication by the hash subkey \(H\).

O3 sample answer. AES-GCM uses the nonce to generate counter blocks and tag material. Reusing the same nonce with the same key can reuse keystream and weaken authentication, breaking the assumptions of the mode.

O4 sample answer. This tutorial studied AES-GCM as an AEAD case study. AES-CTR provides confidentiality by XORing plaintext with AES-generated keystream blocks. GHASH authenticates associated data and ciphertext using XOR and multiplication in \(GF(2^{128})\). The final tag combines GHASH with AES output on \(J_0\). The most important practical warning is that nonce reuse with the same key can be catastrophic.
