How to use this workbook

Note for self-study

The core idea is that cryptographic values have different randomness requirements. Keys need secret unpredictability, salts need uniqueness, and nonces usually need uniqueness under the same key.

Part N: Concept checks

N1 sample answer. Entropy measures uncertainty. In ideal terms, \(n\) bits of entropy means about \(2^n\) possible values, making guessing harder as \(n\) grows.

N2 sample answer. CSPRNGs matter because computers often need many random-looking values from limited entropy sources. A CSPRNG expands secure internal state into outputs that should remain unpredictable to attackers.

N3 sample answer. Secret means attackers do not know the value. Unique means the value does not repeat in a given context. Unpredictable means attackers cannot guess it ahead of time. Keys need secrecy and unpredictability; many nonces need uniqueness; salts need uniqueness but not secrecy.

N4 sample answer. This tutorial explained that cryptography depends on good randomness. Entropy measures uncertainty, and secure values should come from CSPRNGs or operating-system secure random APIs. Different cryptographic values have different needs: keys must be secret and unpredictable, salts should be unique, and nonces usually must not repeat. Bad randomness can produce predictable keys, repeated nonces, weak seeds, and biased outputs.
