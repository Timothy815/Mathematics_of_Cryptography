Public-Key Cryptography: One-Way Functions, Trapdoors, and Key Exchange

The doorway into asymmetric cryptography: how public/private key pairs, one-way functions, trapdoors, and key exchange allow strangers to create shared secrets over open channels.

🏠 Dashboard

Why this tutorial exists

Purpose

Volume 2 focused on hashes, MACs, authenticated encryption, key derivation, passwords, and randomness. Volume 3 begins the asymmetric world. Public-key cryptography lets people publish one key, keep another key secret, and still build secure systems over hostile networks.

Symmetric cryptography says, “First we must share a secret.” Public-key cryptography asks the stranger question: “Can we create trust or secrecy before we already share one?” This is the bridge over the moat.

What you should know first

You should know the difference between encryption, authentication, and key derivation. You should also remember that random keys must be unpredictable and that protocols must bind context carefully. Public-key cryptography does not replace these ideas; it helps establish the secrets and identities that later symmetric cryptography uses.

Symmetric versus public-key cryptography

Symmetric cryptography uses the same secret key, or closely related keys, for both sides. Public-key cryptography uses a pair: one key can be public, while the other must remain private.

TypeKeysMain challenge
SymmetricShared secret keyHow do both parties get the secret safely?
Public-keyPublic key plus private keyHow do we trust that a public key belongs to the right party?
Memory hook

Symmetric cryptography is a shared house key. Public-key cryptography is more like a mailbox slot and a private mailbox key: anyone can drop something in, but only the private-key holder can open it.

The public/private key pair

A public/private key pair is generated together. The public key is meant to be distributed. The private key must be protected. The security comes from a mathematical relationship that is easy to use in one direction but hard to reverse without the private information.

One-way functions

Definition: one-way function

A one-way function is easy to compute in the forward direction but hard to reverse without extra information.

\[ y=f(x) \quad \text{is easy to compute} \] \[ x=f^{-1}(y) \quad \text{is hard to find} \]

In cryptography, “hard” does not mean emotionally unpleasant; it means computationally infeasible for realistic attackers.

Candidate hard problemForward directionHard reverse direction
Integer factoringMultiply large primes.Recover primes from product.
Discrete logarithmCompute repeated multiplication or exponentiation in a group.Recover exponent.
Elliptic-curve discrete logarithmRepeatedly add a point on a curve.Recover scalar from resulting point.

Trapdoor functions

Definition: trapdoor function

A trapdoor function is a one-way function that becomes easy to reverse if you possess special secret information called the trapdoor.

The trapdoor is not a weakness. It is the intended secret shortcut. Public information lets others use the function one way. Private information lets the owner perform the special reverse or secret operation.

IdeaPublic sidePrivate/trapdoor side
RSA-style intuitionPublic modulus and exponent allow public operation.Private exponent allows secret reverse operation.
Signature intuitionAnyone can verify with public key.Only private key holder can produce valid signature.
Encryption intuitionAnyone can encrypt to public key.Only private key holder can decrypt.

Three major public-key jobs

JobWhat public key doesWhat private key does
Public-key encryptionLets others encrypt to you.Lets you decrypt.
Digital signaturesLets others verify your signature.Lets you sign.
Key exchangeLets parties combine public values.Lets each party compute shared secret.
Do not blur the jobs

Encryption keys, signing keys, and key-exchange keys should not be casually treated as interchangeable. Different job, different rules, different risks.

Key exchange: the big idea

Key exchange lets two parties create a shared secret over a public channel. An eavesdropper may see the public messages, but should not be able to compute the final shared secret.

The shared secret is usually not used directly as the final encryption key. It is fed into a KDF to derive traffic keys.

\[ traffic\ keys=KDF(shared\ secret, context) \]

Diffie-Hellman preview

Diffie-Hellman is the classic key exchange idea. The full math comes next, but the shape is this:

PartyPrivate valuePublic valueShared result
Alice\(a\)\(g^a\)\((g^b)^a=g^{ab}\)
Bob\(b\)\(g^b\)\((g^a)^b=g^{ab}\)
\[ (g^b)^a=(g^a)^b=g^{ab} \]

Alice and Bob arrive at the same shared value. An eavesdropper sees \(g^a\) and \(g^b\), but should not be able to recover \(g^{ab}\) without solving the hard problem.

The authentication problem

Key exchange alone does not automatically prove who is on the other side. If an attacker can sit between Alice and Bob, they may perform separate key exchanges with each side.

AttackWhat happens?Needed defense
Passive eavesdroppingAttacker only listens.Hard key-exchange problem may protect shared secret.
Man-in-the-middleAttacker substitutes public values.Authentication of public keys or transcript.
Fake public keyAttacker gives you their key instead of the real party’s key.Certificates, fingerprints, signatures, or trusted key directories.
Core warning

Public-key math can create a shared secret. Authentication tells you whom you shared it with.

Common mistakes

Mistake 1: thinking public means unsafe

Public keys are meant to be public. The private key is the secret.

Mistake 2: thinking key exchange automatically authenticates identity

Key exchange creates a shared secret. Authentication is a separate requirement.

Mistake 3: using shared secret directly as traffic key

Feed shared secrets into a KDF with context to derive separate keys.

Mistake 4: using one key pair for every job

Encryption, signing, and key exchange have different meanings and risks.

Practice problems

  1. What is the difference between a public key and a private key?
    Show solution

    A public key is meant to be shared; a private key must be kept secret and protected.

  2. What is a one-way function?
    Show solution

    A function that is easy to compute forward but hard to reverse.

  3. What is a trapdoor?
    Show solution

    Secret information that makes a one-way function easy to invert or use in a special private way.

  4. What does key exchange accomplish?
    Show solution

    It lets parties derive a shared secret over a public channel.

  5. Why is authentication still needed with key exchange?
    Show solution

    Because otherwise an attacker may perform a man-in-the-middle attack and establish separate secrets with both sides.

  6. Why do real systems often use hybrid cryptography?
    Show solution

    Public-key cryptography helps establish secrets and authenticate parties; symmetric cryptography is faster for protecting bulk data.

Why this matters for cryptography

Public-key cryptography changes the logistics of secrecy. It lets people publish keys, verify signatures, and create shared secrets over public networks. But it also introduces new questions: whose public key is this, what job is this key meant for, and how is the resulting shared secret bound to context?

  • Public-key cryptography uses public/private key pairs.
  • One-way functions are easy forward and hard backward.
  • Trapdoors are secret shortcuts.
  • Public-key encryption, signatures, and key exchange are different jobs.
  • Key exchange creates shared secrets over public channels.
  • Authentication is needed to prevent man-in-the-middle attacks.
  • Hybrid cryptography combines public-key setup with symmetric data protection.
  • Shared secrets should usually feed a KDF before becoming traffic keys.

One-page recap

What to remember
  • Public-key cryptography is asymmetric.
  • A key pair contains a public key and a private key.
  • The public key may be shared openly.
  • The private key must be protected.
  • A one-way function is easy to compute forward but hard to reverse.
  • A trapdoor is secret information that makes the hard operation easy for the key holder.
  • Public-key encryption lets others encrypt to your public key.
  • Digital signatures let others verify messages using your public key.
  • Key exchange lets parties derive a shared secret over a public channel.
  • Diffie-Hellman shape: \((g^b)^a=(g^a)^b=g^{ab}\).
  • Key exchange without authentication can suffer man-in-the-middle attacks.
  • Hybrid cryptography uses public-key methods for setup and symmetric methods for bulk protection.
