How to use this workbook

Focus on the central public-key move: one value may be shared openly while another related value must remain private. The public side opens doors for communication; the private side is the trapdoor, signature authority, or secret ingredient.

Note for self-study

The core idea is asymmetry: public information enables interaction, while private information gives the owner a special capability that attackers should not be able to reproduce.

Part A: Warm-up vocabulary

QuestionAnswer
A1. What is a public key?A key intended to be shared openly.
A2. What is a private key?A secret key mathematically related to the public key and protected by the owner.
A3. What is a key pair?A matched public/private key pair generated together.
A4. What is a one-way function?A function easy to compute forward but hard to reverse.
A5. What is a trapdoor?Secret information that makes a hard reverse or private operation easy for the holder.
A6. What is key exchange?A protocol that lets parties derive shared key material over a public channel.
A7. What is a shared secret?A secret value both parties compute and later feed into key derivation.
A8. What is authentication?Proving who or what is involved, preventing impersonation and active tampering.
A9. What does MITM stand for?Man-in-the-middle.

Part B: Symmetric versus public-key cryptography

TypeKeysMain challenge
SymmetricShared secret keyHow do both parties get the secret safely?
Public-keyPublic key plus private keyHow do we trust that a public key belongs to the right party?

B1 sample answer. Public-key cryptography reduces the need to pre-share a secret with everyone. Public keys can be distributed openly, but systems must still solve the problem of authenticating that the public key belongs to the intended person or server.

Part E: Trapdoor functions

IdeaPublic sidePrivate/trapdoor side
RSA-style intuitionPublic modulus and exponent allow public operation.Private exponent allows secret reverse operation.
Signature intuitionAnyone can verify with public key.Only private key holder can produce valid signature.
Encryption intuitionAnyone can encrypt to public key.Only private key holder can decrypt.

E1 sample answer. The trapdoor is the intended secret capability. It lets the legitimate private-key holder perform an operation that outsiders should not be able to perform.

Part F: Three major public-key jobs

JobWhat public key doesWhat private key does
Public-key encryptionLets others encrypt to you.Lets you decrypt.
Digital signaturesLets others verify your signature.Lets you sign.
Key exchangeLets parties combine public values.Lets each party compute shared secret.

F1 sample answer. Each job has different security meaning and different risks. A signing key proves authorship, an encryption key protects confidentiality, and a key-exchange key helps establish secrets. Mixing them can create protocol confusion.

Part G: Key exchange big idea

G1 sample answer. Key exchange lets two parties compute a shared secret over a public channel, even though an eavesdropper sees the public messages.

G2 sample answer. The shared secret is usually raw key material. A KDF binds it to protocol context and derives separate traffic keys for encryption, authentication, and directions.

Part H: Diffie-Hellman preview

PartyPrivate valuePublic valueShared result
Alice\(a\)\(g^a\)\((g^b)^a=g^{ab}\)
Bob\(b\)\(g^b\)\((g^a)^b=g^{ab}\)

H1 sample answer. Alice receives \(g^b\) and raises it to her private value \(a\), giving \(g^{ba}\). Bob receives \(g^a\) and raises it to \(b\), giving \(g^{ab}\). Since multiplication of exponents gives the same product, both reach \(g^{ab}\).

H2 sample answer. The eavesdropper sees \(g\), \(g^a\), and \(g^b\). It should remain hard to compute \(g^{ab}\) without knowing \(a\) or \(b\).

Part I: The authentication problem

AttackWhat happens?Needed defense
Passive eavesdroppingAttacker only listens.Hard key-exchange problem may protect shared secret.
Man-in-the-middleAttacker substitutes public values.Authentication of public keys or transcript.
Fake public keyAttacker gives you their key instead of the real party’s key.Certificates, fingerprints, signatures, or trusted key directories.

I1 sample answer. Key exchange can produce a shared secret, but without authentication you may share that secret with an attacker instead of the intended party. Authentication binds the key exchange to identity or trusted context.

Part J: Hybrid cryptography

LayerUsed forWhy?
Public-key cryptographyKey exchange, authentication, signatures.Solves the initial trust/secret-establishment problem.
Symmetric cryptographyBulk encryption and authentication.Fast and efficient for large data.

J1 sample answer. Public-key cryptography is useful for setup, identity, and key agreement, while symmetric cryptography is faster for protecting data. Hybrid systems use each tool where it shines.

Part M: Concept checks

M1 sample answer. Public-key cryptography uses a public/private key pair. The public key may be shared openly, while the private key must remain secret. This allows encryption to a public key, signature verification with a public key, or key exchange using public values.

M2 sample answer. A one-way function is easy to compute forward but hard to reverse. A trapdoor is special secret information that makes the hard reverse or private operation easy for the legitimate key holder.

M3 sample answer. Key exchange lets two parties derive a shared secret over a public channel. However, unauthenticated key exchange can be attacked by a man in the middle, so protocols need authentication of public keys, transcript data, or identities.

M4 sample answer. This tutorial introduced public-key cryptography as asymmetric cryptography based on public/private key pairs. It explained one-way functions, trapdoors, public-key encryption, digital signatures, and key exchange. Diffie-Hellman-style exchange lets two parties compute the same shared secret, but authentication is needed to prevent man-in-the-middle attacks. Real systems often use hybrid cryptography: public-key methods for setup and symmetric methods for bulk data protection.
