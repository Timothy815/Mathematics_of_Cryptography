Elliptic Curve Cryptography: Points, Addition, and Scalar Multiplication

A first climb onto elliptic curves: curve equations, points, the point at infinity, point addition, point doubling, scalar multiplication, public keys, and the elliptic-curve discrete logarithm problem.

🏠 Dashboard

Why this tutorial exists

Purpose

Tutorials 56 through 59 introduced public-key cryptography through Diffie-Hellman and RSA. This tutorial begins elliptic-curve cryptography. Elliptic curves give us public-key systems with smaller keys, elegant group structure, and a hard reverse problem based on point multiplication.

Elliptic-curve cryptography looks strange at first because the “numbers” are points, and “multiplication” means repeated geometric addition. It is arithmetic wearing a mapmaker’s coat: points, slopes, reflections, and a hidden trail that is easy to walk forward but hard to retrace.

What you should know first

You should know modular arithmetic, public/private key pairs, Diffie-Hellman’s “private exponent creates public value” shape, and the idea of a hard reverse problem. Elliptic curves reuse the same public-key theme, but replace modular exponentiation with scalar multiplication of points.

The elliptic curve equation

A common simplified elliptic-curve shape over ordinary real numbers is:

\[ y^2=x^3+ax+b \]

For cryptography, the curve is usually defined over a finite field, so the coordinates are modular values rather than continuous real-number coordinates. But the visual picture over the real numbers gives the first intuition.

Non-singular curve condition

For curves of the form \(y^2=x^3+ax+b\), we need the curve to avoid cusps or self-intersections. A common condition is:

\[ 4a^3+27b^2\ne0 \]

This keeps the curve smooth enough for the group law to work properly.

Points as group elements

In elliptic-curve cryptography, the objects we operate on are not ordinary numbers. They are points on a curve. The curve points form a group, meaning we can define a consistent addition operation.

In RSA, the action was exponentiating numbers. In ECC, the action is adding points again and again.

The point at infinity

Definition: point at infinity

The point at infinity, written \(\mathcal O\), is the identity element for elliptic-curve point addition. It acts like zero does for ordinary addition.

\[ P+\mathcal O=P \] \[ P+(-P)=\mathcal O \]

You do not need to picture \(\mathcal O\) as a normal point sitting somewhere on the graph. It is a formal point added to make the group operation complete.

Point negation

Over the real-number picture, if \(P=(x,y)\), then the opposite point is:

\[ -P=(x,-y) \]

The point and its negative have the same \(x\)-coordinate but opposite \(y\)-coordinates. On the graph, this is reflection across the \(x\)-axis.

Finite-field version

In modular arithmetic, \(-y\) means the additive inverse modulo the field prime. If the field modulus is \(p\), then \(-y\equiv p-y\pmod p\) when \(y\ne0\).

Point addition geometry

To add two distinct points \(P\) and \(Q\), draw a line through them. That line intersects the curve at a third point. Reflect that third point across the \(x\)-axis. The reflected point is \(P+Q\).

StepGeometric action
1Draw a line through \(P\) and \(Q\).
2Find the third intersection point with the curve.
3Reflect that third point across the \(x\)-axis.
4The reflected point is \(P+Q\).

Point doubling

To compute \(2P=P+P\), we cannot draw a line through two different points. Instead, we draw the tangent line at \(P\). That tangent intersects the curve at another point. Reflect that point to get \(2P\).

\[ 2P=P+P \]
StepGeometric action
1Draw the tangent line at \(P\).
2Find where the tangent line intersects the curve again.
3Reflect that point across the \(x\)-axis.
4The reflected point is \(2P\).

Curves over finite fields

The smooth curve drawing helps intuition, but real elliptic-curve cryptography usually works over finite fields. That means coordinates are reduced modulo a prime:

\[ y^2\equiv x^3+ax+b\pmod p \]

Instead of a continuous curve, we get a scattered set of valid points. The addition rules still work, but all arithmetic happens modulo \(p\).

Scalar multiplication

Definition: scalar multiplication

Scalar multiplication means adding a point to itself repeatedly.

\[ kP=\underbrace{P+P+\cdots+P}_{k\text{ times}} \]

For example:

ExpressionMeaning
\(1P\)\(P\)
\(2P\)\(P+P\)
\(3P\)\(2P+P\)
\(7P\)\(P+P+P+P+P+P+P\)

In real systems, scalar multiplication is computed efficiently with double-and-add or more advanced methods, not by literally adding one at a time.

Double-and-add intuition

Just as modular exponentiation can be sped up with repeated squaring, elliptic-curve scalar multiplication can be sped up with doubling and adding.

To compute \(13P\), use binary:

\[ 13=8+4+1 \] \[ 13P=8P+4P+P \]

The elliptic-curve discrete logarithm problem

Definition: Elliptic-Curve Discrete Logarithm Problem

Given \(G\) and \(Q=dG\), the elliptic-curve discrete logarithm problem is to recover \(d\).

\[ Q=dG \quad \text{easy if you know } d \] \[ d \quad \text{is hard to recover from } G \text{ and } Q \]

This is the ECC version of the public-key trap: easy to go forward by scalar multiplication, hard to reverse by finding the scalar.

Memory hook

Scalar multiplication is like taking many turns through a city of modular intersections. The public point tells where you ended up. The hard problem asks how many steps you took to get there.

Elliptic-curve Diffie-Hellman preview

Elliptic-curve Diffie-Hellman follows the same shape as classical Diffie-Hellman, but uses point scalar multiplication.

PartyPrivate scalarPublic pointShared point
Alice\(a\)\(A=aG\)\(aB=a(bG)=abG\)

\[ a(bG)=b(aG)=abG \]

As with classical Diffie-Hellman, the raw shared result should feed a KDF, and authentication is needed against man-in-the-middle attacks.

Common mistakes

Mistake 1: thinking ECC points are ordinary coordinates only

In cryptography, points are group elements with a special addition operation.

Mistake 2: thinking scalar multiplication is normal multiplication

\(kP\) means repeated point addition, not multiplying the coordinates by \(k\).

Mistake 3: ignoring the point at infinity

\(\mathcal O\) is the identity element that makes the point group complete.

Mistake 4: forgetting authentication

Elliptic-curve Diffie-Hellman still needs authentication to resist active attackers.

Why this matters for cryptography

Elliptic-curve cryptography is another way to build public-key systems from a hard reverse problem. RSA hides the factorization of \(n\). Diffie-Hellman hides exponents. ECC hides the scalar used to produce a public point. The result is compact, efficient public-key cryptography that appears throughout modern secure systems.

  • ECC operates on points on a curve.
  • Point addition combines two points to get another point.
  • The point at infinity \(\mathcal O\) is the identity element.
  • Point doubling means \(2P=P+P\).
  • Scalar multiplication means repeated point addition.
  • ECC public keys often have the form \(Q=dG\).
  • The private key is the scalar \(d\).
  • The public key is the point \(Q\).
  • The hard problem is recovering \(d\) from \(G\) and \(Q\).
  • ECDH mirrors Diffie-Hellman: \(a(bG)=b(aG)=abG\).
