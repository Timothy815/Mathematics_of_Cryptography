How to use this workbook

Focus on the conceptual machinery first: ECC uses points, not ordinary integers; point addition is the operation; scalar multiplication is repeated point addition; and the public key hides the private scalar.

Note for self-study

The central ECC idea is: a private scalar repeatedly adds a public base point to create a public point. Forward is efficient; reversing the scalar is hard.

Part D: Point at infinity and negation

ExpressionMeaning
\(P+\mathcal O=P\)Adding the identity changes nothing.
\(P+(-P)=\mathcal O\)A point plus its inverse gives the identity.
If \(P=(x,y)\), then \(-P=(x,-y)\)The inverse reflects across the \(x\)-axis in the real-number picture.
In modular arithmetic, \(-y\equiv p-y\pmod p\)The negative coordinate is the modular additive inverse.

D1 sample answer. \(\mathcal O\) is like zero because adding it to a point leaves the point unchanged.

Part E: Point addition geometry

StepGeometric action
1Draw a line through \(P\) and \(Q\).
2Find the third intersection point with the curve.
3Reflect that third point across the \(x\)-axis.
4The reflected point is \(P+Q\).

E1 sample answer. To add two distinct points, draw the line through them, find where it hits the curve again, then reflect that third point to get the sum.

Part F: Point doubling

StepGeometric action
1Draw the tangent line at \(P\).
2Find where the tangent line intersects the curve again.
3Reflect that point across the \(x\)-axis.
4The reflected point is \(2P\).

F1 sample answer. Adding two distinct points uses a line through both points. Doubling uses the tangent line at one point because both inputs are the same point.

J1 sample answer. The private key is a scalar \(d\). The public key is computed by scalar-multiplying the public base point \(G\) by \(d\), giving \(Q=dG\).

Part K: Elliptic-curve discrete logarithm problem

K1 sample answer. Given \(G\) and \(Q=dG\), the elliptic-curve discrete logarithm problem is to recover the hidden scalar \(d\).

K2 sample answer. Computing \(Q=dG\) is efficient using scalar multiplication. Reversing the process to find \(d\) from \(G\) and \(Q\) is believed to be computationally hard on well-chosen curves.

Part L: Elliptic-curve Diffie-Hellman preview

PartyPrivate scalarPublic pointShared point
Alice\(a\)\(A=aG\)\(aB=a(bG)=abG\)
Bob\(b\)\(B=bG\)\(bA=b(aG)=abG\)

L1 sample answer. Alice multiplies Bob’s public point by \(a\), giving \(a(bG)=abG\). Bob multiplies Alice’s public point by \(b\), giving \(b(aG)=abG\). Both get the same shared point.

L2 answer. A KDF is needed to derive keys from the shared point, and authentication is needed to resist man-in-the-middle attacks.

Part P: Concept checks

P1 sample answer. In ECC, the elements are points on an elliptic curve. These points form a group because we can add two points and get another point, use \(\mathcal O\) as the identity, and use \(-P\) as the inverse of \(P\).

P2 sample answer. Scalar multiplication means repeatedly adding a point to itself. \(kP\) does not multiply the coordinates by \(k\); it applies the elliptic-curve point addition operation repeatedly and efficiently.

P3 sample answer. An ECC private key is a scalar \(d\). The public key is \(Q=dG\), where \(G\) is a public base point. Computing \(Q\) from \(d\) is efficient, but recovering \(d\) from \(G\) and \(Q\) is the elliptic-curve discrete logarithm problem.

P4 sample answer. This tutorial introduced elliptic-curve cryptography as public-key cryptography built from points on a curve. It explained point addition, the point at infinity, point doubling, scalar multiplication, ECC public-key derivation, and the hard reverse problem. It also previewed ECDH, where Alice and Bob both compute \(abG\) from private scalars and public points.
