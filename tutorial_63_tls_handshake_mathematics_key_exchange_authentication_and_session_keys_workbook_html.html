How to use this workbook

Track the handshake as a sequence of linked jobs: negotiate choices, create shared material, authenticate identity, bind the transcript, derive keys, prove agreement, then use symmetric AEAD to protect records.

Quick reference

ECDHE core
\[ A=aG,\quad B=bG \] \[ S_A=aB=a(bG)=abG \] \[ S_B=bA=b(aG)=abG \]
TLS key schedule ideas
\[ transcript\ hash=H(handshake\ messages) \] \[ traffic\ keys=KDF(shared\ secret, transcript\ hash, labels, context) \] \[ Finished=MAC(finished\ key, transcript\ hash) \]

Note for self-study

TLS is a protocol braid: certificates authenticate, ECDHE contributes fresh secret material, transcript hashing binds the conversation, KDFs derive separated keys, Finished messages verify agreement, and AEAD protects records.

Part A: Warm-up vocabulary

QuestionAnswer
A1. What does TLS stand for?Transport Layer Security.
A2. What is ClientHello?The client’s opening message listing supported versions, cipher suites, extensions, random value, and key share.
A3. What is ServerHello?The server’s response selecting compatible parameters and providing its key share.
A4. What does ECDHE stand for?Ephemeral Elliptic Curve Diffie-Hellman.
A5. What does the certificate authenticate?The server identity and public key, assuming validation succeeds.
A6. What is the transcript?The ordered set of handshake messages used in hashes and authentication checks.
A7. What is a handshake secret?A secret derived during the handshake by the key schedule.
A8. What are traffic keys?Symmetric keys used to protect TLS records.
A9. What is the Finished message?An authenticated check over the transcript proving agreement on secrets and messages.
A10. What does AEAD stand for?Authenticated Encryption with Associated Data.
A11. What is forward secrecy?A property where later long-term key compromise does not automatically reveal past session keys.

Part B: TLS handshake big picture

StageMain job
NegotiationAgree on protocol version, cipher suite, groups, and extensions.
Key exchangeUse ephemeral Diffie-Hellman to create shared secret material.
AuthenticationUse certificates/signatures to prove server identity.
Key derivationUse a KDF to derive handshake and application traffic keys.
Finished verificationProve both sides agree on the transcript and keys.
Record protectionUse symmetric AEAD for application data.

B1 sample answer. The TLS handshake authenticates the peer, creates shared secret material, derives traffic keys, and verifies that both sides agree on the secure session.

Part D: Ephemeral key exchange

ExpressionMeaning
\(A=aG\)Client/Alice public ECDHE share.
\(B=bG\)Server/Bob public ECDHE share.
\(S_A=aB=a(bG)=abG\)Alice computes the shared point.
\(S_B=bA=b(aG)=abG\)Bob computes the same shared point.

D1 sample answer. The shared ECDHE value is raw shared material. It must go through the TLS key schedule/KDF to produce separated keys for handshake and application traffic.

D2 sample answer. Ephemeral means temporary and fresh for that session.

Part O: Concept checks

O1 sample answer. The TLS handshake negotiates protocol choices, performs ephemeral key exchange, authenticates the server with certificates, derives secrets through a KDF, verifies the transcript with Finished messages, and then protects application data with symmetric AEAD traffic keys.

O2 sample answer. Certificates authenticate the server identity and public key. They must validate for chain, trusted root, hostname, time, and usage. They do not directly encrypt all application data by themselves.

O3 sample answer. Transcript binding means handshake messages are hashed and included in key derivation and verification. Finished messages use derived keys to authenticate the transcript, proving that both sides saw the same handshake and derived matching secrets.

O4 sample answer. This tutorial connected certificates, ECDHE, KDFs, transcript hashing, Finished messages, AEAD, and forward secrecy into the TLS handshake. TLS uses public-key cryptography to authenticate and establish shared secret material, then uses symmetric traffic keys to efficiently protect data.
