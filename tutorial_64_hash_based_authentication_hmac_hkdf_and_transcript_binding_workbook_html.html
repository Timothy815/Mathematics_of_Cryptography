How to use this workbook

Treat this as a bridge between hash functions and full protocols. The core question is: when does a hash merely summarize data, and when does it authenticate, derive, or bind protocol state?

Quick reference

MAC and HMAC
\[ tag=MAC(K,m) \] \[ Verify(K,m,tag)=valid/invalid \] \[ HMAC(K,m)=H((K'\oplus opad)\parallel H((K'\oplus ipad)\parallel m)) \]
HKDF and transcript binding
\[ PRK=HMAC(salt,IKM) \] \[ OKM=HKDF\text{-}Expand(PRK,info,L) \] \[ Finished=MAC(finished\ key,transcript\ hash) \]

Note for self-study

The central idea is that hashes become protocol tools when they are keyed, labeled, contextualized, and bound to transcript state.

Part B: Plain hash versus keyed authentication

ConstructionUses secret key?What it proves
\(H(m)\)NoOnly a digest/fingerprint of the message.
\(HMAC(K,m)\)YesMessage was authenticated by someone who knows \(K\).

B1 sample answer. Anyone can compute a plain hash of a message, so a plain hash does not prove knowledge of a shared secret. A keyed MAC is needed for that.

Part C: Message Authentication Codes

C1 answer. A MAC provides message integrity and authenticity under a shared secret key.

C2 answer. Verification should fail because the tag no longer matches the modified message.

Part D: HMAC formula interpretation

PieceMeaning
\(K'\)The key adjusted to the hash block size.
\(ipad\)Inner padding constant.
\(opad\)Outer padding constant.
\(\oplus\)Exclusive OR.

D1 sample answer. HMAC is designed to be robust and analyzable. Simply hashing \(K\parallel m\) can interact badly with some hash constructions and is not the standard secure MAC construction.

Part F: HKDF big picture

StageFormula shapePurpose
Extract\(PRK=HMAC(salt,IKM)\)Condense raw input into a clean pseudorandom key.
Expand\(OKM=HKDF\text{-}Expand(PRK,info,L)\)Generate output key material for specific purposes.

F1 sample answer. Extract is like refining raw secret material into cleaner internal key material. Expand is like casting that material into separate tools for different protocol jobs.

Part G: HKDF-Extract

InputRole
\(IKM\)Raw input keying material, such as an ECDHE shared secret.
\(salt\)Optional non-secret value that helps extraction.
\(PRK\)Pseudorandom key output from extraction.

G1 sample answer. Raw shared material may not be perfectly uniform or directly suitable as key material. Extract condenses it into a cleaner pseudorandom key.

Part H: HKDF-Expand

InputRole
\(PRK\)Clean pseudorandom key from Extract.
\(info\)Context label describing the purpose.
\(L\)Length of output key material needed.
\(OKM\)Output key material.

H1 answer. HKDF-Expand produces output key material of the desired length for a specific context or purpose.

Part J: Transcript binding

J1 sample answer. The transcript hash changes, so transcript-bound derived keys or checks change too.

J2 sample answer. Transcript binding means tying keys or authentication checks to the exact protocol conversation, so changing the conversation changes the cryptographic results.

Part K: Finished messages

K1 sample answer. A Finished message proves that the party knows the finished key and agrees on the transcript hash.

K2 sample answer. The MAC is computed from both the finished key and transcript hash. If either differs, the recomputed MAC will not match.

Part O: Concept checks

O1 sample answer. A hash is an unkeyed digest of data, so anyone can compute it. A MAC uses a secret key and a message to produce a tag, so it can prove that someone with the shared secret authenticated that message.

O2 sample answer. HKDF has two main stages. Extract uses HMAC with a salt and input keying material to produce a pseudorandom key. Expand uses that pseudorandom key, context information, and a desired length to produce output key material for specific purposes.

O3 sample answer. Domain separation uses different labels or context strings so derived keys are not confused across purposes. Transcript binding includes a hash of protocol messages in derivation or authentication so keys and checks are tied to the exact conversation.

O4 sample answer. This tutorial showed how hashes become authentication and key-derivation tools. HMAC uses a secret key to authenticate messages, HKDF uses HMAC to extract and expand key material, labels provide domain separation, and transcript binding ties protocol keys and Finished checks to the exact handshake.
