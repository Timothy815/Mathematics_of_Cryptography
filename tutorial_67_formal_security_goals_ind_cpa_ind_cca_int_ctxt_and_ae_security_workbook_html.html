Part D: IND-CPA

1. Attacker may ask for encryptions of chosen messages.
2. Attacker chooses two equal-length messages m0 and m1.
3. Challenger flips secret bit b.
4. Challenger returns Enc(K, mb).
5. Attacker guesses b.
6. Secure means attacker cannot do much better than 50%.

Part F: IND-CCA

1. Attacker can ask encryption and decryption questions.
2. Attacker chooses m0 and m1.
3. Challenger returns challenge ciphertext C* = Enc(K, mb).
4. Attacker may continue asking allowed decryption questions, but not for C* itself.
5. Attacker guesses b.
6. Secure means attacker cannot do much better than 50%.

Note for self-study

The central idea is that “secure” becomes meaningful only after you define the attacker’s powers and the attacker’s winning condition.

Part D: IND-CPA

D1 sample answer. IND-CPA requires that even if an attacker can ask for encryptions of chosen plaintexts, they cannot distinguish which of two equal-length chosen challenge messages was encrypted better than chance.

D2 sample answer. If the messages have different lengths, the ciphertext length might reveal which message was encrypted, creating a trivial win unrelated to encryption strength.

Part E: Deterministic encryption and IND-CPA

Scheme behaviorIND-CPA problem
Same plaintext always gives same ciphertext.Attacker can build a codebook of encryptions.
Nonce/randomness not used correctly.Patterns may leak.
Repeated messages are visible.Equality information leaks.

E1 sample answer. Deterministic encryption leaks equality patterns. An attacker can ask for encryptions of known messages and compare them to the challenge ciphertext.

Part F: IND-CCA

F1 answer. The attacker gets controlled access to a decryption oracle, not just an encryption oracle.

F2 answer. If the attacker could decrypt the exact challenge ciphertext, the game would be trivial because the oracle would directly reveal the challenge plaintext.

Part G: Malleability and decryption oracles

PropertyMeaningRisk
MalleableCiphertext changes can cause controlled plaintext changes.Attackers may manipulate messages.
Decrypt-and-report errorsSystem reveals different failure reasons.Can become an oracle.
Authenticate before releaseInvalid ciphertexts are rejected.Limits decryption-oracle leakage.

G1 sample answer. Different error messages or timing behavior can reveal information about plaintext, padding, tags, or internal state. Attackers can use those reactions as clues.

Part H: INT-CTXT

H1 sample answer. INT-CTXT prevents attackers from creating a new ciphertext/tag pair that decrypts successfully unless it was previously produced by the legitimate encryption oracle.

H2 answer. Tamper resistance/integrity.

Part I: Authenticated Encryption security

GoalFormal flavorPlain-language meaning
ConfidentialityIND-CPA or strongerAttacker cannot learn which plaintext was encrypted.
IntegrityINT-CTXTAttacker cannot forge a valid new ciphertext.

I1 sample answer. It is like a locked box because it hides the plaintext, and like a tamper-evident seal because modified ciphertexts or associated data should be rejected.

Part J: Encrypt-then-MAC intuition

J1 answer. The MAC authenticates the associated data, nonce, and ciphertext.

J2 sample answer. Verifying first prevents attackers from using the decryption process as an oracle and prevents unauthenticated plaintext from being processed or trusted.

Part O: Concept checks

O1 sample answer. IND-CPA means indistinguishability under chosen-plaintext attack. The attacker can request encryptions of chosen messages, then chooses two equal-length challenge messages and receives an encryption of one of them. Security means the attacker cannot guess which one was encrypted much better than chance.

O2 sample answer. IND-CCA is a stronger confidentiality game where the attacker gets controlled decryption-query access. The attacker still must not distinguish the challenge encryption, and cannot simply ask for the challenge ciphertext to be decrypted.

O3 sample answer. INT-CTXT means attackers cannot forge new ciphertexts that decrypt successfully. AEAD security combines confidentiality with ciphertext integrity and associated-data binding, so messages are hidden and tampering is detected.

O4 sample answer. This tutorial explained how formal security games define attacker powers, challenges, and winning conditions. It covered advantage, IND-CPA, IND-CCA, INT-CTXT, malleability, encrypt-then-MAC intuition, and why AEAD combines confidentiality with integrity.
