How to use this workbook

Treat each “security level” as an estimated attack cost, not merely a number of stored bits. Your job is to connect parameter size, best known attack, data lifetime, and threat model.

Quick reference

Security and brute force
\[ n\text{-bit security}\approx 2^n\ attack\ work \] \[ keyspace=2^n \] \[ average\ brute\ force\ work\approx 2^{n-1} \]
Growth and birthday bound
\[ 2^{n+1}=2\cdot 2^n \] \[ 2^{n+10}=1024\cdot 2^n \] \[ collision\ work\approx 2^{n/2} \]

Note for self-study

The central idea is that bits of security estimate attack cost. Parameter size matters only after you ask what the best known attack against that parameter actually costs.

Part A: Warm-up vocabulary

QuestionAnswer
A1. What is the security parameter \(\lambda\)?The adjustable size parameter controlling attack difficulty.
A2. What are bits of security?An attack-cost estimate measured in powers of two.
A3. What is a work factor?The estimated computational effort required for an attack.
A4. What is brute force?Trying every candidate, such as every possible key.
A5. What is polynomial time?Runtime growing like \(n^c\) for some constant \(c\).
A6. What is exponential time?Runtime growing like \(2^n\) or \(c^n\).
A7. What does negligible mean?Too small to matter at cryptographic scale, formally shrinking very quickly with the security parameter.
A8. What is the birthday bound?The fact that collisions in an \(n\)-bit output become likely around \(2^{n/2}\) samples.
A9. What is a classical attacker?An attacker using ordinary non-quantum computers.
A10. What is a quantum attacker?An attacker using quantum algorithms/computers.
A11. What is a security margin?Extra buffer between current known attacks and the intended security target.

Part I: Symmetric versus public-key parameter sizes

Misleading thoughtBetter thought
“RSA-2048 means 2048-bit security.”RSA-2048 gives much less than 2048 bits of security.
“A 256-bit ECC key equals a 256-bit symmetric key.”A common 256-bit curve targets roughly 128-bit classical security.
“All key bits count the same way.”Security depends on the best known attack against the problem.

I1 sample answer. Parameter size is representation length. Bits of security estimate attack cost. RSA, ECC, hashes, and symmetric keys have different best known attacks, so their sizes translate differently into security levels.

Part J: Classical versus quantum attackers

Attack contextClassical estimateQuantum concern
Symmetric brute force\(2^n\)Grover-like search roughly \(2^{n/2}\) queries.
Hash collisionBirthday-style \(2^{n/2}\)Quantum collision algorithms can change estimates, but not simply by halving every security goal.
RSA factoringSub-exponential classical attacks.Shor breaks factoring efficiently on a large enough quantum computer.
Discrete logarithm / ECCExponential-ish generic attacks for ECC.Shor breaks discrete log efficiently on a large enough quantum computer.

J1 sample answer. Grover gives a quadratic speedup for search, reducing the effective exponent for symmetric brute force. Shor is more dramatic for RSA and traditional ECC because it attacks the mathematical trapdoor problems directly.

Part O: Concept checks

O1 sample answer. Bits of security estimate the cost of the best known attack in powers of two. A system with 128-bit security should require about \(2^{128}\) work to break by the best known method.

O2 sample answer. The birthday bound says that collisions appear much sooner than a full search would suggest. For an \(n\)-bit hash, collision attacks take about \(2^{n/2}\) work, so a 256-bit digest gives about 128-bit collision security.

O3 sample answer. Symmetric keys, hashes, RSA, and ECC have different mathematical structures and best known attacks. A 2048-bit RSA modulus, a 256-bit ECC key, and a 256-bit symmetric key do not all provide the same security level.

O4 sample answer. This tutorial explained how security parameters, bits of security, brute force, exponential growth, birthday attacks, public-key sizing, quantum attacks, parameter aging, and security margins shape practical cryptographic choices.
