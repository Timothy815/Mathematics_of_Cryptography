Quick reference

Constant-time comparison shape
result = 0
for i from 0 to length-1:
    result = result OR (a[i] XOR b[i])
return result == 0
Accumulated difference
\[ result = \bigvee_i (a_i \oplus b_i) \]

Any difference makes the final result nonzero.

Note for self-study

The central idea is that secure mathematics must be implemented without secret-dependent whispers through time, memory, power, errors, or physical behavior.

Part A: Warm-up vocabulary

QuestionAnswer
A1. What is a side channel?Information leaked through implementation behavior rather than intended algorithmic output.
A2. What is a timing leak?A leak where secret-dependent execution time reveals information.
A3. What is a cache leak?A leak where cache or memory-access behavior reveals information.
A4. What is power analysis?Analyzing power-consumption patterns to infer secret information.
A5. What is an error oracle?Error behavior that reveals hidden cryptographic information.
A6. What does constant-time mean?Avoiding secret-dependent timing, branches, memory access, and similar observable behavior where required.
A7. What is a branch?An if/else control-flow decision.
A8. What is a lookup table?A precomputed array/table used for fast lookup, potentially risky if indexed by secrets.
A9. What is blinding?Randomizing an operation so observations are not directly tied to the secret.
A10. What is masking?Splitting sensitive values into randomized shares to reduce leakage.
A11. What does fail closed mean?Rejecting safely on failure without revealing partial plaintext or helpful details.

Part C: Timing leaks

C1 sample answer. Early-exit comparison may take longer when more leading bytes match, allowing attackers to infer correct prefixes through timing measurements.

Part N: Concept checks

N1 sample answer. A side channel is an unintended leak through implementation behavior rather than intended output. Examples include timing, cache behavior, memory access, power use, electromagnetic signals, and error messages.

N2 sample answer. Timing leaks occur when secret data changes how long a computation takes. Constant-time comparison avoids early exit and processes every byte, accumulating differences so runtime does not reveal where the first mismatch occurs.

N3 sample answer. Cache leaks can occur when secret data controls memory access, while error oracles happen when different failures reveal useful clues. Fail-closed behavior rejects invalid cryptographic data generically before releasing or parsing plaintext.
