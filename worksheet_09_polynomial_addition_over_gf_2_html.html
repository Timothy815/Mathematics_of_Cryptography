Key rules

  • Duplicate terms cancel: same term in both polynomials gives coefficient \(1+1=0\), so it disappears.
  • Unique terms survive: a term in only one polynomial keeps its coefficient of 1.
  • XOR = polynomial addition mod 2: each bit position is independent, no carrying.
  • No carrying: \(1+1=0\), never 2.

Worked example: \(0\mathtt{x57} = x^6+x^4+x^2+x+1\) because 01010111 has 1-bits at positions 6, 4, 2, 1, 0.

Self-study note

For every polynomial addition, go term by term. If a term appears in only one polynomial, keep it. If a term appears in both, it cancels (coefficients \(1+1=0\) mod 2). There is never any carrying.

Part F: Self-XOR Property

F1. \((x^3+x+1)+(x^3+x+1) = 0\). Every coefficient is \(1+1=0\) mod 2.

F2. \(0\mathtt{x4F} \oplus 0\mathtt{x4F} = 0\mathtt{x00} = 0\).

F3. \(A \oplus A = 0\) for any byte \(A\). This is the self-inverse property of XOR.

Part G: Subtraction = Addition

G1. True. In ordinary integer arithmetic, \(5-3=2\) but \(5+3=8\) — different results.

G2. True. In GF(2), \(-1 \equiv 1 \pmod{2}\), so negating any coefficient leaves it unchanged. Thus \(-(x+1)=x+1\) over GF(2), and subtraction is identical to addition.

G3. Addition: \((x^4+x+1)+(x+1)\). The terms \(x\) and \(1\) each appear in both, so they cancel. Result: \(x^4\).

Subtraction: \((x^4+x+1)-(x+1) = (x^4+x+1)+(-x-1)\). Over GF(2), \(-x=x\) and \(-1=1\), so this equals \((x^4+x+1)+(x+1)\). Again the \(x\) and \(1\) terms cancel. Result: \(x^4\).

Both operations give \(x^4\). QED.

Part H: Concept Questions

H1 sample. In GF(2), all arithmetic is done modulo 2. Since \(2 \equiv 0 \pmod{2}\), the result of \(1+1\) is 0, not 2. The field GF(2) only contains the elements \(\{0,1\}\), so any result must be reduced mod 2 to remain in the field.

H2 sample. Each byte can be thought of as eight independent single-bit fields, one per bit position. XOR operates on each bit independently: it produces 1 when the two bits differ and 0 when they are the same. This is exactly the rule for adding two GF(2) coefficients — same result, different name. Because the polynomial representation assigns one coefficient per bit position, the two operations are identical.

H3 sample. The classmate used ordinary integer addition (adding the decimal values 87 + 87 = 174 = 0xAE). But polynomial addition over GF(2) is not integer addition — it is XOR. Each bit is added mod 2 independently with no carrying. The correct computation is \(0\mathtt{x57} \oplus 0\mathtt{x57} = 0\mathtt{x00}\) because XOR-ing any byte with itself always yields zero.

H4 sample. No carrying means each bit position is completely independent of all others. This makes GF(2) addition extremely simple to implement in hardware (a single XOR gate per bit), fast to compute, and easy to analyze algebraically. It also underpins the invertibility and diffusion properties that block ciphers like AES rely on.
